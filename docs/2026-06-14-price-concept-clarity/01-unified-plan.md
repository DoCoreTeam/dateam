# 가격 개념 — "통합" 단일뷰 기획서 (직전 분할안 폐기·재작성)

작성 2026-06-14 · 상태 기획(미구현) · 원칙: **나누지 말고 합친다**

---

## 0. 방향 전환

직전 안(출처 배지 4종 + 결정/대조 박스 분리)은 **개념을 더 쪼갠 것** = 통합 취지 역행. 폐기.
**정답 = 운영자가 "이 GPU 얼마에 팔지" 정하는 데 필요한 것만 남기고 나머지는 하나로 합친다.**

---

## 1. 핵심 통찰 — 지금 7종은 사실 3덩어리, gcube는 2개가 중복

### 7종 → 3덩어리로 합쳐진다
| 지금 (7종, 흩어짐) | 합치면 (3덩어리) |
|---|---|
| 공급원가 · 전파원가 | **① 원가** (우리가 사는 값. 실견적이든 추정이든 "원가 ₩X") |
| 추천 판매가 · 판매가 · 전략가 | **② 판매가** (우리가 파는 값. 단 하나의 숫자 = 전략가, 미설정이면 자동값) |
| 경쟁사 시장가 · gcube 파싱가 | **③ 시장 위치** (우리가 적정한가. 비교용 1신호) |

→ 운영자는 **"원가 → 판매가 → 시장에서 적정?"** 세 줄이면 끝. 7개 숫자를 따로 볼 이유 없음.

### gcube는 2개가 중복 — 하나로 통합해야 함 (혼란의 진짜 뿌리)
| 지금 | 정체 | 문제 |
|---|---|---|
| gcube 공시가(수동 입력, list 견적) | 사람이 손으로 적은 gcube 게시가 | **자동 파싱가와 중복.** 사람이 적은 값 ≠ 실제 사이트값 → 어긋남 |
| gcube 파싱가(자동 크롤) | gcube.ai에서 매일 긁은 실게시가 | 더 정확(실제값) |
→ **둘 다 "gcube 게시가"인데 출처가 달라 값이 어긋나고 헷갈림.** 통합 = **자동 파싱가를 단일 진실로**, 수동 입력은 (자동 실패 시) 백업으로만.

---

## 2. 통합 모델 — "판매가 결정" 한 화면, 세로 한 흐름 (박스 분리 X)

선택 GPU 상세 = **하나의 흐름**, 박스로 나누지 않음:

```
[ A100 160GB · 전용 고성능 ]

  우리 판매가      ₩14,889      ← ② 판매가(전략가). 이게 메인. 크게.
  └ 근거           원가 ₩11,911 + 마진 25%        ← ① 원가에서 비롯. 원가 없으면 "gcube 게시가 기준"
  └ 시장 위치      gcube 게시가 ₩11,970~17,100 안 · 경쟁사 중앙 대비 -5%   ← ③ 비교 1줄
  └ 사이트 반영    미반영  [반영 완료]            ← 실제 gcube.ai 적용 여부

  [수정] [추천가로 반영]
```

- **숫자 1개(판매가)** 가 주인공. 나머지(원가·시장·반영)는 그 아래 "근거/맥락" 한 줄씩.
- "추천가/판매가/전략가" 3개를 **화면에서 1개("우리 판매가")로** 합침 — 내부 계산은 그대로, 표시만 통합.
- "gcube 공시가 + gcube 파싱가" 를 **"gcube 게시가" 1개로** 합침(자동 파싱 우선).
- 경쟁사·gcube 비교를 **"시장 위치" 1줄**로 합침(별도 숫자 나열 X).

---

## 3. "원가 없는데 판매가" 혼란 — 통합 화면에선 자동 해소

근거 줄이 출처를 말해주므로:
- 원가 있음 → `근거: 원가 ₩11,911 + 마진 25%`
- 원가 없음 → `근거: gcube 게시가 기준 (원가 미확보 — 마진 측정불가)`
→ 별도 배지 추가 없이 **"근거" 한 줄**이 다 설명. (직전 안의 배지 4종 불필요)

---

## 4. 변경 요지 (통합 = 빼기 중심)

| 구분 | 지금 | 통합 후 |
|---|---|---|
| 화면 가격 숫자 | 추천가·판매가·전략가·공시가·파싱가 (5개 나열) | **판매가 1개** + 근거/시장/반영 각 1줄 |
| gcube 가격 | 수동 공시가 + 자동 파싱가 (2개) | **gcube 게시가 1개** (자동 우선, 수동 백업) |
| 시장 비교 | 경쟁사·gcube 따로 | **시장 위치 1신호** |
| 가격 결정 탭 구조 | 항목 나열 | **세로 1흐름**(판매가→근거→시장→반영) |

→ **추가 컴포넌트·배지·박스 없음.** 오히려 표시 항목을 줄이고 합침. 계산식/DB 불변(R1).

---

## 5. 단계 (전부 표시·통합, 계산 불변)
- **P1**: 상세 '가격 결정'을 "판매가 1개 + 근거/시장/반영 한 줄씩" 단일 흐름으로 재배치. (배지·구획 추가 안 함)
- **P2**: gcube "공시가/파싱가" 2개 라벨을 "gcube 게시가" 1개로 통합 표시(자동 파싱 우선, 수동은 폴백).
- **P3**: 경쟁사+gcube 비교를 "시장 위치" 1줄 신호로 통합.

---

## 6. 착수 전 결정 1개만 (이것만 정하면 됨)
**gcube 게시가 통합 시 — 수동 입력(현 공시가) 어떻게 할까?**
- (a) 자동 파싱가를 진실로, 수동 입력은 **자동 실패 시에만** 쓰는 백업으로 강등 (권장 — 단일 진실)
- (b) 수동 입력을 자동값 **오버라이드**로 유지(사람이 고치면 그게 우선)
> 이 하나만 정해주시면 P1부터 — 나머지는 전부 "합치고 줄이는" 표시 작업이라 추가 결정 불필요.
