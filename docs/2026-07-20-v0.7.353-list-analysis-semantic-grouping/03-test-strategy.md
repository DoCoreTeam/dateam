## 0. 이번 기능에서 테스트가 특별히 중요한 이유

이 기능은 **단위테스트 초록 + tsc 0인데 실화면이 완전히 망가진 상태**로 3세대를 보냈다.

- 미병합 브랜치는 `tsc 0`, `회귀 26/26`, 🟥 REV/QA 통과 기록이 있으나 **브라우저 실행 이력이 없다**
- 프로덕션은 마이그 161 미적용으로 **세션 저장 성공 이력 0건**이었는데 아무도 몰랐다

→ **실화면 검증이 없는 완료 선언을 금지한다.** (CLAUDE.md "실제 렌더 경로 우선 수정 정책")

## 1. 계층별 전략

### 1-1. 결정론 코어 (Phase 1) — 단위 테스트 중심

AI가 개입하지 않으므로 100% 재현 가능. 여기가 품질의 바닥이다.

| 대상 | 테스트 |
|---|---|
| `structure-tree.ts` | 마크다운 헤딩 / 번호체계(`1.`→`1.1`→`-`) / 들여쓰기 / 표 / 혼합 / 구조 없는 줄글 |
| `doc-meta.ts` | 버전·작성일·상태·작성자 식별. **본문에 등장하는 유사 문자열은 메타로 오분류하지 않을 것** |
| `assemble-groups.ts` | 절단 레벨별 그룹 경계, `source_span` 오프셋 정확성 |
| `coverage-check.ts` | 미귀속 0 / 미귀속 발생 / 경계 중복 / 빈 줄·공백 처리 |

### 1-2. 골든 픽스처 (T1.5) — 회귀 방지의 핵심

실제 문서 3종을 고정 입력으로 둔다:

| 픽스처 | 검증 기대값 |
|---|---|
| 요구사항정의서 | 요구사항 건수 = 그룹 수, 미귀속 0, 메타 4건 분리 |
| 회의록 | 안건 건수 = 그룹 수, 결정사항이 소속 안건에 귀속 |
| 서비스 기획서 (이번 141개 사고 원문) | **141개가 나오지 않을 것.** 그룹 수가 문서의 실제 섹션 수와 일치, `문서 버전: v0.1.0`이 그룹이 아니라 메타로 분류될 것 |

세 번째 픽스처는 **이번 사고의 재발 방지 테스트**다. 반드시 포함한다.

### 1-3. AI 단계 (Phase 2) — 계약 테스트 + 실호출 검증

AI 출력은 비결정적이므로 "정확한 문자열"이 아니라 **계약**을 검증한다.

| 대상 | 검증 |
|---|---|
| `classify-doc.ts` | 반환 스키마 준수, 미지원 유형 시 폴백, **지시에 유형이 명시되면 지시가 이김** |
| `cut-groups.ts` | 반환 스키마 준수, 절단 레벨이 트리 범위 내, 오염 키 거부 |
| 통합 | **동일 문서 + 다른 지시 → 그룹 개수·경계가 실제로 달라짐** ← Phase 2 완료 판정 그 자체 |

실 Gemini 호출 테스트는 골든 3종에 대해 수동 1회 이상 수행하고 결과를 문서화한다 (자동 CI에는 넣지 않음 — 비용·플레이키).

### 1-4. 서버 액션 / DB

| 대상 | 검증 |
|---|---|
| 마이그 166/167/168 | 적용 후 `information_schema.columns` 대조 — **파일 존재가 아니라 실DB 컬럼 확인** |
| RLS | 타 사용자 세션/문서 접근 차단 (default-deny) |
| `regroup` | revision 증가, 이전 리비전 보존, 원문 불변 |
| 멱등성 | 동일 `session+revision+group` 재실행 시 재과금 없음 |

### 1-5. 실화면 E2E (필수 — 생략 시 완료 불가)

Playwright로 **기본 렌더 경로**에서 수행한다.

| # | 시나리오 | 통과 조건 |
|---|---|---|
| E1 | 문서 붙여넣기 + 지시 → 실행 | 그룹 결과가 화면에 렌더됨 |
| E2 | 141개 사고 원문 재현 | **141개가 아닌, 문서 섹션 수 규모의 그룹** |
| E3 | 재지시 "카테고리 단위로 크게 묶어" | 그룹 수가 실제로 줄어듦 |
| E4 | 미귀속 표시 | "미귀속 0줄" 배지 렌더 |
| E5 | 문서 메타 패널 | 버전·작성일이 그룹이 아니라 메타에 있음 |
| E6 | 심화 실행 완주 | 전 그룹 done, 완성 문서 생성 |
| E7 | 429 재개 | 강제 중단 후 재개 시 성공분 유지 |
| E8 | 세션 저장 + 목록 | 저장 성공, "내 분석 문서"에 나타남 |

> E8은 Phase 0 완료 즉시 수행 가능 — 161 적용 검증을 겸한다.

## 2. 회귀 방지 가드

| 가드 | 목적 |
|---|---|
| **평탄화 재유입 차단** | `parseListItems` 계열 평탄화 추출이 신규 경로에 재도입되지 않는지 정적 스캔 |
| **지시 미전달 차단** | 추출 요청 payload에 `command`가 포함되는지 정적 검사 — 이번 사고의 핵심 결함 |
| **마이그 번호 중복 스캔** | `supabase/migrations/`에 동일 번호 prefix 2개 이상 존재 시 실패 |
| **에러 은폐 스캔** | error 분기에서 로그 없이 고정 문자열만 반환하는 패턴 탐지 |

앞의 두 개는 이번 재정의의 계약을 코드로 못 박는 장치다. `lib/datetime/kst-guard.test.ts`와 동일한 정적 스캔 패턴을 따른다.

## 3. 테스트 등록 (누락 주의)

`apps/web/package.json`의 `test` 스크립트는 **수기 파일 목록**이다. `*.test.ts`를 만들어도 자동 포함되지 않는다.

→ 신규 테스트 파일은 반드시 그 목록에 추가한다. 누락 시 초록으로 보이지만 실행되지 않는다.

## 4. 완료 선언 금지 조건

다음 중 하나라도 해당하면 **완료라고 말하지 않는다**:

- [ ] E2E 미수행 (tsc·단위테스트만 통과)
- [ ] 마이그레이션을 파일 존재로만 확인하고 실DB 컬럼 미대조
- [ ] 141개 사고 원문 픽스처 미포함
- [ ] "동일 문서 + 다른 지시 → 다른 그룹" 미검증
- [ ] 미귀속 0 검증이 AI 자기보고에 의존
