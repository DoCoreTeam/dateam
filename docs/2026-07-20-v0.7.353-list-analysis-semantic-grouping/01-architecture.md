## 1. 파이프라인

```
[문서 붙여넣기/업로드]  +  [자유 지시 한 줄]
            │
            ▼
 ① 문서 유형 판정            AI 1콜 (전체 원문)
            │                 "이 문서는 요구사항정의서다"
            ▼
 ② 구조 트리 복원            결정론 코드 (AI 아님)
            │                 헤딩/번호/들여쓰기 → 트리
            ▼
 ③ 그룹 절단                 AI 1콜 — 지시 + 유형 + 트리
            │                 "어느 레벨에서 자를 것인가"
            ▼
 ④ 그룹 조립 + 유실 검증      결정론 코드
            │                 원문 슬라이스 귀속, 미귀속 0 검증
            ▼
      ┌── 그룹 결과 제시 ──┐
      │                    │
      │  "더 크게 묶어"    │ → ③으로 (재그룹핑, 새 리비전)
      │                    │
      └── 확정 ────────────┘
            │
            ▼
 ⑤ 넓이 패스 (선택)          AI 1콜 — 빠진 그룹 후보 제안
            │                 사용자 확인 후에만 추가
            ▼
 ⑥ 그룹별 재가공             AI N콜 (그룹당 1, 병렬)
            │                 유형+전체맥락+그룹원문 전체 투입
            ▼
 ⑦ 정합 패스 + 결정론 조립    코드 (+크리틱 1콜, 비차단)
            │
            ▼
 ⑧ 완성 문서 → 4경로 배출
```

### 핵심 설계 원칙

1. **①③⑤⑥⑦만 AI.** ②④는 결정론 코드 — 유실 검증을 AI 자기보고에 맡기지 않는다.
2. **지시는 ①부터 개입한다.** 현행처럼 ⑥에만 주입하지 않는다. 이것이 이번 재정의의 본질.
3. **재그룹핑 루프(③↔결과)는 ⑥ 이전에 위치한다.** 비싼 단계(⑥ N콜) 전에 구조를 확정하므로 비용이 낮다.
4. **원문 슬라이스는 절대 재작성하지 않는다.** 그룹은 원문을 "가리키는" 것이지 "다시 쓴" 것이 아니다.

### 2-1. 개념

```
문서(session) 1
   ├── doc_meta          문서 속성 (버전/작성일/상태/작성자)
   ├── structure_tree    복원된 원문 구조 트리

```

## 6. 유실 0 보증 메커니즘 (FR-6 상세)

```
원문 전체 줄 집합  L
그룹 귀속 줄 집합  G  = ⋃ group.source_span
문서 메타 줄 집합  M

검증:  L \ (G ∪ M) == ∅
```

- **결정론 코드로 계산.** AI에게 "빠짐없이 했는가" 묻지 않는다.
- 미귀속이 발생하면 화면에 줄 수와 원문을 **명시 노출**하고, 원클릭으로 그룹에 편입하거나 새 그룹으로 승격할 수 있게 한다.
- `source_span` 오프셋 기반이므로 AI가 원문을 재작성해도 검증이 무너지지 않는다 (재작성 자체를 금지하지만, 방어선을 이중으로 둔다).

## 7. 비용·실패 설계 (NFR-2, NFR-5)

| 단계 | AI 콜 | 비고 |
|---|---|---|
| ① 유형 판정 | 1 | 전체 원문 1회 |
| ③ 그룹 절단 | 1 | 트리 요약 투입 (원문 전체 아님) |
| 재그룹핑 | 1/회 | 저렴 — 사용자가 부담 없이 반복 가능 |
| ⑤ 넓이 패스 | 1 | 선택 |
| ⑥ 재가공 | **N** (그룹 수) | 병렬. 유료 키 전제 |
| ⑦ 크리틱 | 1 | 비차단 — 실패해도 문서 산출 |

**429 처리**: 정상 경로로 취급. 자동 백오프 → 재개, 진행률 DB 영속(크론 드레인 유지), 사용자에겐 대기 상태 안내. 성공한 그룹은 재과금하지 않는다(멱등 키 = `session_id + revision + group_id`).
