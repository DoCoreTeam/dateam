# 요구사항 — 모바일 반응형 전체 적용

## 배경
newAX 웹앱은 현재 전체 레이아웃이 inline style 기반으로 구현되어 있어 모바일에서 완전히 깨짐.
Sidebar가 고정 220px로 모바일 화면 폭을 차지하고, 콘텐츠 그리드가 모두 고정 컬럼 수를 가짐.

## 목표
- 모바일(375px) 기준 전 페이지 정상 렌더링
- 터치 친화적 인터랙션 (버튼 최소 44px, 충분한 여백)
- 데스크탑 레이아웃 유지 (변경 없음)

## 브레이크포인트 정책
| 이름 | 폭 | 대상 |
|------|-----|------|
| mobile | < 768px | 스마트폰 세로 |
| tablet | 768px ~ 1024px | 태블릿, 스마트폰 가로 |
| desktop | > 1024px | PC |

## 반응형 전략
1. **Sidebar**: 모바일에서 숨김 → 상단 햄버거 버튼 → 드로어(overlay) 방식
2. **레이아웃**: 모바일에서 단일 컬럼 스택
3. **그리드**: `gridTemplateColumns` 모바일 1col / 태블릿 2col / 데스크탑 유지
4. **테이블**: 모바일에서 가로 스크롤 또는 카드 뷰
5. **폼**: 모바일에서 풀 폭

## 구현 방식
- globals.css에 CSS 변수 및 미디어쿼리 유틸 추가
- 레이아웃 컴포넌트는 Tailwind 반응형 클래스 + CSS 클래스명 활용
- 페이지 컴포넌트는 className 추가 + globals.css 반응형 규칙으로 처리
