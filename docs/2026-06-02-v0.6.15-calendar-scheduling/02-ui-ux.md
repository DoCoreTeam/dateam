# 02 · UI/UX — 등록 모달·추천·연계

## A. 등록 흐름 (D2 — 날짜클릭 모달 + 자연어)
```
[월/주 뷰] 날짜 셀 클릭(또는 "+ 일정")
        ▼
┌─ 일정 등록 ───────────────────────────────┐
│ 💬 자연어: [ 내일 오후 3시 A사 제안 미팅 1시간 ]  [AI 파싱] │
│    ↓ (analyze-work 재사용, 결과로 아래 폼 자동 채움)        │
│ 제목   [ A사 제안 미팅 ]                                  │
│ 시작   [2026-06-03 15:00]   종료 [16:00]  □ 종일          │
│ 반복   [안 함 ▾] (매일/매주 N요일/매월…)                  │
│ 연계   [일일업무 ▾] → "A사 제안서 작성" 검색·선택 (선택)    │
│ 설명   [ ... ]                                            │
│            [취소]            [저장]                        │
└──────────────────────────────────────────┘
```
- 자연어 칸은 **선택** — 비우고 폼만 채워도 OK. 채우면 AI가 폼 프리필, 사용자 검토 후 저장(자동 저장 아님).
- 연계(link): 드롭다운으로 kind 선택 → 해당 객체 검색(미연결 'none' 기본).
- 모바일: 풀스크린 시트, 입력 세로 스택, 터치 44px.

## B. 캘린더 표시 (일정 vs 업무 구분)
- 한 캘린더에 **calendar_events(일정)** + **daily_logs(업무·메모)** 병합.
- 범례/색: 일정=파랑 점·시간표기 / 업무(planned 등)=상태색 / 메모=회색. 아이콘으로 종일·반복(↻) 표시.
- 일정 클릭 → 상세 팝오버: 제목·시간·반복·**연계 업무 링크(클릭 시 /daily 또는 주간보고로 점프)**·[편집][삭제][완료].

## C. AI 추천 (D3)
```
캘린더 상단 또는 우측 "🔮 추천 일정" 카드 (수동 [추천 받기] 버튼)
┌ 추천 일정 (주간계획·미처리메모 기반) ──────────┐
│ • 화 10:00  B사 팔로업    근거: 주간보고 '계획'   [등록][무시] │
│ • 목 14:00  메모 후속 검토 근거: 5일 전 미처리 메모 [등록][무시] │
│ • 금 09:00  영진기술 컨택  근거: 이월된 planned    [등록][무시] │
└──────────────────────────────────────────┘
```
- [등록] → createCalendarEvent(source='ai', link_kind/link_id 자동 연결).
- [무시] → 추천에서 제외(세션/플래그).
- 근거(reason) 노출로 신뢰·설명가능성 확보.

## D. 권한 시각화 (D4)
- 본인 일정 = 편집 가능. 부서장이 관할 팀원 일정 조회 시 = **조회 전용(🔒)**, 색 구분(타인 일정 옅게).
- 부서장 뷰 토글: "내 일정 / 팀 일정"(관할 팀원 일정 오버레이) — 기존 조직 계층 가시성과 일관.

## E. 컴포넌트 (재사용/신규)
| 컴포넌트 | 비고 |
|---------|------|
| `EventModal`(신규) | 등록/편집 모달, 자연어칸+폼. EditorModal/analyze-work 패턴 참고 |
| `EventPopover`(신규) | 일정 상세·연계 점프·액션 |
| `RecommendCard`(신규) | AI 추천 카드 |
| `CalendarMonth/Week`(기존 확장) | 일정 레이어 추가, daily_logs 병합 |
| `DayDetailPanel`(기존 확장) | 읽기전용 → "+ 일정" 진입점 추가 |

## F. 반응형/접근성
- 320~1440 월/주 뷰 + 모달 카드. 모바일 월뷰는 점·개수 요약, 탭 시 일 상세.
- 키보드: 날짜 셀 포커스+Enter로 모달, 모달 트랩/Escape. aria-label, 색대비 4.5:1.
