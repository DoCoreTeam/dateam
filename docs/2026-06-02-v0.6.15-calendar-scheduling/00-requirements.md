# 00 · 요구사항 — 업무 연계형 AI 캘린더 일정 시스템

> 상태: **기획 전용 — 구현 금지** · 기준 v0.6.15 · 2026-06-02
> 산출: DOMANGCHA CEO 파이프라인 (🟦 DC-ANA 캘린더 현황 + 4대 결정 Q&A)

## 1. 문제 (현황)
캘린더(`/calendar`)는 **daily_logs를 날짜뷰로 읽기만** 한다. 일정 "등록"이 불가능한 근본 원인 3중(DC-ANA 확인):
1. **API 부재** — `/api/calendar/`엔 `GET month` 하나뿐, POST 없음.
2. **UI 부재** — `DayDetailPanel`은 읽기 전용, 날짜 클릭 시 `/daily`로 이동만.
3. **데이터 모델 미비** — daily_logs엔 `scheduled_at`(단일 시각)만, start/end/all_day/반복 없음.
- 재사용 가능 자산: AI 파싱 파이프(`/api/ai/analyze-work` → `targetDate`+`scheduledTime` → `scheduled_at`), 조직 계층 RLS 헬퍼(`private.my_readable_user_ids` 등).

## 2. 사용자 요구 (원문 정제)
1. 캘린더에서 **날짜 클릭/UI로 일정 등록**.
2. **자연어 입력**으로도 등록(AI가 날짜/시간 파싱).
3. **AI로 다음 일정 추천** — 일일업무·주간보고·메모 데이터를 종합해 확장.
4. 단순 이벤트가 아니라 **"업무적 등록"** — 캘린더가 업무와 연결.

## 3. 확정된 아키텍처 결정 (Q&A)
| # | 결정 | 선택 |
|---|------|------|
| D1 | 데이터 모델 | **신규 `calendar_events` 테이블 + 업무 링크**(`link_kind`/`link_id`로 daily/weekly/memo 연결) |
| D2 | 등록 UX | **날짜 클릭 모달 + 자연어 1줄**(analyze-work 재사용으로 폼 자동 채움) |
| D3 | AI 추천 | **규칙 1차 + Gemini 2차 종합**(daily+weekly+memo 컨텍스트) → 캘린더 "추천" 카드 |
| D4 | 권한/공유 | **개인 기본 + 조직 계층 RLS 적용**(부서장은 관할 팀원 일정 조회 — 기존 헬퍼 재사용) |

## 4. "업무적 등록"의 의미 (핵심 개념)
일정은 단독 이벤트가 아니라 **업무 객체와 연결**될 수 있다:
- `link_kind = 'daily' | 'weekly' | 'memo' | 'none'`, `link_id` = 그 객체 id.
- 예: "A사 제안서 제출"(일일업무)에 마감 일정 연결 / 주간계획 항목을 일정으로 / 미처리 메모를 후속 일정으로.
- 효과: 캘린더에서 일정→원본 업무로 점프, 업무 완료 시 일정 상태 동기화(선택), AI가 "이 업무 관련 다음 일정" 추천 가능.

## 5. 비기능 요구
- 보안: 일정 RLS는 **본인 + 조직 계층(관할)** — 기존 `private` 헬퍼 재사용, 플래그 게이팅 일관.
- 성능: 월뷰 1쿼리(범위 조회 + 인덱스), 반복 일정은 저장은 rule, 조회는 전개(expand) 캐시.
- AI 비용: 추천은 **수동 트리거 또는 주1회 위젯**(자동 매요청 호출 금지), 규칙으로 1차 필터 후 Gemini 1회.
- 반응형: 월/주/일 뷰 + 모달 모바일 카드, 터치 44px.
- i18n/접근성: 키보드 네비, aria, 색대비.

## 6. 범위
**포함**: calendar_events 데이터모델·등록(클릭모달/자연어)·AI추천(규칙+Gemini)·daily/weekly/memo 연계·계층 RLS·뷰(월/주) 설계.
**제외**: 외부(구글) 캘린더 양방향 동기화 상세(원칙만), 화상회의 연동, 알림(푸시) 상세.

## 7. 성공 정의
- 날짜 클릭 → 모달 → (자연어 or 폼) → 일정 저장 → 캘린더 즉시 반영.
- 일정이 업무(daily/weekly/memo)에 연결되어 상호 점프.
- "추천 일정" 카드에서 원클릭 등록.
- 부서장이 관할 팀원 일정 조회(개인 기본·계층 적용).
