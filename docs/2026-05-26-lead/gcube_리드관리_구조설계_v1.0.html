리드 관리 구조 설계 — 4객체 & 듀얼 입력

리드는 프롬프트, 명함 스캔, 음성, 파일 업로드의 멀티모달 AI 인테이크로 받는다. AI가 입력을 파싱해 충분하면 자동으로 채우고 부족하면 되물어 채운 뒤, 표준 CRM 4객체(거래처·담당자·영업기회)로 적재한다. 민간 고객 DB와 공공 수요예보 엑셀은 파일 업로드로 대량 적재한다.
구조 거래처·담당자·영업기회 + 리드 인테이크
입력 프롬프트·명함·음성·파일
AI 파싱·자동채움·되물음
01

구조 한눈에

구조가 안 잡히던 이유민간 DB는 회사 단위(회사 1개 = 리드처럼 보임), 공공 수요예보는 사업 단위(기관 1개가 사업 여러 개 보유)다. 단위가 달라 한 표에 못 담는다. 해법은 표준 4객체로 나누고, 모든 입력을 하나의 AI 인테이크로 받아 같은 거래처에 매다는 것이다.

02

3객체 필드 스키마 (밸리데이션·어드민)

리드를 제외한 세 객체의 모든 항목은 처리 방식, 밸리데이션(검증) 정책, 어드민 노출 설정을 함께 정의한다. 수치 계산 항목만 수동 입력이 불가하고, 그 외에는 AI가 채운 값도 직접 수정할 수 있다.
파일 파일 임포트 가능   수동 직접 입력   AI AI 채움·추천(수정 가능)   자동계산 수동 불가   노출숨김 어드민 토글   * 필수
제약사항 — 초기 적재는 파일 임포트초기 데이터는 직접 입력이 아니라 두 종류의 엑셀 파일 업로드로 적재한다([파일] 배지가 그 대상 항목이다). 파일 종류·시트·칼럼 구조와 객체 매핑은 05장에 정확히 명시했다. 관리자는 항목별로 노출·숨김, 필수·선택, 밸리데이션 규칙(형식·선택목록·범위·중복키)을 설정한다.

리드 인테이크 — 4모드와 AI 채움

영업담당은 양식을 채우지 않는다. 떠오르는 대로 쓰거나, 명함을 찍거나, 말하거나, 엑셀을 올리면 AI가 알아서 정리한다.

프롬프트

자연어로 메모. "오늘 OO 김부장 미팅, H100 5장 검토".

음성

말로 입력. STT로 텍스트 변환 후 동일하게 파싱.

파일 업로드

공통 규칙

중복 판정 키

거래처는 사업자번호(없으면 정규화 회사명), 담당자는 이메일, 영업기회는 거래처와 사업명 조합으로 중복을 막는다.

1대 다 허용

한 거래처에 영업기회 여러 개. 공공 기관은 사업이 여러 개이기 때문이다.

단계·확률 연동

단계를 바꾸면 확률이 자동으로 따라 바뀌고, 본부 전환율 KPI가 산출된다.

04

AI 레이어 (필수)와 구현

AI가 붙는 다섯 자리

1. 적합도 스코어링

GPU 수요 강도, 규모, 타이밍, 접점을 넣어 0–100 자동 채점. 지금의 수작업 강도 평가를 자동화하는 자리.

2. 자동 보강

회사명·기관명만으로 사업내용·투자·매출·뉴스를 채운다. 공공 사업은 발주 담당부서를 찾아 붙인다.

3. 중복 제거·매칭

같은 거래처·사업을 자동 병합. 대학·병원은 민간과 공공 양쪽에 나오므로 기관명·사업자번호로 합친다.

4. 세그먼트·라우팅

T1·T2·공공으로 자동 분류하고 담당 영업에게 배정한다.

5. 다음 액션 추천

단계·강도에 맞는 제안(무마진 PoC, 등록 20% 쿠폰, 하이퍼큐브)과 멘트를 제시한다.

적재 트리거

파일 임포트 직후, 수동 저장 직후, 단계 변경 시 AI가 자동 실행되도록 워크플로우에 건다.

05

초기 적재 파일 구조와 매핑

제약사항초기 데이터는 직접 입력이 아니라 파일 임포트로 적재한다. 두 종류의 엑셀을 업로드한다. 스키마에서 [파일] 배지가 붙은 항목이 이 파일로 채워지는 칸이다. 임포트 시 거래처·담당자·영업기회 중복은 자동 병합되고, 비어 있는 칸은 AI가 보강한다.
