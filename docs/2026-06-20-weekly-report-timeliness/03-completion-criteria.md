# 완료 기준 (구현 시 검증 항목 — 현재 미진행)

## 기능
- [ ] 주간보고 작성/수정 시 `weekly_report_activity`에 append-only 기록(수정해도 이전 이력 보존).
- [ ] `weekly_report_activity` UPDATE/DELETE가 RLS로 차단됨(불변 증빙).
- [ ] 부서 취합 confirm 시 `confirmed_at` 기록, 재취합 시 최신값 갱신.
- [ ] 판정룰 정확성(단위테스트):
  - [ ] 금요일 내 작성+취합 전 → 🟢 정시
  - [ ] 토 00:00~월 00:00 작성 → 🟡 1차 지연
  - [ ] 취합 이후 원본 수정 → 🟡 지연
  - [ ] 다음 주 월 00:00 이후/미작성 → 🔴 최종 지연
  - [ ] 재취합해도 토/월 캘린더선 고정(지연 우회 불가)
- [ ] org/team 탭에서 멤버별 배지·타임라인·부서 정시율 표시(실제 활성 렌더 경로에서 확인).
- [ ] 로그인 시 미작성/지연자에게 작성 안내 모달(안내 톤), 본 뒤 재안내 억제.
- [ ] admin 전용 정시율/이력 CSV export. 비admin 차단.

## 권한/보안
- [ ] 상태 가시성 = 보고서 가시성과 동일(org-scope RLS 상속), 권한 외 노출 없음.
- [ ] 증빙 export는 admin role만(서버측 `requireAdmin`).

## 품질
- [ ] 배지 색/라벨·포맷 SSOT(토큰·공용함수), 뷰별 인라인 색맵/포맷 없음.
- [ ] `pnpm design:check` 통과, tsc/lint/test green.
- [ ] 기존 데이터 보호: 백필이 기존 보고서 본문/상태플래그를 변형하지 않음.

## 확정 정의
- 캘린더선: 토 00:00 = week_start+5일, 월 00:00 = week_start+7일(다음 주 시작). (미해결 없음)
