# 00 Requirements — 가격 결정 콕핏

## 배경
GPU 가격 신호(공시·공급원가·자동마진가·시장가·전략가)가 4탭에 산재. 자동마진가만 있고 "사람이 정한 전략가"가 없어(Tier3 direct_prices 제외) 비교·결정이 어렵다. 60대 사용자도 한눈에 보는 간결 일원화 필요.

## 확정 결정 (사용자)
- 전략가(strategic_price) 1급 필드 신설 (자동마진가와 분리)
- 신규 "가격 결정" 콕핏 탭 (가격표 탭 유지), 고객판매가격표 흡수
- 전체 기획 한번에 구현 / 60대 간결 / 디자인 토큰 준수

## 기능 요구
- FR1: 모델(구성) 한 행에 [원가·자동마진가·🎯전략가·시장중앙·실효마진%·시장편차%] 동시 표시
- FR2: 🎯전략가 인라인 편집(연필) — 유일 편집지점, 미입력시 자동마진가 fallback(흐림)
- FR3: 3색 시그널 — 실효마진(🔴<10/🟢15~25/🔵>25), 시장편차(🔴>+10/🟢±10/🔵<-10)
- FR4: 행 펼침 — 공시가·시장 min/max·공급사 분포·전략가 변경이력
- FR5: 고객판매가격표가 전략가를 출력(흡수)
- FR6: 60대 간결 — 금액폰트≥18px, 장황 카피 제거(설명은 툴팁/펼침)

## 비기능 (가드레일)
- 정합 SSOT(buildCatalog read-time) 위 구축, 회귀 0
- 디자인 토큰만(하드코딩 0), 공용클래스 SSOT, table-card 반응형, 인라인 style 금지
- admin 게이트 + 전략가 변경 audit
- git push 금지(커밋까지)

## 제외
- 가격 알고리즘 신규 도입, 시장 수집기 변경, 가격표 탭 제거
