# 캘린더 업무/일정 구분 — 디자인 기획 (구현 전, 보고용)

> v0.7.194 후보 · 2026-06-18 · **기획만 — 코드 구현 없음**
> 근거: 🟦 DC-ANA 실코드 조사 + 사용자 요구(3차 누적)

## 0. 사용자 요구 (확정)
1. 캘린더에 **업무·일정이 섞여 에러처럼** 보임 → **셀·우측 상세창 양쪽**에서 업무/일정 구분 필요.
2. 말한 것 외에도 **표기 시 헷갈리는 건 모두 명확히**.
3. **업무·메모는 작성일시 ≠ 마감일시**라 둘을 구분해야 안 헷갈림. **일정은 해당 일정의 시간**이 핵심.

## 1. 문제 — 왜 "에러처럼" 보이나 (원인, DC-ANA 확인)
캘린더는 **두 개의 독립 소스**를 한 화면에 표식 없이 겹쳐 그린다.
- **일정** = `calendar_events` (셀: `.cal-event-chip` 파란칩 / 패널: "일정" 섹션 파란카드)
- **업무·메모** = `daily_logs` (셀: `.cal-preview-{entry_type}` 색칩 / 패널: 레이블 없는 목록)

여기에 **자동 복제**가 겹친다:
- `daily/page.tsx`의 `autoRegisterSchedules()` → `scheduled_at` 또는 `target_date`가 있는 업무를 **`calendar_events`로 자동 INSERT**(`source='rule'`, `link_kind='daily'`, `link_id=업무id`).
- 결과: **같은 제목이 "일정(자동생성)"과 "업무" 양쪽에** 같은 날 셀·패널에 동시 노출 → 중복/에러로 보임. (스샷의 "시넥스 미팅 / NHN…"이 일정 섹션과 예정 업무에 모두 등장)

추가로 발견된 혼동 표기(요구 2 대상):
- **(A) 시간대 오프셋 버그**: 셀의 일정칩은 `ev.start_at.slice(11,16)`로 **원시 UTC 문자열**을 잘라 표시(01:00). 패널은 변환된 값(10:00). → **같은 일정이 셀 01:00 / 패널 10:00**으로 9시간 어긋나 보임. (KST=UTC+9)
- **(B) 업무 시각의 의미 불명**: 패널의 "예정 09:47"은 업무의 시각인데, **작성/기록 시각인지 마감인지 라벨이 없어** 일정 시간처럼 오인.
- **(C) target_date 이중 노출**: 업무의 `target_date`가 작성일과 다르면 `month` API가 **두 날짜 셀 모두**에 같은 업무를 표시(작성일 셀 + 마감일 셀).
- **(D) 범례 누락**: 월 하단 범례는 entry_type 5색만, **일정(calendar_events) 칩은 범례에 없음**.
- **(E) 패널 섹션 무경계**: "일정" 섹션 아래 업무 목록에 **헤더가 없어** 한 덩어리로 보임.

## 2. 설계 원칙
1. **타입 = 1차 시각 축**: 일정/업무/메모는 색이 아니라 **아이콘+섹션+라벨**로 먼저 구분(색은 보조). 색만으로는 entry_type과 충돌.
2. **시간의 의미를 라벨링**: "시간"을 무라벨로 던지지 않는다. 일정=「시작」, 업무·메모=「마감(예정)」 / 「작성」을 명시.
3. **중복은 연동으로 전환**: 자동생성된 일정과 원본 업무가 같이 보일 땐 "중복"이 아니라 "연동(업무→일정)"임을 표식하거나, 한쪽을 대표 표시.
4. **시간대 SSOT**: 모든 표면이 동일 변환(KST)로 시각 표시. 원시 슬라이스 금지.
5. 디자인 토큰·공용 컴포넌트 경유(인라인 하드코딩 금지), 모바일 카드/반응형 유지.

## 3. 타입 구분 체계 (셀 + 패널 공통 시각 언어)
| 타입 | 소스 | 아이콘(예) | 색 토큰 | 시간 라벨 |
|---|---|---|---|---|
| **일정** | calendar_events | 📅 (CalendarClock) | `--brand` 계열 | **시작** HH:MM (KST) |
| **업무** | daily_logs(personal, note 아님) | ✓ (CheckSquare) | entry_type 색(예정/진행/완료/블로커) | **마감/예정일** (+필요시 작성일 보조) |
| **메모** | daily_logs(entry_type='note') | 📝 (StickyNote) | `--warning` 계열 | **작성일** (마감 개념 약함) |

- 셀·패널 모두 **항목 앞에 타입 아이콘**을 둬, 색과 무관하게 한눈에 일정/업무/메모 식별.
- 패널은 **"일정" / "업무" / "메모" 3개 섹션 헤더**로 분리(현재 업무 섹션 무라벨 → 헤더 추가, 섹션 간 구분선·여백). (E 해소)

## 4. 시간 의미 구분 (요구 3 핵심)
- **일정**: `start_at`(필요시 `end_at`)의 **시작시각**이 1차. 라벨 "시작" 또는 시각만(타입 아이콘으로 일정임이 명확하므로). **KST 변환 일원화**로 셀=패널 일치. (A 해소)
- **업무**: 캘린더 앵커 = **마감/예정일**(`target_date` 우선, 없으면 `scheduled_at`/`log_date`). 표기 예: `예정 · 마감 6/18` 또는 시각이 있으면 `예정 · 6/18 10:00`. **작성일시(logged_at)는 일정 시간처럼 단독 노출 금지** — 필요하면 "작성 6/17" 보조 회색. (B 해소)
- **메모**: 작성일 기준. "메모 · 작성 6/18". 마감 시간 강조 안 함.
- **표시 변환 SSOT**: `lib/`에 시각 포맷 공용 함수(예: `formatCalTime(kstIso)`)를 두고 셀·패널·미니캘린더가 import. 뷰마다 `slice(11,16)` 복붙 금지(원시 UTC 노출 원인).

## 5. 중복 처리 (자동복제 일정 ↔ 원본 업무)
선택지(기획안, 구현 시 택1 — §6 결정 필요):
- **A안 (대표 표시·권장)**: 셀에서 `calendar_events.link_kind='daily'`(업무 자동생성)인 일정이 있으면, **같은 `link_id` 업무의 미리보기 칩은 셀에서 숨김**(일정 칩이 대표). 패널에서는 업무 섹션에 그대로 두되 일정 카드에 "**업무 연동**" 배지.
- **B안 (연동 배지만)**: 양쪽 다 두되, 자동생성 일정 카드에 "업무에서 생성" 배지 + 업무 항목에 "캘린더 등록됨" 배지 → 중복이 아니라 연결임을 명시(숨김 없음).
- **C안 (자동복제 중단)**: `autoRegisterSchedules` 자동등록을 끄고, 업무는 캘린더에 업무로만 표시(일정화는 사용자가 명시 등록). 가장 단순하나 기존 동작 변경 큼.
> 권장: **A안**(셀 중복 제거 + 패널 연동 표식). 사용자 체감 "중복 사라짐" + 데이터 보존.
