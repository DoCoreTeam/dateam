# 00 — 요구사항: USAI(통합 자가적응 흡수) 엔진

## 배경 / 문제 (실측 확정)
GPU 통합입력의 catalog 경로가 비정형 다중블록 xlsx를 "단일 헤더 평면표"로 강제 해석 → 오독.
실측(현 코드 실행)으로 확정된 9개 실패 모드:

| # | 실패 모드 | 근거 |
|---|---|---|
| F1 | 다중 블록(4개 가격표)을 1개로 오인 | detectHeaderRow가 21행 1개만 선택 |
| F2 | 옆 표(담당자명부 P열)가 vendor로 혼입 | T4 → "NHN클라우드"(P22) |
| F3 | 단위 오선택(8장서버 시간당을 단가로) | T4 = 6.481481…(J22) vs 정답 0.81(J37) |
| F4 | 통화 혼재(KRW D열 / USD J열) 미구분 | 둘 다 데이터로 유입 |
| F5 | 다중 시트 누락 | 3시트 중 '시간당'만 읽음 |
| F6 | 병합셀 59개 손실 | 파서 미사용 |
| F7 | 분류 오류(우리 목표가→경쟁사) | target:'competitor' 하드코딩 |
| F8 | 빈 견적양식(하단) 잡음 | 빈 블록 |
| F9 | 원시 float 표시(6.481481…) | 표시 정규화 부재 |

## 핵심 철학 (확정)
"규칙 vs AI"는 비율이 아니라 **단계별 책임 배치**다. 미지 형식 무한 → AI가 구조이해·추출을 주도.
코드는 **형식불변 항진명제 검증만**(도메인 밴드/enum 금지). 불확실은 **사람 확정 게이트**로.

## 기능 요구
- FR1: 모든 입력(xlsx/csv 우선, 추후 pdf/img/text)을 **좌표보존 표현**으로 정규화 (전 시트, 병합 포함)
- FR2: AI가 시트 내 **블록 발견 + 역할분류**(가격표/명부/스펙/잡음) + bbox + 헤더
- FR3: AI가 블록별 추출 — 각 값에 **출처좌표(provenance) 필수**, 통화/단위 토큰, 신뢰도
- FR4: **형식불변 정합**(출처존재·부호·산술정합) + 통화/단위 정규화(선언적 lookup SSOT)
- FR5: **자기일관성 검증** + 신뢰도 임계 → auto vs needs_human 라우팅
- FR6: AI **분류 게이트** — target = own_target/competitor/supplier (블록 단서 기반)
- FR7: 기존 **사람 확정 게이트(review_items)** 유지 — 자동반영 금지

## 비기능 요구
- NFR1: SSOT 재사용 — dedup/validate/intake-routing/prompt-governance/review_items 재사용
- NFR2: 신규 의존성 0 (현 xlsx/SheetJS + Gemini만)
- NFR3: 운영 데이터 오염 0 — is_test/throwaway로만 검증
- NFR4: 회귀 0 — 정상 카탈로그(gcube)는 그대로 통과
