🔍 모델·메모리·공급사·경쟁사 검색…

💡 세그먼트를 눌러 7각도를 직접 전환해보라(컬럼 교체). 행의 ⤓원천 버튼 → 백데이터(모든 견적·시장가 이력·감사) 펼침 — 미구현 구간은 빨간 배지로 표기.

각도 = 컬럼 프리셋

행 정체성(GPU/구성)은 고정, 컬럼만 교체 → 각도를 바꿔도 "같은 줄"을 본다. 비교가 끊기지 않는다.

⤓원천 = 드릴다운

집계 뒤의 원천 행(견적 전수·시장가 시계열·감사)을 같은 줄에서 펼침. 화면이 "다 보여줄 수 있는지"를 정직하게 표기.

행마다 CRUD

어느 각도든 행에서 수정·삭제·직판·매핑 액션. 무엇을 눌러도 편집 경로가 있다.

05

백데이터 — 다 볼 수 있나? (냉정)

집계 뒤 원천 레코드의 실제 조회 가능성.

⚠️ 냉정한 답: 지금은 "다" 못 본다. 화면은 집계(최저가·최신가)만 보여주고, 원천 전수·시계열·이력은 조회 라우트가 대부분 없다. 통합테이블 드릴다운을 제대로 하려면 신규 read API 4개가 필요하다.
원천 데이터지금 보이나?현재 라우트갭 / 필요 신규 API
product의 모든 공급견적확정·대기·만료·반려부분quotes?product_id (confirmed만)pending/expired/rejected 안 보임 → quotes?product_id&status=* 신규

review_item의 추출 이력iteration·evidence불가없음(append만)review_iterations GET 없음 → review/[id]/iterations 신규
감사 로그변경/삭제 추적부분audit (전체·300상한)product/actor/기간 필터 없음 → audit?filter 신규
competitor ↔ 시장가 역추적단방향market (경쟁사→가격)가격→경쟁사 역추적 없음(우선순위 낮음)

● 가능 · 부분 일부만 · 불가 라우트 없음. 핵심: 통합테이블 ⤓원천 드릴다운은 위 4개 API를 먼저 만들어야 "다 보인다"가 진짜가 된다. 목업의 ⤓원천 드로어가 이 갭을 그대로 표시한다.

왜 지금 안 보이나 (이유)

  • 화면이 의사결정용 집계(최저가/최신가) 위주로 설계됨 — 원천 전수는 검토대기·감사 탭에서 부분적으로만.
  • append-only 테이블(review_iterations·revisions)은 쓰기만 있고 읽기 API가 안 붙음.
  • 시계열(market_prices)은 최신 1행만 맵에 담아 비교에 사용 → 과거값은 DB엔 있으나 노출 경로 없음.

구현 영향 (명확)

  • 드릴다운은 프론트만으로 불가 — read API 4개 신규(로드맵 P5).
  • 모두 읽기 전용·RLS read라 위험 낮음(쓰기 변경 없음).
  • DB 변경 없음(기존 테이블 조회) → 마이그레이션 불필요, 라우트만 추가.
  • 통합테이블 ⤓원천 UI는 API 준비 전까지 "신규 API 필요" 배지로 단계 노출.
