# 일일업무 — "원본 보존 + AI 정제 펼침" 기획 (구현 전, 보고용)

> **상태: 기획만. 코드 구현 없음.** 사용자 승인 후 단계별 착수.

## 한 줄 요약
사용자가 쓴 **일일업무 원본 텍스트를 그대로 보존·표시**하고, 그 **아래 펼침(드로어)으로 AI 분석 결과**(업무 분해 + ①중복 ②일정·캘린더 ③누락 메모 정제)를 **별도로** 보여준다. AI는 원본을 덮어쓰지 않고 **제안만**, 등록·병합은 사용자가 확정한다.

## 가장 중요한 발견 (저비용 기회)
**원본은 이미 저장되고 있다.** `daily_logs.original_input` + `daily_log_origin_groups.original_input`에 원본 전문이 보존 중이고, `origin_group_id`로 분해 항목들이 묶여 있다. **표시 UI만 없다.** → "원본 보존형 AI"라는 신뢰 가치를 **DB 변경 거의 없이** 즉시 확보 가능.

## 현행 vs 제안 (개념)
```
[현행]  자유입력 → AI가 즉시 개별 태스크로 "분해 저장" → 분해 항목만 타임라인 표시
                                                   (원본은 DB에만, 화면엔 없음)

[제안]  자유입력 → 원본 블록(그대로 보존·표시)
                     └▾ 펼침 드로어(기본 접힘):
                         · AI 업무 분해 결과(후보 체크 → 사용자 확정)
                         · ① 중복 의심 표기      (감지·제안, 자동삭제 X)
                         · ② 일정 후보 → 캘린더   (후보 체크 → 사용자 확정, 자동등록 X)
                         · ③ 놓친 메모 환기        (기존 메모 시스템 연계)
```

## 우선순위 (DC-BIZ 판단)
| 단계 | 내용 | 임팩트 | 난이도 | 근거 |
|---|---|---|---|---|
| **P0** | 원본 표시 + AI 드로어 분리 + 누락 메모 환기 | 高 | 低 | 데이터·메모 시스템 이미 완비 → UI 연결만 |
| **P1** | 중복 감지·표기(제안까지) | 中 | 中 | 결정론 로직 신규(유사도). 자동병합은 제외 |
| **P2** | 일정성 항목 → 캘린더 반영(확인형) | 高 | 高 | 자동생성 미구현 + **오등록=신뢰 훼손** 리스크 → 별도 스프린트 |

## 핵심 설계 원칙
1. **원본 = SSOT.** AI 분해는 파생(후보). 원본을 절대 덮어쓰지 않음.
2. **AI는 제안, 사용자가 확정** (newAX AI UI 표준 5-3: 추출=후보 체크리스트, 자동등록 금지).
3. **기존 자산 최대 재사용** — origin_groups·메모·관계 테이블·analyze-work·ai_prompts 거버넌스 그대로 활용.
4. **드로어 기본 접힘** — 원본/AI 이중 표시의 인지부하 관리.
