# 01 — 목표 구조: 표기 변형에 깨지지 않는 흡수 파이프라인

> 기획 문서. **구현 없음.**

## 설계 원칙 (한 줄)
> **AI는 "무엇인지" 판단하고, 코드는 "얼마인지" 계산하고, 게이트는 "빠진 게 없는지" 감시한다.**

각자 잘하는 것만 시킨다. 지금은 코드가 세 가지를 다 하려다 첫 번째에서 깨진다.

---

## 1. 역할 재배치

### AI가 하는 일 — 인식 (경우의 수 무한)
원문에서 **구조화된 관측 레코드**를 뽑는다. 산술은 절대 시키지 않는다.

```
{
  competitor_name: "SoftBank",          // 회사명 인식
  model: "GB200",                        // 모델 (폼팩터·메모리·수량 제외한 순수 모델)
  form_factor: "SXM6" | null,            // 폼팩터 축 분리
  memory_gb: 288 | null,                 // 메모리 축 분리
  gpu_count: 4,                          // 장수 축 분리
  amount: 4569000,                       // 원본 금액 (환산 전 — 나눗셈 금지)
  currency: "JPY",                       // ISO 코드로 인식 (￥·¥·円·전각 전부 AI가 판단)
  unit: "month",                         // enum
  component_kind: "flat",                // 요금 성격
  provenance: "月額 | ￥4,569,000"        // 원문 근거 (필수 — 검증 대조용)
}
```

핵심: **AI에게 정규화된 enum을 요구**한다. `￥`를 보고 `"JPY"`라고 답하는 건 AI가 100% 하는 일이다.
지금은 AI가 원문 문자열만 주고 코드가 `￥`를 몰라서 깨졌다.

### 코드가 하는 일 — 산술 + 검증 (결정론, 100% 재현)
- 시간 환산: `hours.ts` SSOT (÷720 등)
- 통화 환산: `fx_rates_multi` 실환율
- 정액요금 주기: `MONTHS_PER_PERIOD` (달력 계수)
- **AI 출력 검증**: enum 화이트리스트 밖이면 거부, 금액 범위 밖이면 거부

AI가 준 `currency: "JPY"`는 그대로 쓰되, `currency: "돈"` 같은 헛소리는 코드가 막는다.
**AI는 인식만, 채택 여부는 코드가 결정.**

### 열거표의 새 역할 — 1차 채택자에서 **2차 안전망**으로 강등
`CURRENCY_TOKENS`·`COMPETITOR_RULES`·`GPU_MODEL_SIGNAL`을 없애지 않는다. 순서를 뒤집는다.

- **지금**: 코드 표가 1차 판정 → 표에 없으면 실패
- **목표**: AI 인식이 1차 → 코드 표는 **교차검증·폴백**

표에 없는 표기가 와도 AI가 처리하고, 표에 있는 건 교차검증에 쓰인다.

---

## 2. 핵심 — 이중 독립 추출 + 교차검증

**이것이 "조용히 틀리지 않음"을 만드는 장치다.**

```
원문 ─┬─→ [AI 인식]      ─→ 관측 A
      └─→ [결정론 파서]   ─→ 관측 B      (기존 parseHourlyProse 등 재활용)
                    ↓
              [교차 대조]
                    ↓
       ┌────────────┼────────────┐
    A≡B          A만 있음        A≠B
   자동확정      단독채택+       ★보류★
   (신뢰 高)     낮은 신뢰도     (사람 확인)
```

- **일치** → 자동 확정. 두 독립 방법이 같은 답을 냈으므로 신뢰도 최상
- **불일치** → **보류**. 조용히 한쪽을 고르지 않는다. 이게 오늘 사고의 핵심 방지책
- **한쪽만 있음** → 채택하되 신뢰도 하향 + 근거 표시

기존 결정론 파서를 버리지 않고 **검증자로 재배치**하므로, 지금까지 쌓은 정규식 자산이 살아난다.

---

## 3. 완전성 게이트 결선 (이미 있는 코드를 붙이기만)

`completeness-reconcile.ts`는 **이미 만들어져 있고 테스트도 통과한다. 결선만 안 됐다.**

```
원본 스냅샷 ─→ 통화 동반 금액 토큰 전수 스캔 ─┐
                                              ├─→ 대조 ─→ 미커버 목록
추출 결과의 모든 금액 ────────────────────────┘
```

미커버가 1건이라도 있으면:
- 화면에 **"원본에 있으나 추출 못 함: ￥4,569,000"** 명시
- 자동확정 차단 → 검수 큐

**오늘 GB200 사고는 이것만 붙어 있었으면 즉시 잡혔다.**

## 4. 축 분리 (폼팩터를 컬럼으로)

`gpu_products`에 `form_factor` 컬럼 신설. 4축을 4컬럼으로.

| 축 | 현재 | 목표 |
|---|---|---|
| 모델 | `"A100 SXM"` (뭉침) | `model_name: "A100"` |
| 폼팩터 | 〃 문자열에 섞임 | `form_factor: "SXM"` |
| 메모리 | `memory` | `memory` (유지) |
| 장수 | `gpu_count` | `gpu_count` (유지) |

**매칭 규칙**: 모델은 정확 일치 요구, 폼팩터는 **세대 표기 흡수**(`SXM4`·`SXM5`·`SXM6` → `SXM`).
A100의 SXM은 SXM4뿐이고 H100은 SXM5뿐이므로, 모델이 확정된 뒤의 폼팩터 세대 병합은 안전하다.
(모델 확정 전에 병합하면 위험 — 순서가 중요)

이러면 verda `A100 SXM4` ↔ 카탈로그 `A100 SXM`이 매칭된다. 지금은 보류된다.

## 5. 조용한 실패 제거

모든 드롭·null에 **사유를 값으로 남긴다**.

```
hold_reason: "currency_unrecognized" | "fx_missing" | "model_unmatched"
            | "ai_deterministic_mismatch" | "amount_out_of_range"
hold_detail: "통화 인식 실패: \"￥4,569,000\""
```

화면에는 "가격미상"이 아니라 **"통화 인식 실패: ￥4,569,000"** 이 뜬다.
그러면 다음 오류는 몇 시간이 아니라 **몇 분**짜리가 된다.

현재 조용한 드롭 지점(🟦 DC-ANA 확인): `market-refresh-item.ts:81-83`(경쟁사명 빈값→무기록 드롭),
`transcription-to-items.ts:124`(라벨 없는 행 스킵), `deterministic-table.ts:44,49`(빈 배열 반환).

---

## 6. 두 경로 통합
`review/stream`과 `market/refresh`가 **같은 SSOT 파이프라인**을 호출하게 한다.
입력만 다르고(사용자 붙여넣기 vs 자동 크롤) 처리는 동일해야 한다.
현재의 정책 불일치(캐노니컬·도메인 폴백·피벗 유무)를 해소한다.

---

## 7. 사이트 회귀 코퍼스
검증한 사이트의 **실제 원문**을 코퍼스에 축적한다(소프트뱅크·verda는 오늘 이미 박제).
새 사이트를 고치다 기존 사이트를 깨뜨리는 일을 막는다.
새 오류를 막지는 못하지만 **되돌아오는 오류는 막는다.**
