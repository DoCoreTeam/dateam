# 03 — 단계별 실행안 (효과/비용 순)

> 기획 문서. **구현 없음.** 착수 여부·순서는 사용자 결정.

각 단계는 **독립적으로 가치가 있고 단독 롤백 가능**하도록 나눴다.

---

## P1. 완전성 게이트 결선 — 효과 최대 / 비용 최소 ★최우선
**하는 일**: `completeness-reconcile.ts`(이미 완성·테스트 통과)를 추출 경로에 연결.
원본 금액 전수 스캔 ↔ 추출 결과 대조 → 미커버 금액을 화면에 명시 + 자동확정 차단.

- **왜 1순위**: 새 코드를 거의 안 쓴다(결선만). 그런데 **오늘 사고 6건 중 4건을 즉시 가시화**한다.
  GB200(￥4,569,000 누락)·소프트뱅크 시간제(3성분 누락) 전부 이 게이트에 걸린다.
- **리스크**: 낮음. 기존 저장 로직 무변경, 경고 표시만 추가.
- **부수 효과**: 기존 사이트에서도 그동안 몰랐던 누락이 드러날 수 있음(좋은 일이나 검수 부하 증가)

## P2. 실패 사유 노출 (hold_reason)
**하는 일**: 드롭·null 지점마다 사유 코드 + 원문 조각을 값으로 남기고 화면에 표시.

- **왜 2순위**: P1이 "빠졌다"를 알려주면, P2가 "왜 빠졌는지"를 알려준다. 둘이 합쳐져야 진단이 완성된다.
- **리스크**: 낮음(필드 추가·표시).
- **대상**: `market-refresh-item.ts:81-83`, `transcription-to-items.ts:124`, `deterministic-table.ts:44,49`,
  통화·환율·모델매칭 실패 전 지점.

## P3. AI 인식 계층 도입 + 교차검증
**하는 일**: AI가 구조화 관측(모델·폼팩터·메모리·장수·금액·통화ISO·단위·성분성격·provenance)을 산출.
기존 결정론 파서와 **독립 실행 후 대조** → 일치=자동확정 / 불일치=보류.

- **왜 3순위**: 가장 효과가 크지만 가장 큰 변경. P1·P2로 감시망을 먼저 깔고 들어가야 안전하다.
  (감시 없이 추출 방식을 바꾸면 무엇이 좋아지고 나빠졌는지 측정 불가)
- **리스크**: 중간. AI 비용·지연 증가, 비결정성. → 골든 코퍼스로 회귀 감시.
- **선결**: P1·P2 완료.

## P4. 두 경로 통합 (review/stream ↔ market/refresh)
**하는 일**: 두 진입점이 동일 SSOT 파이프라인을 호출. 정책 불일치(캐노니컬·도메인폴백·피벗) 해소.

- **리스크**: 중간(자동 경로 변경 — fire-and-forget이라 무음 실패 주의).
- **선결**: P2(사유 노출)가 있어야 자동 경로 실패를 감지할 수 있음.

## P5. 축 분리 (form_factor 컬럼)
**하는 일**: `gpu_products.form_factor` 신설. 모델 확정 후 폼팩터 세대 병합(SXM4/5/6 → SXM).

- **효과**: verda `A100 SXM4` ↔ 카탈로그 `A100 SXM` 매칭. 표기 차이로 인한 보류 해소.
- **리스크**: 중간~높음. 스키마 변경 + 기존 데이터 백필(자유문자열 역추출이라 불완전).
- **완화**: additive only. 신규만 분리, 기존 하위호환. 백필은 별도 판단.

## P6. 죽은 코드 정리 + 사이트 코퍼스 확장
- `parsePivotFlat` ↔ `reconstructPivot` 중복 2벌 → 통합 또는 제거
- `self-eval-loop`·`conformance`·`golden-set` 결선 여부 결정(붙이거나, 안 붙일 거면 삭제)
- 검증한 사이트 원문을 코퍼스에 누적

---

## 권장 순서와 근거

```
P1 (완전성 게이트) → P2 (실패 사유) → [여기서 재평가] → P3 (AI 인식) → P4 → P5 → P6
```

**P1+P2만 해도 오늘 겪은 고생의 대부분이 사라진다.** 오류가 안 나게 되진 않지만,
오류가 **스스로 드러나므로** 내가 URL 넣고 DB 뒤지는 일이 없어진다.

P3(AI 인식)이 근본 해결이지만, **P1·P2 없이 P3부터 하면 안 된다.**
감시망 없이 엔진을 바꾸면 좋아졌는지 나빠졌는지 측정할 수 없기 때문이다.
오늘 v0.7.352에서 "결선 완료"라고 커밋했는데 실제로는 한 번도 동작하지 않았던 일이
정확히 그 실수였다.

## 미결 — 사용자 결정 필요
1. **AI 호출 비용/지연 증가를 감수할 것인가** (P3의 전제)
2. **폼팩터 축 분리 시 기존 데이터 백필을 할 것인가** (P5)
3. **`self-eval-loop`·`conformance`를 살릴 것인가 삭제할 것인가** (P6)
4. 화면 경고 오탐 처리 — 현재 "12행 중 4행만 추출"이 사양 항목까지 누락으로 세는 문제
