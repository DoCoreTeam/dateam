# 02 — 보장 수준과 한계 (정직하게)

> 사용자 요구: "절대 문제 되지 않는 방향". 그 요구에 대해 **약속할 수 있는 것과 없는 것**을 구분해 적는다.
> 여기서 과장하면 오늘 같은 일이 반복된다.

## A. 약속할 수 없는 것

### "새 사이트에서 오류가 절대 안 난다" — 불가능
이유:
1. **AI도 틀린다.** 인식률이 코드 정규식보다 월등히 높지만 100%는 아니다.
2. **원문 자체가 모호한 경우가 있다.** "$2.10"이 시간당인지 월정액인지 페이지에 안 적힌 경우,
   어떤 방법으로도 확정할 수 없다. 사람도 못 한다.
3. **페이지가 가격을 아예 안 보여주는 경우**(로그인 필요·이미지·JS 렌더 실패)가 있다.

이걸 "안 난다"고 적으면 거짓말이다.

### "폼팩터 자동 병합이 항상 안전하다" — 조건부
`SXM4→SXM` 병합은 **모델이 먼저 확정된 뒤**에만 안전하다. 순서가 뒤집히면 오병합이 난다.
설계에 순서를 못박아야 하며, 이 부분은 골든 테스트로 고정해야 한다.

## B. 약속할 수 있는 것 — 이게 핵심

### B-1. "조용히 틀리지 않는다"
오늘 사고의 본질은 **틀린 것 자체가 아니라 티가 안 난 것**이다.
GB200은 금액·장수 다 맞고 통화만 못 읽었는데, 화면엔 "가격미상"만 떴다. 내가 DB를 뒤져서 찾았다.

목표 구조에서는:
- **완전성 게이트**가 원본 금액을 전수 스캔해 "추출에 없는 금액"을 즉시 지목
- **교차검증**이 AI와 결정론 파서의 불일치를 보류로 올림
- **hold_reason**이 실패 사유를 값으로 남김

→ 틀리면 **화면이 먼저 알려준다.** 이것은 구조적으로 보장 가능하다.

### B-2. "틀린 값이 조용히 저장되지 않는다"
확신 없는 값은 저장하지 않고 보류한다. 이미 있는 정책(가짜가격 유입 금지)을 유지·강화한다.
**"모르면 비워두고 명시" > "그럴듯한 값 채우기"**.

### B-3. "같은 오류가 두 번 나지 않는다"
사이트 회귀 코퍼스로 고정. 새 오류는 못 막지만 재발은 막는다.

### B-4. "고치는 시간이 짧아진다"
실패 사유가 화면에 뜨므로, 오늘처럼 URL 넣고 DB 뒤지고 파서 단독 실행하며 추적할 필요가 없다.
**시간 단위 → 분 단위.**

## C. 정직한 기대치

| 지표 | 현재 | 목표 구조 |
|---|---|---|
| 새 사이트 첫 시도 성공률 | 낮음(오늘 2개 사이트 모두 실패) | 높음(AI 인식) — 단 100% 아님 |
| 실패 시 인지 | **불가**(조용히 미상) | **즉시**(화면에 사유·누락 금액) |
| 틀린 값 저장 | 가능 | 차단(보류) |
| 진단 시간 | 시간 단위 | 분 단위 |
| 재발 | 가능 | 코퍼스로 차단 |

**핵심 전환: "오류가 안 나는 시스템"이 아니라 "오류가 스스로 드러나는 시스템".**
전자는 불가능하고, 후자는 가능하며, 실무적으로 후자가 더 안전하다.

## D. 이 기획이 틀릴 수 있는 지점 (반대 논거)

정직하게 적는다.

1. **AI 호출 비용·지연 증가.** 인식을 AI에 맡기면 토큰·시간이 는다.
   완화: 결정론 파서가 먼저 성공하면 AI 호출 생략(비용 최적화)은 가능하나, 그러면 교차검증이 사라진다.
   **비용 vs 검증 강도는 사용자 결정 사항.**
2. **AI 비결정성.** 같은 입력에 다른 출력이 날 수 있다. temperature 0 + 스키마 강제로 줄이지만 0은 아니다.
   완화: 교차검증·완전성 게이트가 이를 흡수. 골든 코퍼스로 드리프트 감시.
3. **축 분리는 스키마 변경 + 기존 데이터 백필**을 수반한다. 백필 난이도가 높다
   (기존 `model_name`에서 폼팩터를 역추출해야 하는데 자유문자열이라 완벽하지 않음).
   완화: 신규만 분리하고 기존은 하위호환 유지(additive) → 백필은 별도 판단.
4. **"AI를 더 믿자"는 방향이 v0.7.351 원칙과 충돌하는 것처럼 보인다.**
   실제로는 충돌이 아니다 — 산술은 계속 코드가 한다. 바뀌는 건 **인식**의 담당자뿐이다.
   다만 이 구분이 흐려지면 다시 AI가 계산하는 사고로 회귀할 수 있으므로, 프롬프트·스키마에
   "산술 금지"를 명시 유지해야 한다.
