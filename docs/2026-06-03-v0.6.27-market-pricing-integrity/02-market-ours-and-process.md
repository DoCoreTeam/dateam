## 1. 시장비교에 "우리 공급사" 가격 동시 표시

### 현황
- 시장비교는 `competitors`(RunPod 등 12곳) 가격만 카드/포지션바에 표시.
- 우리 측은 `our_price_usd`(공급최저가×마진=판매가) 마커 1개만 표시 — **공급사 개별 가격은 안 보임**.
- 우리 공급사 = `suppliers` (gcube 104건·High Reso·Konsttech·Tensordock 확정 견적 보유).

### 요구
경쟁사 목록에 **우리 공급사 가격도 같은 리스트에** 노출하되 **"공급사" 배지**로 구분 → 시장 내 우리 위치·격차를 한눈에.

### 설계
1. **market/route.ts 응답 확장**: 상품별 `our_suppliers: [{ supplier_id, name, color, unit_price_usd(공급원가), sell_price_usd(원가×마진), per_gpu_usd, is_ours:true }]` 추가. (supply_quotes 확정분, §B 1장당 전파 반영)
2. **AnalyzePanel(경쟁사별 카드)**: 우리 공급사를 경쟁사 카드와 **동일 컴포넌트**로 렌더 + `공급사` 배지(우리 색/보더 구분). 경쟁사와 한 그리드에 섞여 정렬 → 위치 비교.
3. **PositionBar / 인사이트 / 진영 카운트**: 우리 항목 포함해 "우리 N위/M곳", "최저 경쟁사 대비 ±x%" 계산.
4. **시각 구분**: 경쟁사=회색 계열, 우리 공급사=accent 보더 + `공급사` 배지(+gcube는 `자사` 보조 배지 가능).

### 확정 (2026-06-03 승인)
- 우리 공급사 배지 항목에 **공급원가 + 우리 판매가 둘 다** 표시 (예: `원가 $1.95 · 판매 $2.44`).
- **포지션바·진영 비교는 판매가 기준**(경쟁사 판매가와 단위 일치) → 우리 위치 일관 표시.

## 2. 테스트 데이터 전략 (UI 등록 검증 + 나중 정리)

### 원칙 (사용자 지시)
- **DB 직삽 금지에 가깝게 — UI에서 실제 등록 플로우로 검증**(견적 등록·재고 입력·공급사 지정). DB만으론 릴레이션·트리거·뷰 반영을 정확히 못 봄.
- 등록한 **테스트 데이터는 표식**해 두고 나중에 일괄 정리 가능하게.

### 방법
1. **태깅 규칙**: 테스트로 UI 등록하는 레코드는 식별 가능한 표식 부여
   - 견적/가격: `notes`(또는 `source_url`)에 `[[TESTDATA 2026-06-03]]` 프리픽스.
   - 공급사/경쟁사 신규 생성 시: 이름에 표식 불가하면 → 매니페스트로만 관리.
2. **매니페스트 파일**: `docs/2026-06-03-v0.6.27-market-pricing-integrity/test-data-manifest.md`
   - 표: `테이블 | id | 설명 | 등록경로(UI) | 등록시각`. UI로 넣을 때마다 즉시 기록.
3. **정리 스크립트**: `docs/.../cleanup-testdata.sql` — 매니페스트 id 기준 DELETE(또는 status='rejected'). **검토 후 사용자 승인 시 실행**(자동 삭제 금지 — 데이터 보호).
4. **릴레이션 검증 시나리오**(UI에서 직접):
   - 신규 견적 등록 → 가격표 그룹/1장당 전파 → 시장비교 우리공급사 → 재고 공급사 노출까지 **연쇄 반영** 확인.
   - 재고 수량 입력 → 재고 헬스바/요약 갱신.
   - 공급사 미지정 견적 → 인라인 공급사 지정 → 즉시 반영.
   - 재고 제거(가용량 0/만료) → 목록에서 상태 전환 정상.

## 3. 공용 컴포넌트 — CRUD 일관성 (수정 1곳)

### 요구
리스트/표시는 **동일 컴포넌트 1벌**. 새 견적 입력·없는 재고 제거 등 **모든 CRUD가 같은 컴포넌트에서 일관 동작** → 나중에 한 곳만 고치면 전부 반영.

### 설계 (01 문서의 공용화 + CRUD 확장)
- `components/gpu/CategoryGroup.tsx` — Tier→모델 접기 그룹(공용).
- `components/gpu/GpuRow.tsx` — **상품/구성 1행 공용 표시 컴포넌트**. props로 모드(`price` | `market` | `inventory` | `catalog`)와 우측 슬롯(가격/수량/액션) 주입. 모델·뱃지·스펙·공급사 표기는 **공용**.
- `components/gpu/SupplierBadge.tsx` — 공급사/경쟁사/자사 배지 공용(시장비교·가격표·재고 모두 사용).
- CRUD 훅: `useQuoteMutations`(등록/공급사지정/삭제), `useAvailabilityMutations`(수량 입력/제거) — 변경 후 `mutate` 키 일괄 무효화로 **모든 탭 동시 갱신**.
- 효과: 새 견적/재고 추가·제거 시 4개 탭이 같은 컴포넌트·같은 갱신 경로 → 표시 버그 1곳 수정으로 전파.

### 트레이드오프
- 공용 행 컴포넌트는 모드 분기가 생겨 다소 복잡 → props 슬롯으로 표시/액션만 주입해 300줄 이내 유지.
