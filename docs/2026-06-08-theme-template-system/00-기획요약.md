# 디자인 테마(템플릿) 전환 시스템 — 기획 (구현 0)
> 2026-06-08 · admin에서 디자인 테마 선택 → 전 서비스 적용 · **절대 구현 금지(기획만)**

## 한 줄 요약
이번에 만든 토큰 시스템(`--brand/--ink/--accent/--border-w/--radius/--shadow-*`)을 **테마별로 값만 스왑**(`[data-theme="X"]`)하고, 선택값을 `system_settings`에 저장→SSR로 `<html data-theme>`에 주입하면, admin에서 디자인 템플릿을 골라 전 사용자에게 적용할 수 있다. **브랜딩(브랜드명/로고) 관리와 동일한 패턴**을 테마로 확장.

## 왜 지금 가능한가 (이번 작업의 진짜 가치)
- 기존엔 색이 인라인 하드코딩이라 테마 전환 불가였음 → 이번에 **토큰화 + 브릿지**를 해둬서, 토큰을 참조하는 영역은 **값만 바꾸면 즉시 테마 전환**됨.
- 보더두께(`--border-w`)·radius(`--radius`)·그림자(`--shadow-md`)까지 토큰이라 → "classic(인디고 soft) ↔ NB(하드)" 가 토큰값 차이만으로 재현 가능.
