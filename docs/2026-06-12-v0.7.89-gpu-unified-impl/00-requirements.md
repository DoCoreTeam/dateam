# 00 요구사항 — GPU 가격관리 리팩토링 (구현)

> 상위 SSOT: `docs/2026-06-12-v0.7.86-gpu-refactor-plan/gpu-refactor-plan.html` (14챕터 기획서)
> 본 폴더는 그 기획을 **실제 구현**하기 위한 작업 문서.

## 목적
보기 10탭 → **통합 표 1개(마스터·디테일 + 보기 전환)**, 입력 6경로 → **통합 입력 1개(멀티모달)**.
영업/견적 담당이 탭 이동 0회로 판단·등록. 계산 SSOT 보존, 표현·입력만 재배치.

## 사용자/권한
- member(영업): 상품·가격·시장가·검토 대기 읽기/쓰기, 마스터(공급사·경쟁사·마진) **읽기만**
- admin: 전체 + 마스터·거버넌스 쓰기

## 기능 요구 (P1~P5)
- P1 통합 표: 좌 목록(보기 프리셋 전환: 판매가·가격 결정·시장 비교·재고·고객가) + 저장된 보기·컬럼 토글
- P2 상세 패널(우측 고정): 공급원가/시장 비교/변동 이력/스펙 탭 + 인라인 CRUD
- P3 통합 입력: 멀티모달(텍스트·메일·PDF·이미지·URL·CSV) + 신뢰도 자동 게이트 + 변경분 diff
- P4 RBAC: 마스터 쓰기 admin, member 읽기 / 메뉴 영업·관리 분리
- P5 읽기 API 4개 + 약점 보강(직판 원가연결·일괄수정·만료 D-7·파트너 할인)

## 비기능
- 성능: 통합 표 ≤1.5s, 보기 전환 재요청 없이 컬럼 교체, 목록 서버 페이지네이션
- 반응형: 모바일 목록→상세 풀스크린, 가로 스크롤 금지
- 접근성: 상태 색+텍스트/아이콘 병기, 키보드, 대비 4.5:1

## 제약(가드레일 7)
1 계산 로직 import만 / 2 GPU_TERMS SSOT / 3 마스터·디테일+자동게이트(인라인 드로어 금지)
4 RBAC requireAdminApi+RLS / 5 CSV 수식 인젝션 무력화 / 6 읽기 API 4(읽기전용·DB변경 없음) / 7 반응형·a11y
+ feature flag 병존·무중단 / main 브랜치·커밋까지만
