# 주간보고 조직현황 취합 미동작 진단 (구현 0 — 분석 전용)
> 2026-06-08 · "카테고리 취합 + 내용 다듬기 왜 안 되나"

## 한 줄 답
**카테고리 취합과 내용 다듬기 둘 다 Gemini AI에 의존** → 두 기능이 동시에 안 되면 **gemini_api_key 미설정(1순위)**이 거의 확실. 추가로 조직도 매핑(head_user_id)·closure self-row 누락이 취합을 막을 수 있음.

## 데이터 흐름
- 조직현황 탭 노출: `hasOrgScope`(org-scope.ts:119) — 조직도 등록자면 보임(조회)
- "AI 취합" 버튼: `editable && !busy`(OrgWeeklyView.tsx:258) — **부서장(head_user_id)만** 보임
- 취합: OrgWeeklyView onAggregate → `POST /api/reports/aggregate-stream` → resolveOrgScope → deptMemberUserIds → weekly_reports(admin client, RLS우회) → **Gemini 스트리밍** → dept_weekly_reports upsert
- 다듬기: "내 보고" 탭 WeeklyReportForm → `POST /api/weekly-report/refine` → Gemini

## 근본원인 Top 3
### 1순위 — gemini_api_key 미설정 (가장 유력)
- 취합: aggregate-stream/route.ts:83-85 `if(!apiKey) return 400 'Gemini 키 없음'`
- 다듬기: refine/route.ts:57-60 `if(!apiKey) return 500 'AI 키 미설정'`
- `org_content.META.gemini_api_key`가 비어있으면 **둘 다 실패** → 증상 정확히 일치
- 확인: admin → 시스템 설정 → AI 모델 연동에 Gemini 키 저장돼 있는지

### 2순위 — org_nodes.head_user_id 미매핑
- editableDeptIds = head_user_id===본인 부서(org-scope.ts:53). 미지정 시 **취합 버튼 자체 미노출**(조회만)
- 확인: 조직도 관리에서 부서장 지정 여부

### 3순위 — org_node_closure self-row 누락
- deptMemberUserIds(org-scope.ts:109)가 closure 서브트리로 부서원 산출. self-row(ancestor=descendant,depth0) 없으면 `[]` → route.ts:61 400 '부서원 없음'
- 046 트리거는 신규 insert만 → 기존 노드 백필 누락 가능. 확인: org_node_closure self-row 존재 여부

## 부차 — 카테고리 누락(스트리밍 파싱)
- 카테고리 통합은 100% Gemini 위임. `extractObjects`(route.ts:34-45)가 스트림 종료 시 마지막 미완성 버퍼를 flush 안 함 → 마지막 카테고리 누락 가능. 취합은 "완료"로 뜨나 일부 카테고리 빠짐.

## 실패 가시성
- 키 없음/부서원 없음/주차보고 없음 → OrgWeeklyView 헤더 하단 파란 띠 메시지(setMsg) 표시(silent 아님, 단 눈에 잘 안 띔)
- 다듬기 실패 → 폼 하단 refineError(눈에 잘 안 띄는 위치)

## 권장(구현 방향만)
1. **admin에 Gemini 키 설정 확인**(1순위 — 둘 다 안 되면 99% 이것)
2. 조직도 부서장(head_user_id) 매핑 확인
3. org_node_closure self-row 백필 확인(DB)
4. 스트리밍 파싱 종료 flush 보강(카테고리 누락 방지)
5. 에러 메시지를 더 눈에 띄게(현재 위치 약함)
