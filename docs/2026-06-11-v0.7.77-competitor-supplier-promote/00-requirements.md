# 00 요구사항 — 경쟁사=공급사 이중역할 모델A 보강 (v0.7.77)

## 배경
Spheron·Elice·NHN Cloud 등은 시장비교 대상(경쟁사)이면서 동시에 우리가 시스템 연계로 원가 공급을 받는 "우리 공급사"인 이중역할 회사다. v0.7.76은 `competitors.supplier_id` FK를 만들었으나, 해당 회사들이 suppliers에 없어(교집합 0, 전부 NULL) 연결 자체가 불가했고, 공급사 메뉴는 suppliers만 노출해 겸업 회사가 보이지 않았다.

## 사용자 요구
1. 시장비교에서 경쟁사를 "우리 공급사"로 **1클릭 지정** → 사전등록 불필요(자동생성).
2. 공급사 메뉴에 **순수공급사 + 경쟁사겸공급사 통합 노출**, 겸업 구분 표시.
3. 그 회사 경쟁가(원가)에 마진 붙은 우리 판매가가 자연히 더 높게 형성되어 시장비교에도 보임.
4. 소싱 기밀: 공개 API에 공급관계 비노출.

## 채택 모델
모델 A 보강(현행 FK 유지 + 자동생성 + 통합 노출). 모델 B(회사 통합 테이블)는 22개 규모에 과투자라 기각(DC-BIZ 26 vs 20).

## 비기능
- 멱등(중복 supplier 생성 금지), 디자인 토큰/공용클래스, 반응형, RLS·admin 게이트, git push 금지.
