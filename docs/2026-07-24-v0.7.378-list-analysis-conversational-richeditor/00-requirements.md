# 00 — 요구사항 (목록 심층분석 대화형 재설계 + 리치에디터 구조충실도)

버전: v0.7.378~ (다단계) · 대상: `/ai-chat/analyze` (admin 전용)

## 배경
6-에이전트 조사(2026-07-24)로 4개 버그(①②③⑤)는 v0.7.375~376에서 해소. 남은 2대 재설계를 사용자가 승인.

## 요구 (사용자 확정)

### R1. ⑥⑦ 리치 에디터 + 구조 충실도 (입력→AI→출력 end-to-end)
- **R1-1** '자료 붙여넣기'를 plain textarea → **리치 에디터**(Tiptap, 표 지원)로 교체. 표/헤딩/리스트/서식 붙여넣기가 구조 그대로 들어와야 한다.
- **R1-2** 저장은 **둘 다**: (a) AI·그룹핑·검색용 **마크다운 정규화본**(파이프 표 포함), (b) **원본 HTML** 무손실 보존. (사용자: "둘다 되어야 하는것 같은데")
- **R1-3** AI 입력은 **마크다운 파이프 표**로 전달(HTML 직접 주입 금지 — CLAUDE.md html-to-plain 정책·토큰비용).
- **R1-4** 그룹핑(cut-groups)이 **표를 하나의 의미블록**으로 인식(표 중간 절단 금지).
- **R1-5** **출력물(md/txt/docx/pdf)·화면 렌더에 표/서식이 그대로** 반영(⑦). RichText 화이트리스트에 표 태그 추가.

### R2. ④ 완전 대화형 지시 흐름 (사용자 확정: "완전 대화형 재설계")
- **R2-1** 흐름: `입력 → AI 그룹핑(쪼갬) → 각 항목(의미블록)을 채팅처럼 개별 지시·주고받기 → 항목별 완성 → 전체 종합(단일 문서)`.
- **R2-2** 항목별 **다회차 대화**(사용자 지시 → AI 응답 → 재지시 …) 영속. 죽은 경로 `analyzeItem.customInstruction` 배선.
- **R2-3** 전체 종합은 기존 유지(단일 문서, `ai_analysis_documents`).

### R3. "내가 놓친 부분" (사용자 명시 요청 — 반드시 반영)
아래 사이드이펙트/누락을 설계에 박제(01-architecture §리스크 참조):
- html-to-plain SSOT 폭발반경(14+ 호출부) · 유실0 계약 · 붙여넣기↔파일첨부 상호배타 UX · 재그룹(regroup) 표 인식 일관성 · XSS sanitize · 토큰비용 · officeparser 표 출력 실측 · 저장 스키마 text 가정 · 대화 이력 영속/재열람 로드 · 종합이 대화까지 반영.
