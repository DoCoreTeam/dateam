# 00 — Requirements · GPU 경쟁사 시세 무손실·무오염·수렴 재설계 (v0.7.351)

## 문제 (실측 확정)
소프트뱅크 AI데이터센터 URL 통합입력 시 5개 요금 중 3개(월정액 GB200/H100/A100)만 추출되고
A100 시간제(기본료 30,000/월 + 7.2엔/분 + 스토리지 1,000/100GB)·GB200 가격이 소실. 매 실행마다 값이 달라짐.

## 3개 적대분석이 확정한 근본원인
- 🟦 DC-RES: **"100% 자동 무오차"는 현존하지 않음** (SOTA 84~95%, 산업 최고 STP 60~89%). 목표를 "자동 100%"에서 **"확정분 오차 0 + 미확정분 명시 보류 + 수렴"**으로 재정의해야 함.
- 🟦 DC-ANA: **재설계 코드(pivot/obs/classify)가 수동 경로(review/stream)에만 결선**, 자동 일일수집 `market/refresh?auto=1`는 옛 경로(AI가 ÷730/÷720 직접 나눗셈·15k 절단·obs 미충전·JS 미렌더). CLAUDE.md "실제 렌더 경로 우선" 위반.
- 🟦 DC-BIZ: **골든셋:116이 번들÷720÷8을 "정답"으로 박제** ↔ classify는 같은 걸 comparable=false로 판정(오라클↔분류기 모순). 번들 프리미엄 15~40%가 밴드 오염.

## 담을 수 없는 것 (스키마 구조적 결함)
1. **복합/다부 요금제**: obs_amount 스칼라 1개 → 기본료+종량+스토리지 3성분 중 1개만 저장(max 휴리스틱), 나머지 폐기.
2. **per-GB 단위 부재**: obs_pricing_unit enum = 시간 단위만.
3. **기본료(계정단위 고정비)**: validate 게이트가 "非GPU 라벨→reject"로 테스트 고정 → 저장 경로 자체 없음.
4. **동일모델·다른요금구조**: pricing_model enum에 "월정액 flat vs 종량제" 축 없음 → 같은 mapping_id 충돌, is_latest 뷰 없음.

## 캡처와 무관한 silent 오차원
720/730 이원화(±1.4%), 세금 税抜/税込 미보정(±10%), FX 시점 이원(스냅샷 vs 최신), 신선도 무필터(6개월 20~40%), 통화 중의성($=USD/SGD/HKD 고정).

## 목표 (달성 가능한 것)
> 어떤 스타일이 와도 **조용히 틀리지 않는다**: 놓치면 반드시 보이고(reconciliation), 비교불가는 밴드에 안 섞이고(격리), 확정분은 오차 0(결정론+교차확인), 나머지는 명시 보류 후 사람 확정. 검수 오류는 골든 코퍼스로 환류(수렴).

## 비목표
- 100% 자동추출(불가 — 폐기). vision을 주 추출 경로로(구조인식 F1 17~22% — 기각, 교차검증 채널로만).
- "2부 요금제의 보편 단가" 생성(수학적 불가 — 사용량의 함수).

## 범위(사용자 확정: 전체 한 번에)
선결 3건 + 1:N 요금성분 스키마 + 시나리오 기반 실효비용 비교 + reconciliation 완전성 게이트.
