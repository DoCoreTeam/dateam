# 냉정한 전략 재검토 — "왜 땜질이 수렴하지 않는가" (구현 금지)

일자: 2026-06-25 · 분석 전용 · 🟦 DC-ANA + 🟦 DC-BIZ 증거 기반

## 한 줄 결론
**우리는 틀린 문제를 풀고 있다.** "임의 스프레드시트를 AI로 완벽 파싱" — 이건 *우리가 직접 만드는* 파일을 *역공학*하는 자해형 트레드밀이다. 게다가 파싱이 완벽해도 **FX 이중진실**로 USD가 틀리고, **골든셋이 죽은 코드**라 수정이 회귀를 못 막는다. 버그는 품질문제가 아니라 *통제 가능한 입력에 AI-parse-anything을 택한 설계 결정의 예고된 산출물*이다.

## 재고해야 할 전제 7가지 (냉정·객관)

### R-1. ⛔ 정정됨 — FX는 실시간 수신 중. "9.5% 오차" 주장은 틀렸음
- **정정(사용자 지적 반영, 코드 직접 재확인):** 업로드 파일이 타는 review/stream 경로는 **fx_rates(koreaexim) 실시간 환율을 fetch해 사용**(`review/route.ts:217` "환율 변환 마, 서버가 실환율로 처리" + `:362-367` fx_rates fetch→toUsdPerGpuHour). 하드코딩 1370 **안 씀.**
- `1370` 잔존처 3곳 모두 파일 업로드와 무관: 죽은 테스트(`golden-set.ts`), 웹크롤 경로(`market/refresh`), 그리고 `schema-contract.ts:8`의 **모순 잔재 텍스트**(프롬프트에 아직 붙지만 같은 프롬프트 `:217`이 무효화) — **AI 혼선 유발 청소대상(MINOR)**이지 9.5% 시스템 오차 아님.
- → **USD가 이상한 원인은 환율이 아니라 표시 포맷**(format-price.ts 2자리 round 고정 + raw `${}` 노출 5곳, 01 문서 RC-2). 앞 라운드의 FX 오차 주장은 **철회.**

### R-2. 🆕 골든셋이 죽은 코드 — 땜질이 안 붙는 구조적 이유
- `golden-set.ts`는 **합성 클린 문자열만**. 실제 문제shape(다중블록·병합헤더·`H200 141GB.`·X/확인중/Custom) **0건.**
- 정확도 케이스는 "별도 검증서 커버" 주석뿐, **그 검증 미존재.**
- `golden-eval.test.ts`는 **`vitest` import인데 vitest 미설치 + `pnpm test` 목록에도 없음 → 절대 실행 안 됨.**
- 실패 업로드를 영구 fixture로 만드는 메커니즘 **전무.** → 모든 수정이 합성 문자열로만(혹은 0으로) 검증 → **실제 파일 회귀 자유 재발.** ← "몇 번 고쳐도 또 터짐"의 진짜 뿌리.

### R-3. ⛔ 정정됨 — 취합파일은 데이터가 풍부하고 구조가 일관됨. "파싱 포기" 결론 철회
- **정정(사용자 지적 반영):** 취합파일은 데이터가 없는 게 아니다. 실제 벤더 가격이 명확히 존재 — konst tech(A100 1.08, H100 2.4), FPT(H200 3.19/3.09), highreso(3.39/19750.32, 수량14), massedcompute(2.29), Morgen(B200 17000/102000/204000), Crusoe(수량 250/250/4375). 일부 `X/확인중/Custom`은 "데이터 없음"이 아니라 **아직 회신 안 온 칸.**
- **13개 공급사 블록이 전부 동일 구조**(B열 병합헤더=공급사 / 그 아래 A100~B300 / D~G 가격) — 즉 **학습가능한 일관 레이아웃 + 명확한 공급사 라벨 + 실가격.**
- → 🟦 DC-BIZ의 "파싱 포기→수기입력 전환"은 **과한 결론, 철회.** 파일이 못 읽을 만큼 엉망인 게 아니라 **추출 로직이 (a) 병합헤더↔아래행 공급사 연결 실패, (b) 키 정규식 깨짐(F1)** 일 뿐. 핵심은 RC-1·F1을 고치는 것이지 입력방식을 바꾸는 게 아니다.

### R-4. 아키텍처 자기모순 — AI 거세 + 깨지는 결정론 코드
- `transcription.ts:25`가 AI에게 "판단·정규화 하지 마" → 판단을 떠안은 결정론 정규식(`canonical-model.ts:19`)이 버그 온상. **AI로 안 생각하게 만들고, 깨지는 코드로 생각하게 함.**

### R-5. 입력이 '가격표'가 아님 — 범주 오류
- 실제 파일 = 13벤더블록 + 담당자명부 + KRW/USD 거울표(시간당/월720h) + 병합헤더 + 자유서술 노트("2천 H200 UNIT Europe 7월"). 이걸 "가격 행 추출"로 보는 건 범주 오류. 압축·bbox 서브그리딩이 AI가 필요한 2D 맥락을 오히려 파괴.

### R-6. 경로 난립 — SSOT 침식이 재발을 *보장*
- 업로드→데이터 경로 **7+개**(supplier/competitor/stream/USAI·catalog/csv/own-target/competitor-import), FX 전략 3개, 타깃 2개(review_items vs market_prices). slice(0,50)·1400 폴백 복붙 다수. **분기 자체가 불일치를 양산.**

### R-7. 리스크 비대칭 — 정확성이 편의보다 압도적으로 중요
- 이 파이프라인은 **판매가를 결정.** 무음 오파싱 → 잘못된 cost_basis → 잘못된 strategic_price → **실거래 마진 손실.** 저빈도(주~월, 2-3명)·고위험 작업에서 직원 10분 절약은 오파싱 1건 값어치도 없음. 현재 전략은 **싼 변수(클릭)를 최적화하고 비싼 변수(정확성)를 희생** — 거꾸로.

## 전략 권고 (🟦 DC-BIZ verdict)
1. **문제 재정의**: "임의 견적 파싱"이 아니라 "직원이 모은 CSP 가격을 *정확히* DB에 넣기."
2. **입력 통제 활용**: 우리가 만드는 레이아웃을 역공학 말고, **구조화 입력(모델=스펙카탈로그 드롭다운, 수량/통화/리전/약정 타입필드)**. → 모델명 오염 버그클래스 *구조적으로 소멸*.
3. **AI 파서 강등**: 기본경로 → **진짜 외부포맷(벤더 PDF) 전용 탈출구**로, 사람확정 필수(기존 review 큐+admin commit 재사용).
4. **정규식에 버전 그만 올리기**: 새 결정론 정규화기 = 아키텍처가 틀렸다는 증상이지 진보가 아님.
5. **FX 단일화**: AI 프롬프트에서 환산 제거 → AI는 `original_price`+`original_currency`만, 모든 경로가 `normalize-money.toUsdPerGpuHour`에 단일 `krwPerUsd` 주입(경쟁사/USAI는 이미 그럼 — 확장만). 파일 내장 환율 인식 여부도 결정.
6. **회귀 코퍼스 구축**: 실패 업로드 → 영구 fixture 캡처 루프 + 실제 실행되는 테스트 러너(vitest 죽은 코드 제거 or 설치). 텔레메트리(02 문서)가 이 코퍼스의 공급원.
