## Phase 1 (MVP) 완료 조건

### 사용자 요구 직접 충족
- [ ] 월 캘린더가 1차 진입면으로 표시된다
- [ ] 각 날짜 셀에 **그날 작성 인원수**가 보인다 (FR-1)
- [ ] 날짜를 클릭하면 하단에 **그날 작성자 리스트**가 뜬다 (FR-4)
- [ ] 리스트에 **작성일시 컬럼**이 있다 (FR-4)
- [ ] 같은 화면에서 **미작성자 명단**이 보인다 (FR-5)
- [ ] 하루씩 화살표로 넘기지 않아도 월 전체를 한눈에 본다

### 감사·평가 신뢰성 (NFR)
- [ ] 요약 "작성 N명"과 리스트의 distinct 작성자 수가 **항상 일치** (NFR-1)
- [ ] 작성시각/수정시각 구분 — 수정된 로그에 "수정됨" 표시 (NFR-2/3)
- [ ] 모든 시각 KST 일관 표시 (NFR-4)
- [ ] is_onboarding(실습행) 집계·리스트에서 제외 (NFR-5)
- [ ] 10px 미만 폰트 없음, 디자인 토큰 준수, "잘 보임" (NFR-6)

### 기능 구현 기본 정책 (신규 lib 모듈 — Read 중심, 쓰기 없음)
> 본 화면은 **조회 전용**(읽기). CRUD의 C/U/D는 기존 멤버 작성 경로 소관 → 본 기획 대상 아님.
- [ ] **List/Read**: 월 집계 + 선택일 리스트 — 행수준 권한(admin 전용, service-role 명시 필터)
- [ ] 검색(q, sanitize)·정렬(화이트리스트)·필터(부서/타입/kind 화이트리스트) — *Phase2*
- [ ] 서버 페이지네이션(page/limit + count 메타) — *Phase2*
- [ ] 상태 URL 동기화 + 로딩/빈/에러 3종 UI
