## 1. 어느 위치에 (Location)
홈 레이아웃은 `home-layout`(flex column): 헤더 → `home-section-widgets`(데스크탑 3열 그리드) → 미니캘린더 (globals.css:1033~1059).

**제안 — 2단 배치:**
- **(A) 헤더 직하 "오늘 챙길 부서업무" 알림 띠 (full-width, 조건부)**: 기한경과 또는 블로커가 1건이라도 있으면만 노출되는 적색/경고 띠. "🔴 기한 지난 부서업무 2건 · 🚧 블로커 1건 → 바로보기". 0건이면 미표시(노이즈 0). → "빠짐없이 챙김"의 핵심 안전장치.
- **(B) 위젯 영역에 "부서업무" 카드 추가**: 현재 3열(오늘업무·메모·주간보고)에 **4번째 컬럼**으로 추가. `home-section-widgets`를 `repeat(3,1fr)`→`repeat(auto-fill,minmax(280px,1fr))`로 바꾸면 4열 자동 줄바꿈+반응형 동시 해결(태블릿 2열, 모바일 1열). 모바일은 이미 1fr 스택이라 순서만 정의(권장: 오늘업무→**부서업무**→메모→주간보고, 중요도 상위 배치).

> 대안: 부서업무가 "매우 중요"이므로 위젯 한 칸이 좁다면 → 위젯행은 3열 유지하고 **미니캘린더 위에 부서업무 전용 와이드 섹션**(2/3 폭 리스트 + 1/3 요약)도 가능. 1차는 (B) 위젯, 데이터·중요도 검증 후 와이드 승격 권장.

## 2. 어떻게 보여줄 (Display) — "내가 챙길 것"만
홈은 전체 목록이 아니라 **개인화된 챙김 큐**. RLS가 가시범위를 1차 강제하고, 그 위에 개인 필터:
- 담당자 = 나 (`assignee_user_id = me`)
- OR 내 부서의 미지정 업무 (`department_id ∈ 내부서 AND assignee IS NULL`) — 부서장/부서원이 줍줍해야 할 것
- 상태 미완료만 (`planned·doing·blocker`, done 제외)

**정렬(챙김 우선순위)**: ①기한경과(overdue) → ②블로커 → ③오늘/임박(D-3 이내) → ④우선순위(urgent>high) → ⑤기한 가까운 순(null 후순위). (DC-ANA 권장 쿼리 기반, 신규 `listMyDeptTasks` 서버액션 1개 필요.)

## 3. 위젯 와이어프레임 (요약 카드)
```
┌─ 부서업무 ───────────────  전체 12 · ⚠️기한경과 2 · 🚧블로커 1 ─┐
│ [전체] [내담당] [기한임박]            ← 필터 칩(선택)             │
│ ──────────────────────────────────────────────────────────── │
│ 🔴 지남 D+2 | gcube Advisor 리뷰    [블로커] 담당:김성구  ▓▓░ 50%│
│ 🟠 오늘    | 충남 데모 준비          [진행중] 담당:나      ▓░░ 30%│
│ ⚪ D-3     | 견적 회신               [예정]   담당:나      ░░░  0%│
│ ──────────────────────────────────────────────────────────── │
│                         + 9건 더 보기 →  (/dept-tasks)         │
└──────────────────────────────────────────────────────────────┘
```
- 한 줄 = 기한칩 + 제목(ellipsis) + 상태뱃지 + 담당 + 진행률 미니바. 위젯엔 **상위 3~4건만**, 나머지는 카운트+더보기.
- 색: 상태=`STATUS_COLORS`(SSOT), 우선순위=신규 `PRIORITY_COLORS` SSOT(현재 각 파일 하드코딩 → 통합 필요). 기한칩: 지남=danger / 오늘·D-1=warning / 그외=muted.

## 4. 어떻게 상세화 (Detail / Drill-down)
- **3단 깊이**: 홈 요약줄(카운트) → 홈 카드(1줄 요약) → 클릭 시 `/dept-tasks?selected={id}`로 이동해 기존 DeptTaskDetail 패널(상태·진행률·체크리스트·댓글·수정) 표시. 홈에서 상세를 재구현하지 않음(SSOT, 재사용).
- 필터 칩 클릭 → `/dept-tasks?filter=overdue|mine|due` URL 상태로 이동(목록 화면이 같은 필터 적용). 홈↔목록 일관.

## 6. 어떻게 가독성 (Readability)
- **시각 위계**: 기한경과>블로커>오늘>예정 순서가 곧 색 강도(danger→warning→muted). 한눈에 "급한 것이 위·빨강".
- **기한 상대표기**: 절대일자(2026-06-08) 대신 "지남 D+2 / 오늘 / D-3". 절대일자는 hover/상세에서.
- **진행률 미니바**: 숫자+8px 바(체크리스트 자동 산출값). 완료 임박 인지.
- **토큰 일관**: 색/치수 전부 토큰, 상태·우선순위 색맵 SSOT. 다크/테마 자동 대응.
- **밀도 조절**: compact 카드(메모위젯 compact 패턴), 1줄/카드. 모바일은 담당·진행률을 2번째 줄로 접어 카드화.

## 7. 데이터가 많아져도 빠짐없이 챙기는 구조 (No-miss / Scalability)
"홈에 다 안 보여도 누락 없이 챙긴다"를 보장하는 5개 장치:
1. **챙김 큐 정렬**: 급한 것(경과·블로커·임박)이 항상 최상단 → 스크롤 안 해도 위험부터 노출.
2. **요약 카운트줄**: 표시 3~4건이어도 "전체 12 · 경과 2 · 블로커 1"로 총량·위험 총합을 항상 인지(숨겨진 것의 존재를 숨기지 않음).
3. **헤더 알림 띠**: 경과/블로커가 있으면 위젯을 안 봐도 강제 인지.
4. **필터 칩 + 더보기**: [전체][내담당][기한임박] + "+N건 더 보기"→목록(서버 페이지네이션 page/limit). 홈은 큐의 머리, 목록은 전수.
5. **Inbox 0 모델**: 미완료가 0이 되면 "오늘 챙길 부서업무 없음 ✅" 빈 상태 → 남은 게 없음을 명확히(메모위젯 빈상태 패턴). 역으로 1건이라도 남으면 카운트로 계속 환기.
