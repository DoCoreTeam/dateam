# 추가 진단 — 모델명이 실제와 다르게 찍히는 근본원인 (v0.7.246 재현)

> 증상: URL+이미지 첨부 분석 결과, 분류(시장가/Nebius)는 정상이나 **모델↔가격 매핑이 틀림**.
> 예: 실제 HGX B300($4.30/$7.85)가 "H100 80GB"로, HGX B200은 통째 누락, RTX PRO 6000은 "RTX 6000 Ada"로.

## 실측 증거

### 근거 1 — URL은 데이터원이 아니었다 (JS 렌더링)
- `curl https://nebius.com/prices` → **1,484바이트 빈 껍데기**(에러 셸 HTML). GPU 모델 문자열 0개.
- nebius는 클라이언트(JS) 렌더 → 서버 단순 fetch엔 가격표가 **HTML에 존재하지 않음**.
- 결론: 이번 결과는 **100% 첨부 이미지(Vision)** 에서 나옴. v0.7.245의 표보존·상한 상향은 *적용될 표 자체가 없어* 이 사이트엔 무효.

### 근거 2 — 모델 식별이 근본적으로 불신뢰 (가장 심각)
DB 실측: 우리 `gpu_products` 카탈로그(65종)에는 **B300·B200·H100·H200·RTX Pro 6000·RTX 6000 Ada·L40S 모두 존재**. (GB300·GB200만 없음)

그런데도 결과는:
| 실제(Nebius) | 가격 | 찍힌 모델 | 판정 |
|---|---|---|---|
| HGX B300 | $4.30/$7.85 | **H100 80GB** | ❌ 오표기(B300 있는데 H100으로) |
| HGX B200 | $3.95/$7.15 | (없음) | ❌ 누락(B200 있는데 드롭) |
| HGX H200 | $2.45/$4.50 | H200 141GB | ✓ |
| HGX H100 | $2.15/$3.85 | H100 80GB | ✓ |
| RTX PRO 6000 | $0.95/$1.80 | **RTX 6000 Ada 48GB** | ❌ 다른 GPU로 오매핑 |
| L40S | from $0.90/$1.82 | L40S 48GB | ✓ |
| GB300/GB200 | Contact us | (없음) | — (가격없음·카탈로그없음) |

**원인**: `lib/gpu/extract-helpers.ts` `loadSpecContext()`가 추출/분류 프롬프트에 다음을 주입 —
> "입력 모델명이 표준과 다르면 **카탈로그에서 가장 일치하는 표준 model_name으로 매핑하세요**" + 우리 카탈로그 나열.

→ Vision이 이미지를 읽되, **정확 매칭(B300)이 카탈로그에 있어도 근사·환각**해 H100으로 찍거나 B200을 드롭. 메모리(80GB/141GB/48GB)도 실제 표(800/346/200GB)와 무관하게 **카탈로그 값을 가져다 붙임**. 이 "표준 매핑"은 공급가(우리 인스턴스 정규화)엔 맞지만 **경쟁사 시장데이터엔 모델 정체성을 파괴**한다.

### 근거 3 — 모델당 2행 분할
preemptible·on-demand 두 가격을 **각각 별도 행**으로 추출 → 같은 모델이 2번. 약정구분 표시도 없어 중복처럼 보임.

## 결론 (신뢰 불가의 본질)
**LLM 자유추출 + 강제 카탈로그 정규화로는 정형 가격표를 신뢰성 있게 옮길 수 없다.** 정확 매칭이 있어도 근사·드롭·중복이 발생한다. 이것이 "이걸 어떻게 믿냐"의 실체.

## 수정 방향(설계만 — 구현은 승인 후)
1. **경쟁사 모델명 원문 보존**: competitor 경로에선 `loadSpecContext`의 "표준 매핑" 강제를 **끄고**, 원문 모델명(HGX B300 → "B300"/"HGX B300")을 그대로 기록. 카탈로그 매핑은 *분리된 사용자 확인 단계*로.
2. **메모리/스펙 환각 차단**: 추출은 원문에 적힌 값만. 카탈로그 스펙을 빈칸에 채워넣지 말 것.
3. **약정구분 보존**: preemptible/on-demand를 모델 1행의 두 필드로(중복 행 금지).
4. **JS 렌더 사이트**: 서버 fetch 무효 명시 → 헤드리스 렌더링 도입 또는 "이미지/표 붙여넣기" 유도. URL이 빈손이면 사용자에게 분명히 고지.
5. **정형원 신뢰**: 깨끗한 표(이미지/HTML table)는 행 단위로 사용자가 확인·교정 후 반영(추출=후보, 자동반영 금지).
