# 05 지속 학습(Continuous Learning) 설계

## 결론
**네, 계속 학습하게 설계됩니다. 게다가 새 발명이 아니라 이 레포에 이미 돌고 있는 패턴의 재사용입니다.**
단, "모델 IQ가 오른다"는 학습이 아니라 **"우리 데이터·정정·임계값이 누적돼 점점 정확해진다"**는 학습입니다(정직한 정의).

## 이미 존재하는 학습 자산 (재사용 — 코드 확인됨)
| 자산 | 위치 | 무엇을 학습하나 |
|---|---|---|
| 프롬프트 거버넌스 | `lib/gpu/prompt-governance.ts` (evalPromptCandidate·autoActivatePrompt·**monitorAiPromptOutcome·autoRollbackIfDegraded**) | AI가 새 프롬프트를 만들고 평가·자동활성, **성능 떨어지면 자동 롤백** |
| daily 거버넌스 | `lib/daily-prompt-governance.ts`, `docs/2026-06-15-daily-ai-governance/` | 일일 AI 프롬프트 자가보정 |
| 통계 자동학습 | `price_range_learned` 테이블 (mig 029, sample_size≥5면 active) | 데이터 쌓이면 정상 가격밴드(p10/p90/median) **자동 재계산** |
| 임베딩 | `lib/gemini-embedding.ts` | 의미 벡터 — 신규 데이터 들어올수록 매칭 풀 확대 |

→ "자가합성→평가→자동활성→모니터→자동롤백" 루프가 **실제 가동 중**. autolink도 이 골격을 그대로 탄다.

## 학습의 3계층 (autolink에 적용)

### Level 1 — 임계값·통계 학습 (price_range_learned 패턴 재사용)
- 사용자 피드백(연결 유지/해제/승격)을 `autolink_feedback`에 누적.
- 주기 배치가 밴드별 **precision 재계산** → τ_auto/τ_suggest **자동 보정**(해제율↑이면 문턱↑). 
- 관계유형·엔티티종류별 별도 임계값 학습(거래처는 더 엄격, 업무는 느슨).
- price_range_learned처럼 **sample_size 충분할 때만 active**(콜드스타트 보호).

### Level 2 — 정정사례 메모리 (핵심 "계속 똑똑해짐")
사용자의 해제/확정이 그냥 사라지지 않고 **다음 판단에 직접 반영**:
- **부정 캐시**: 해제된 (업무패턴↔엔티티) 쌍은 재추천 억제.
- **Few-shot 주입**: 확정/해제 사례를 LLM judge 프롬프트에 예시로 삽입 → 우리 회사 맥락에 맞게 판정이 개선(프롬프트 거버넌스가 버전관리).
- **별칭 사전 학습**: 확정된 엔티티 매칭에서 "삼성→삼성전자(주)" 같은 **별칭/표기 매핑을 누적** → 다음부터 문자열 매칭 정확도↑(현재 완전일치 한계 해소).

### Level 3 — 프롬프트 자가보정 (prompt-governance 그대로)
- judge/extract 프롬프트가 저성능이면 `monitorAiPromptOutcome`→`autoRollbackIfDegraded`로 **자동 롤백**, 또는 자가합성으로 개선안 생성→eval 통과 시 자동활성. 모든 변경 감사·되돌리기 가능.

### 안 하는 것 (정직)
- 임베딩/LLM **가중치 파인튜닝 안 함** — 비용·운영 과대, 불필요. 위 3계층으로 충분.

## "계속" 학습을 보장하는 장치
- **피드백 = 학습신호 SSOT**: 모든 해제/승격/무수정이 `autolink_feedback`에 기록(누가·언제·무엇).
- **주기 재보정 잡 + drift 감지**: precision이 기준 아래로 떨어지면 알림 + 자동 강등(자동확정→추천).
- **golden-set 회귀**: 라벨셋으로 매 보정 후 정확도 재측정(퇴행 방지).
- **감사·되돌리기**: 모든 자동 보정 이력 보존 → 나빠지면 롤백.

## 정직한 한계
- 초기엔 피드백(라벨)이 적어 학습 신호가 약함 → **보수적으로 시작**(높은 문턱·추천 위주), 데이터 쌓이며 자동확정 확대.
- 학습은 "우리 조직 데이터/정정"에 적응하는 것 — 범용 지능 향상이 아님.
- 피드백이 편향(특정 사용자만 정정)되면 학습도 편향 → 다중 사용자·기간 가중 필요.
