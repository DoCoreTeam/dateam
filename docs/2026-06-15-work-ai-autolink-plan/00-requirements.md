# 00 요구사항 — 업무 AI 완전 자동 연관 연결 (기획 전용, 미구현)

## 사용자 의도 (원문)
"AI가 있기 때문에 기존 데이터들과 연관성을 (AI가) 연결해주길 바란다. 버튼 눌러서 되는 게 아니라. 완전 자동이 맞다. 그걸 할 수 있는 기획을 해서 보고."

→ **목표 = 완전 자동.** 사용자가 연결 버튼을 찾아 누르거나 후보를 일일이 확정하는 행위 없이,
업무를 입력/저장하면 AI가 **즉시 스스로** 기존 데이터(과거 업무·거래처·딜·연락처·GPU견적)와 연관을 찾아 연결한다.

## "완전 자동"의 재정의 (이 기획의 핵심 전제)
완전 자동 = **무개입 연결 생성**. 단, 오연결이 신뢰를 무너뜨리지 않도록 3원칙을 자동의 일부로 내장:
1. **투명(Glass-box)**: 모든 자동 연결은 "왜 연결됐는지" 근거 문장 + 신뢰도를 항상 표시. 침묵 연결 금지.
2. **가역(Reversible)**: 틀린 연결은 1클릭(또는 스와이프) 해제. 자동 생성이므로 원복 비용 0(파괴적 쓰기 아님).
3. **자가보정(Self-correcting)**: 사용자의 해제/수정이 곧 학습 신호 → 임계값·매칭이 점점 정확해짐.

→ 즉 "자동이냐 수동이냐"가 아니라 **"자동 + 항상 되돌릴 수 있고 스스로 나아짐"**. 이래야 완전 자동이 독이 안 됨.

## 기능 요구 (FR)
- FR1: 업무 저장(생성/수정) 시 **자동 트리거** — 사용자 액션 없이 백그라운드로 연관 탐색·연결 생성.
- FR2: 연결 대상 = ①과거/현재 업무(daily_logs) ②거래처(accounts) ③딜(deals) ④연락처(contacts) ⑤(확장)GPU견적.
- FR3: 신뢰도 밴드별 자동 처리 — 高=자동 확정 연결, 中=자동 "약한 연결(점선/추천)"로 표시, 低=연결 안 함(로그만).
- FR4: "업무 플로우/관계도" 패널이 자동 생성된 연결(daily_log_relations, 엔티티 링크)을 **즉시 렌더** — 근거·신뢰도·해제 버튼 포함.
- FR5: 기존 전체 데이터 **백필 임베딩** — 신규 업무가 과거 이력과 연결되려면 과거도 임베딩돼 있어야 함.
- FR6: 해제/수정 피드백 수집 → 임계값 자동 보정(거버넌스).

## 비기능 (NFR)
- NFR1 정확도: 자동 확정(高밴드) precision 목표 ≥ 92% (golden-set 측정). 미달 시 밴드를 추천으로 강등.
- NFR2 비용/지연: 임베딩=저장 시(가벼움), LLM 판정=비동기 큐(저장 응답 막지 않음). token-logger 추적.
- NFR3 안전: 자동 연결은 created_by='ai'로 태깅, 가역. DB 파괴적 변경 없음(연결 행 추가/삭제만).
- NFR4 권한/RLS: 본인/조직 범위 데이터만 연결(default-deny). 타 조직 데이터로 연결 금지.
- NFR5 SSOT: 연결 엔진을 lib/ 단일 모듈로 — daily·부서업무·주간·딜 전 화면 재사용.

## 제외 (이번 범위 아님)
- 실제 구현(이번은 기획·문서만).
- 자동 "삭제"(연결 해제는 사용자/저신뢰 강등만, 원본 업무 자동삭제 금지).
