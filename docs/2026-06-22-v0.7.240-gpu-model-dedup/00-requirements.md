## 배경 (실측 확정)
- gpu_products 352행/77모델 — 모델당 평균 4.6행.
- **352행 중 254행이 확정견적 0건**. 그중 **219행이 2026-06-10 하루에 일괄 생성된 유령 ×N 사다리**(pricing_mode=quote·multi·gcube無·견적0) = `ensureStandardConfigs` 폭증. 35행=gcube 단일카드 시드(존치). 109행(05-28)=초기 시드.
- 유령 사다리에 48GB 단가가 전파되어 96GB가 +355% 오가격(RTX Pro 6000: 실제견적 48GB 1건뿐).
- 순수 표기차 중복=0(정규화 매칭이 케이스변형 흡수 중). 최신화(supply_quotes supersede)는 작동.

## 사용자 요구 (종료조건)
- 앞으로 중복 0 + 기존 중복/유령 자동정리(안전·되돌리기 가능).
- 보수적 캐노니컬 — 확실할 때만 합치고 애매하면(Ada/Pro/Quadro/숫자차) 별도 유지(오병합 0).
- 완전 자동 — 추가 화면 0, 사용자 추가 작업 0.
- 모델 유니크 + 세부데이터(memory·gpu_count)로만 구분.
- 브라우저 테스트.

## 기능 요구
- FR1: 확정 시 model_name을 캐노니컬로 정규화(결정론+보수적 alias). confident=false면 원본 유지.
- FR2: 확정 매칭을 (canonical, memory, gpu_count) 키로 → 기존행 있으면 견적 최신화(supersede), 없으면 1행만 생성. 중복 0.
- FR3: 유령 사다리 자동생성 중단 — 견적 없는 ×N 파생행 신규 생성 금지.
- FR4: 기존 219 유령행 안전 정리(소프트삭제 deleted_at + 백업테이블, 롤백 가능). 시드/gcube/견적행 보존.
- FR5: 전부 서버/코드 자동 — 신규 UI/사용자 단계 0.
