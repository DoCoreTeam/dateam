## 0. 한 줄 결론

> 대표님 방향(**인입을 공급사/경쟁사 탭으로 분리**)은 **옳고 채택**합니다. 다만 그것은 두 고통 중 **하나(분류 모호)만** 풉니다. 나머지 고통(**수동검토 폭주**)은 탭 분리로 안 풀리고 이미 만들어져 있으나 꺼져 있는 **자동확정 게이트를 배선**해야 풀립니다. 그리고 이 둘보다 먼저 **데이터 계약(FK·tier·통화 SSOT)** 을 세워야 "자꾸 이슈가 생긴다"가 재발하지 않습니다.
>
> **집행 순서 = 전략:** ① 데이터 계약 → ② 인입 탭 분리 → ③ 자동확정 게이트 → ④ IA 통합. **빅뱅 재설계 금지, 4단계 순차·독립 롤백.**

## 1. 대표님 요구 vs 코드 현실 (오해 교정 포함)

| 대표 요구 | 코드 현실 | 판정 |
|---|---|---|
| "경쟁사/공급사 받는 부분을 별도로" | 통합입력이 실제로 이 2종을 한 화면에서 받고, 4단 추측 로직으로 분류 → 신뢰 불가 | ✅ **타당, 채택** |
| "명함·사업기회도 통합입력에서 받으니 탭으로" | 명함(lead-intake/contacts)·사업기회(deals)는 **이미 완전 별도 시스템**. GPU 인입과 코드 안 섞임 | ⚠️ **일부 오해** — GPU 인입은 2종뿐. 4탭 신설이 아니라 **2탭 분리**로 범위 절반 축소(=비용 절감) |
| "대량 수동검토가 시스템 존재이유를 없앤다" | 자동확정 3밴드(`confidence-gate.ts`)가 **표시 전용**으로만 존재, 실제 게이팅 미배선 → 100% 사람이 검토 | ✅ **타당, 단 탭분리와 별개 처방** |
| "릴레이션이 안 맞고 이슈가 자꾸" | tier 이중저장·통화 SSOT 우회·model_name 텍스트조인·감사로그 이원화·가격결정 UI 중복 | ✅ **구조적 사실 확인** |

## 2. 두 고통은 다른 병 — 처방·우선순위

| 고통 | 근본원인 | 처방 | 탭분리로 풀리나 |
|---|---|---|---|
| ① 분류가 이랬다저랬다 | 입력 의도를 **시스템이 추측**(키워드→AI→화이트리스트→fallback=supplier 4단) | **탭 분리**(사용자가 탭으로 의도 선언 → 추측 소멸) | ✅ |
| ② 수동검토 폭주 | `confidence-gate`가 표시 전용, 모든 항목 무조건 `pending` | **auto 밴드 자동확정 배선** | ❌ (탭 나눠도 검토함은 똑같이 쌓임) |

**우선순위: ① → ② (논리적 의존성 있음).** 지금 confidence 점수엔 "분류를 맞게 했나"라는 불확실성이 섞여 자동확정 임계를 못 믿는다. 탭 분리로 분류가 결정적이 되면 confidence가 **추출 정확도만** 재는 순수 지표가 되고, 그때 자동확정을 안전하게 켤 수 있다.

## 3. 4개 증상의 근본원인 (직전 진단 종합)

1. **Tier — "T3 고정, T2로 바꿔도 무반응"**
   - `gpu_products.tier`(진짜 카탈로그값) ↔ `supplier_model_tier`(공급사별 라벨, board/cockpit 미반영) **두 저장소 단절**.
   - 공급사 모달 "T2 지정" 드롭다운은 라벨(B)에만 저장 → 가격표 미반영. 코드 주석이 스스로 "가격 불변"이라 명시.
   - `gpu_products.tier`에 **사용자 지정 보호 플래그(override) 없음** → 정규식 자동판정(RTX 4090=무조건 T3)이 언제든 이김.
   - 통합입력 확정(`resolveProductId`)은 tier를 매칭에 **아예 미사용**(SELECT에도 없음).
   - 기본 활성 경로(unified ON)에는 tier를 올바르게 고치는 UI 자체가 없음(정상 UI는 죽은 구뷰에만).

2. **통화 — "원으로 넣었는데 달러 표시"**
   - DB는 결백: `original_currency`/`original_price` 정상 보존. SSOT `fmtMoneyFromOriginal`도 이미 존재(경쟁사 화면엔 배선됨).
   - 공급사 모달(`SuppliersTab.tsx:444`)이 `fmtUSD` **하드코딩** → 무조건 $. 가격표 상세는 전역 토글(기본 KRW) 환산 → 우연히 ₩. **표시 계층의 SSOT 우회**가 원인.

3. **정렬 — "공급사 리스트 정렬 안 됨"**
   - **회귀 아님, 처음부터 미구현.** `SuppliersTab.tsx` 전체에 정렬 코드 0건. 서버 고정정렬(`.order('name')` 등)만 반환.

4. **릴레이션 전반**
   - `model_name` **텍스트 조인**(FK 아님) → 오타·표기변형 시 고아 스펙/tier 행.
   - **감사로그 이원화**: `gpu_audit_logs`(전용) vs 범용 `audit_log`(트리거) → 변동이력 탭이 전자만 조회 → 신규 변경 누락 위험.
   - **가격결정 UI 중복**: 구뷰 `PriceCockpitTab` ↔ 신뷰 `PricingDecisionSection` 병존(URL로 구뷰 도달 가능) → "고쳤는데 화면 그대로" 사고 온상.
   - **market_prices는 gpu_products 직접 미참조**(mapping 경유) → 매핑 누락 시 시세 고아.

## 4. IA 재설계 — 12탭 난립 → 5개 업무 개념

현재 unified 플래그가 탭을 **숨기기만** 해서 IA가 이중(숨긴 라우트 생존 + 구뷰 URL 도달). 숨김이 아니라 **개념 통합**으로:

| 그룹 | 포함 | 사용자 여정 |
|---|---|---|
| **1. 인입** | 공급사 견적 탭 / 경쟁사 시장가 탭 / 검토함 | "넣는다" |
| **2. 가격결정** | board+cockpit **단일 화면**(구뷰 제거) | "값을 정한다" |
| **3. 시장·경쟁** | market / competitors | "시장을 본다" |
| **4. 마스터데이터** | catalog / specs / suppliers / inventory | "기준을 관리한다" |
| **5. 이력·감사** | log / AI조회 + 감사로그 단일화 | "추적한다" |

## 5. 정합성 로드맵 (순서 = 전략, 구현 아님)

**Phase 0 — 데이터 계약 확정 (기반: 이게 먼저면 나머지가 쉬워짐)**
- `model_name` 텍스트조인 → **FK 전환**(고아행 리포트→백필→검증 선행, 덮어쓰기 금지)
- **tier SSOT 단일화**: `gpu_products.tier`=진실, `supplier_model_tier`=참고라벨 강등 + **override 보호 플래그** 신설
- **통화 표시 SSOT 강제**: `fmtMoneyFromOriginal` 우회 화면 회수 + 정적 가드(kst-guard 방식 재활용)

**Phase 1 — 인입 탭 분리 (분류 결정화)**
- 공급사/경쟁사 별도 탭 + 중간단계 필드 분리(`unit_price_usd` 슬롯 공유 해소: 원가 필드 vs 시장가 필드 처음부터 분리)
- 4단 캐스케이드 판정 제거, "→경쟁사" 이동 버튼은 **예외 처리**로 격하

**Phase 2 — confidence-gate 실배선 (검토 폭주 해소 = 시스템 존재이유 회복)**
- auto 밴드(≥임계) 자동확정 + 감사로그·되살리기 연동
- 초기 임계 **보수적(95+)** + 샘플링 검수 병행 후 점진 하향 (유실0 백본 위에서만 가동)

**Phase 3 — IA 통합·표면 정리**
- 구뷰 제거(리다이렉트 유예 1릴리즈), 5그룹 재편, 감사로그 이원화 통합

> **핵심 논리: 스키마 정합 → 인입 결정성 → 자동화 → 표면 정리.** 역순(탭 UI부터)이면 겉만 바뀌고 "이슈가 자꾸 생긴다"의 본질이 남는다.

## 6. 리스크

- ⚠️ **빅뱅 금지** — 가격결정은 gcube 실판매가 직결 매출 경로. 각 Phase 독립 배포·독립 롤백.
- ⚠️ **FK 전환 = 고아행 노출** — 마이그 전 고아 리포트→백필→검증. 기존데이터 덮어쓰기 금지.
- ⚠️ **자동확정은 유일하게 "사람 없이 쓰는" 변경** — 감사·되살리기 백본 위, 보수적 임계로 시작. 성급한 임계 하향 = "신뢰불가" 불만이 자동화 불신으로 전이(최악).
- ⚠️ **구뷰 제거 시 롤백 경로 소멸** — 신뷰 완전검증 전 삭제 금지, URL 리다이렉트 유예.
- ⚠️ **습관 회귀** — 혼합붙여넣기 익숙한 사용자 반발 → 경고+행이동으로 완충.

## 7. 영업 관점 (fable5)
"비정형 대량 인입 → AI 추출 → 임계 자동확정 → 감사·복원" 파이프라인은 GPU 외 도메인 재사용 레퍼런스 자산. **Phase 2까지 가야** "AI가 검토까지 줄여준다" 데모 스토리 성립 — 탭분리만 하고 멈추면 영업 스토리 안 나옴.

## 8. 신뢰 가능/불가 데이터 (대표 안심 포인트)
- ✅ **안전(유실 없음)**: 입력한 원본 단가·통화(`original_*`), 공급가(`cost_basis_krw`). **다시 넣을 필요 없음.**
- ❌ **표시만 틀림(값 정상)**: 통화 $/₩ 표기, 공급사 모달 tier 라벨.
- ⚠️ **의도와 다를 수 있음**: tier 표시값(자동판정이 수동값 덮을 수 있는 구조).
- 결론: 표시·배선을 고치면 **지금 DB 값이 그대로 올바르게 보임.**
