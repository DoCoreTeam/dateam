## 0. 원칙
1. **토큰 이름 고정, 값만 테마별 스왑** → 치환 후 화면 외형은 현재(NB)와 동일해야 함(무회귀).
2. **CSS 속성별 의미로 매핑** — 같은 hex라도 `color:`→텍스트, `border`→보더, `background`→표면 토큰. (단순 hex→1토큰 일괄 치환 금지)
3. **시맨틱 상태색(성공/위험/경고/정보)은 테마 공통**이되 토큰화(테마가 바꾸고 싶으면 가능).
4. 치환 완료 후 **하드코딩 hex 잔여 = 0**(그라디언트·예외 제외 목록만 남김).

## 1. 추가할 시맨틱 토큰 카탈로그 (globals.css :root, 현재 NB값으로)

  --text-faint:  #94a3b8;  /* 흐린/플레이스홀더 */
  /* 표면 */
  --surface:        #ffffff;  /* 카드/입력 배경 */
  --surface-bg:     #f8f8f6;  /* 페이지 배경 */
  --surface-muted:  #f1f5f9;  /* 약한 강조 배경(칩/hover) */
  /* 보더 (기존 --border-color 재사용) */
  --border-color:   #0a0a0a;
  --border-subtle:  #cbd5e1;  /* 약한 구분선 */
  /* 브랜드 (기존 --brand 재사용) + 틴트 */
  --brand: #7c3aed; --brand-dark: #5b21b6;
  --brand-soft:   #f3effe;  /* 연보라 배경 */
  --brand-soft-2: #ede9fe;
  /* 시맨틱 상태 — fg / bg / border 3종씩 */
  --success: #16a34a; --success-bg: #f0fdf4; --success-border: #bbf7d0;

## 3. 치환 방법 (속성 인지 codemod — 단일 실행)
정규식이 아닌 **CSS 속성 컨텍스트 기준** 치환 스크립트 1개로 일괄 처리:
- 인라인 `style={{ color: '#64748b' }}` → `color: 'var(--text-muted)'`
- `borderColor`/`border: 'Npx solid #...'` → `var(--border-color|--border-subtle)`
- `background`/`backgroundColor` → surface/bg/status-bg 토큰
- `boxShadow` 색 → `var(--shadow-color)` (이미 대부분)
- globals.css의 동일 패턴도 동시 처리
- 매핑은 §2 표를 dict로. 속성+hex 조합으로 결정. 애매하면(같은 hex 다속성) 속성으로 분기.

**1회 실행 스크립트 구성(예시 — 구현 시):**
1. `scripts/tokenize.mjs` — 파일 순회, 각 인라인 style 선언을 파싱해 (property, hex)→token 치환
2. status 의미색은 SSOT 경유로 치환(직접 hex 잔존 제거)
3. 실행 후 `grep -r '#[0-9a-fA-F]\{6\}'`로 잔여 리포트 → 예외목록과 대조
