# 04 · 테스트 전략 & 엣지케이스

## A. 권한 매트릭스 테스트 (핵심)
6인 조직 픽스처: 대표(전사) / 영업본부장 / 영업1팀장 / 영업1팀원 / 영업2팀장 / 개발본부장.

| 주체 | 영업1팀원 원본 | 영업1팀 취합본 | 영업2팀 취합본 | 개발본부 취합본 | 편집/취합 가능 |
|------|:--:|:--:|:--:|:--:|--|
| 영업1팀원 | ✅(본인) | 👁 | ❌ | ❌ | 자기 원본만 |
| 영업1팀장 | ✅ | ✏️ | ❌ | ❌ | 영업1팀 |
| 영업본부장 | 👁 | 👁 | 👁 | ❌ | 영업본부(직속) |
| 개발본부장 | ❌ | ❌ | ❌ | ✏️ | 개발본부 |
| 대표 | 👁 | 👁 | 👁 | 👁 | 전사 조회/자기직속 |

✅작성·✏️편집/취합·👁조회전용·❌차단. **각 셀을 RLS 레벨(직접 쿼리)·API 레벨 양쪽에서 검증**.

## B. Closure 정합성 테스트
- 노드 INSERT/이동/삭제 후 closure가 재귀CTE로 계산한 정답과 100% 일치.
- 사이클 이동 시도 → BEFORE 트리거 `RAISE EXCEPTION`.
- 서브트리 이동: 외부조상 경로만 삭제, 내부경로 보존.
- 백필 결과 = 트리거 누적 결과 동일.

## C. RLS 안전·성능 (DC-SEC/DC-RES)
- **재귀 회귀 방지**: closure/헬퍼함수 도입 후 PostgREST JOIN에서 무한재귀 없음(003 회귀 테스트).
- **우회 차단**: `authenticated` 롤로 `org_node_closure` 직접 SELECT 거부.
- **성능**: 수백명 픽스처에서 `=ANY(ARRAY(select fn()))` 패턴 응답시간 측정(목표 수ms), 행단위 함수호출 패턴과 비교.
- **격리 누수**: 타 부서 보고서가 어떤 경로(목록/검색/AI취합 입력/DOCX export)로도 새지 않음.

## D. 엣지 비즈니스 케이스 (DC-BIZ — 80% 시스템화, 20% 운영)
| 케이스 | 기대 동작 | 테스트 |
|--------|----------|--------|
| 겸직 부서장 | 부서별 독립 권한, readable=두 서브트리 합집합 | 다중 head_user_id 픽스처 |
| 부서 이동 직원 | 과거 보고서=옛 부서(동결), 신규=새 부서 | department_id 스냅샷 검증 |
| 부서장 공석 | 상위 부서장이 자동 대행(fallback), 시스템 거부 안 함 | head=null 부서 |
| 조직개편(통폐합) | 과거 보고서 경로 보존, 현 권한은 현 조직도 기준 | parent 재배치 후 가시성 |
| 본부간 예외 열람 | 명시적 grant만(묵시 예외 금지) | (P3+ grant 테이블 시) |
| 대표 열람 | 전사 조회 + access_log 기록 | is_executive 경로 |

## E. UI/UX 테스트
- 역할별 탭 노출(팀원=트리 없음, 부서장=취합, 상위=조직현황, 대표=대시보드).
- 조회전용 영역에 편집 컨트롤 **미렌더**(disabled 아님).
- 반응형 320/768/1024/1440: 트리→드롭다운 축약, 카드그리드 1열, 가로스크롤 0.
- 키보드 네비/aria/대비 4.5:1.

## F. 롤아웃 테스트
- 피처플래그 OFF=기존 전원열람 동일(무회귀).
- Shadow→Soft→Hard 전환 시 데이터 가시성 변화가 의도대로.
- 롤백: 정책 swap 후 전원열람 즉시 복귀.

## G. 테스트 도구
- 권한/RLS: Supabase 로컬 + SQL 픽스처로 역할별 세션 시뮬레이션(`set request.jwt.claim.sub`).
- 순수 로직(closure 계산·권한집합 파생): node 단위테스트.
- UI: Playwright 역할별 시나리오 + 반응형 스냅샷.
