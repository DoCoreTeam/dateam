# 02 · UI/UX 설계 — 역할별 화면 (최상안)

> 기획 전용. 와이어프레임은 ASCII 스케치. 반응형·카드레이아웃·터치44px 정책 준수.

## 설계 원칙
1. **하나의 라우트, 권한에 따라 자라는 화면** — `/weekly-report` 한 곳에서 역할에 따라 보이는 범위가 점진 확장(별도 어드민 화면 불필요, 요구 2번).
2. **"내 보고 → 우리 부서 → 하위 조직"의 동심원 구조** — 항상 자기 위치에서 시작해 권한이 닿는 만큼 바깥으로.
3. **편집 가능(자기부서)과 조회 전용(하위)을 시각적으로 명확히 구분** — 색·잠금 아이콘·배지.
4. **계층은 좌측 조직트리로 항상 맥락 제공**, 우측은 선택 부서의 취합본.
5. **1:N 일반화 — "다부서 보유 노드"는 누구나 멀티부서 대시보드를 본다.** 대표이사는 그 최상위 특수 케이스. 화면은 역할 하드코딩이 아니라 **선택 노드의 자식 부서 수**로 분기(≥2 카드그리드 / 1 단일뷰 / 0 잎). 동일 컴포넌트(`ScopeDashboard`)를 scope만 바꿔 재사용.

## 역할별 IA (정보구조)

### 팀원 (부서장 아님)
```
/weekly-report
 ├ [탭] 내 주간보고   ← 작성/수정 (기존 폼)
 └ [탭] 우리 부서 취합 ← 조회 전용(부서장이 확정한 취합본). 편집 불가, 🔒
```
- 다른 부서 탭/조직트리 **노출 안 함**(격리). 좌측 트리 영역 자체가 렌더되지 않음.

### 부서장
```
/weekly-report
 ├ [탭] 내 주간보고
 └ [탭] 우리 부서        ← ★취합 권한
      ├ 좌: 부서원 원본 목록(체크/상태)
      ├ 우: [AI 취합 실행] → 취합본 편집기(draft) → [확정]
      └ 확정 시 dept_weekly_reports 스냅샷 저장 → 팀원에게 "팀 전체 취합" 노출
```

### 상위 부서장(본부장)
```
/weekly-report
 ├ [탭] 내 주간보고
 ├ [탭] 우리 부서        ← 편집/취합(자기 본부)
 └ [탭] 조직 현황        ← ★계층 뷰
      ┌ 좌: 조직트리(내 관할만) ┬ 우: 선택 부서 취합본(조회 전용 🔒)
      │  ▼ 영업본부(나)        │   [영업1팀 취합본] confirmed ✓
      │   ├ 영업1팀            │   성과/계획/이슈 …
      │   └ 영업2팀            │   (편집 불가 — 하위부서)
      └─────────────────────┴────────────────────────────
```
- 자기 본부=편집 가능, 하위 팀=조회 전용. 트리 노드에 ✏️(편집권) / 👁(조회권) 아이콘으로 즉시 구분.
- **자기 노드가 자식 부서 ≥2개면** "조직 현황" 탭 기본 화면이 아래 **멀티부서 대시보드**(카드 그리드)로 전환 — 본부장도 미니 전사뷰를 갖는 셈.

### ★ 멀티부서 대시보드 — 다부서 보유 노드 **공통** (대표이사는 특수 케이스)
> **핵심(1:N 매핑):** "부서 카드 그리드 + 드릴다운"은 대표이사 전용이 아니다.
> **자식 부서를 2개 이상 가진 모든 노드의 장(長)**은 동일한 멀티부서 대시보드를 본다.
> 대표이사 = 최상위 노드가 모든 본부를 자식으로 갖는 **특수 케이스**일 뿐, 본부장(N개 팀 보유)·실장(N개 그룹 보유)도 자기 관할 범위로 동일하게 동작한다.

```
규칙: render(node) =
   자식 부서 ≥ 2  →  멀티부서 대시보드 (자식들을 카드로) + 드릴다운(재귀)
   자식 부서 = 1  →  단일 취합본 뷰로 바로
   자식 부서 = 0  →  자기 부서 취합본(잎 노드)
```

예시 ① 대표이사 (최상위, 본부 N개):
```
[주차 ◀ 2026-W22 ▶]                              [전체 펼쳐보기 ⤢]
 ┌── 전사 현황 (직속 자식 = 본부 카드) ─────────────────┐
 │ ┌─영업본부──────┐ ┌─개발본부──────┐ ┌─경영지원──────┐ │
 │ │ 취합 4/4팀 ✓   │ │ 취합 2/3팀 ⚠   │ │ 취합 2/2팀 ✓   │ │
 │ │ 완료율 72%     │ │ 완료율 55%     │ │ 완료율 90%     │ │
 │ └──[드릴다운]───┘ └──[드릴다운]───┘ └──[드릴다운]───┘ │
 └──────────────────────────────────────────────────┘
```
예시 ② 영업본부장 (본부 1개 = 자식 팀 N개) — **동일 컴포넌트, scope만 다름**:
```
[주차 ◀ 2026-W22 ▶]                              [본부 전체 펼쳐보기 ⤢]
 ┌── 영업본부 현황 (직속 자식 = 팀 카드) ────────────────┐
 │ ┌─영업1팀───────┐ ┌─영업2팀───────┐ ┌─영업3팀───────┐ │
 │ │ 취합 ✓ 80%     │ │ 취합 ⚠ 40%     │ │ 미제출         │ │
 │ └──[드릴다운]───┘ └──[드릴다운]───┘ └──[드릴다운]───┘ │
 └──────────────────────────────────────────────────┘
 ※ 자기 본부 직속 = 취합/편집 가능, 하위 팀 = 조회 전용(🔒)
```
- **드릴다운은 재귀**: 본부 카드 클릭 → 그 본부가 또 자식 부서 여럿이면 다시 멀티부서 대시보드, 잎이면 취합본.
- 카드 = 부서 단위 요약 타일(취합 진척·완료율·미제출·핵심이슈 수).
- 두 모드: **드릴다운**(평소) / **전체 펼쳐보기**(관할 전체 취합본 세로 스택 — 검토·DOCX). 요구 7번을 **모든 다부서 노드로 일반화**.
- 시작 노드 = 사용자의 **최상위 관할 노드**(`my_readable_dept_ids` 중 조상이 없는 것). 대표=전사 루트, 본부장=자기 본부.

## 핵심 컴포넌트
| 컴포넌트 | 역할 | 비고 |
|---------|------|------|
| `OrgScopeTree` | 좌측 관할 조직트리 | 기존 `OrgPublicTree` 재사용 + 관할 필터(`my_readable_dept_ids`) + ✏️/👁 권한 배지 |
| `DeptReportEditor` | 부서장 취합 편집기 | 기존 `AdminReportsPreview`의 AI취합 로직 재사용, 위치만 사용자화면으로 |
| `DeptReportView` | 취합본 조회(읽기전용) | 🔒 잠금 상태·확정 배지·편집자·갱신시각 |
| `ScopeDashboard` | **다부서 노드 공통** 자식부서 카드 그리드 (대표이사·본부장 등 모두 재사용, scope 노드만 prop) | `responsive-grid-cols-3`, 모바일 1열. 자식부서 수에 따라 카드/단일뷰/잎 자동 분기 |
| `WeekPicker` | 주차 네비 | 월요일 기준, 기존 week_start 규칙 |

## 상태·피드백 디자인
- 취합본 상태 배지: `초안(draft)` 회색 / `확정(confirmed)` 녹색 / `재취합 필요`(원본 변경 감지=source_hash 불일치) 주황.
- 조회 전용 영역: 좌상단 🔒 + "조회 전용 — {부서명}" 라벨 + 편집 컨트롤 비표시(disabled 아님, 아예 미렌더).
- 미제출/공석 부서: 카드에 ⚠ + 사유(부서장 공석/미취합) 명시.

## 접근성·반응형
- 조직트리/카드 모두 키보드 네비(`aria-expanded`, `aria-controls`), 색 대비 4.5:1.
- 모바일: 좌측 트리 → 상단 드롭다운(부서 선택)으로 축약, 카드 그리드 1열, 취합본은 카드 레이아웃(가로 스크롤 금지).
- 터치 타깃 ≥44px.

## 어드민 화면 변화
- `/admin/reports`("주간보고 취합")의 **취합 기능은 사용자화면(부서장)으로 이관**. 어드민엔 "전사 감사/내보내기"만 잔류하거나 제거(요구 2번: "어드민엔 불필요").
- 이관은 P3 정착 후 — 그 전까지 어드민 취합 **병행 유지**(부서장 적응 기간, DC-BIZ 권고).
