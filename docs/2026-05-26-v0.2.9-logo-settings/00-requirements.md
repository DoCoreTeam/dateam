# 00. 요구사항 정의 — 시스템 로고·브랜드명 관리

## 배경
현재 시스템 이름("AX사업본부")이 사이드바·로딩 애니메이션·로그인 화면에 하드코딩되어 있어,
브랜딩 변경 시 코드 수정·배포가 필요함. 어드민에서 로고 이미지와 브랜드명을 동적으로 관리할 수 있도록 한다.

---

## 기능 요구사항

### FR-01. 로고 업로드
- admin 권한 보유자만 접근 가능
- 허용 형식: PNG / JPG / SVG / WebP
- 권장 사이즈: 가로 400px 이상, 세로 200px 이하 (비율 자유)
- 최대 파일 크기: 2MB
- 업로드 후 미리보기 즉시 표시
- 저장소: Supabase Storage (`branding` 버킷, public read)
- 파일명: `logo-{timestamp}.{ext}` (충돌 방지)

### FR-02. 브랜드명 편집
- 텍스트 최대 30자
- 로고 이미지가 없을 때 사이드바·로딩 화면에 브랜드명 텍스트 표시
- 로고 이미지가 있으면 이미지 우선 표시

### FR-03. 로고 반영 위치
| 위치 | 현재 | 변경 후 |
|------|------|---------|
| 사이드바 상단 | "AX사업본부" 텍스트 하드코딩 | 로고 이미지 or 브랜드명 텍스트 동적 |
| 로그인 화면 | 별도 로고 없음 | 로고 이미지 or 브랜드명 표시 |
| 로딩 애니메이션 | orgName="AX사업본부" 하드코딩 | DB에서 읽은 브랜드명 / 로고 이미지 |

### FR-04. 기본값 (폴백)
- 로고 미설정 시: "로고" 텍스트가 포함된 회색 플레이스홀더 이미지
- 브랜드명 미설정 시: "AX사업본부" (시스템 기본값)

### FR-05. 반영 시점
- 저장 즉시 DB/Storage 업데이트
- 다른 사용자는 **다음 페이지 로드** 시 반영 (SSG revalidate 또는 서버 컴포넌트 fetch)
- 저장한 admin은 저장 직후 UI 즉시 반영 (클라이언트 상태 갱신)

---

## 비기능 요구사항
- NFR-01. 이미지 로드 실패 시 브랜드명 텍스트로 자동 폴백
- NFR-02. Supabase Storage public URL 사용 (CDN 별도 불필요)
- NFR-03. 로고 이미지 alt 텍스트 = 브랜드명 (접근성)
- NFR-04. 구 로고 파일은 Storage에서 삭제 (스토리지 낭비 방지)
