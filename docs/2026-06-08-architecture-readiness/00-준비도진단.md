# 아키텍처 준비도 심층 진단 (구현 0 — 분석 전용)
> 2026-06-08 · 질문: "디자인 샘플로 일괄 변경 가능한가 / 새 기능·화면도 정확히 동작하는가"

## 한 줄 답
**색 변경은 거의 준비됨(95%). 그러나 (1) 간격·폰트크기 토큰이 아예 없고 (2) 공용 컴포넌트가 만들어만 두고 채택 0% (3) 하드코딩 차단 장치(CI/pre-commit)가 없어서 — "어떤 디자인 샘플이든 일괄"과 "새 화면 자동 정합"은 아직 미완.**

## 준비도 스코어카드
| 능력 | 수준 | 블로커 |
|------|------|--------|
| 색 디자인샘플 일괄 적용 | 95% ✅ | api계열 5파일 hex 잔존 |
| 형태(두께/radius) 일괄 적용 | 40% ⚠️ | TSX 인라인 borderRadius 113건이 var(--radius) 우회 |
| **간격/폰트 일괄 적용** | 5% ❌ | **spacing·font-size 스케일 토큰 자체가 없음** (fontSize 1,474 + padding 853 리터럴) |
| 세 번째 테마 추가 | 높은비용 ⚠️ | classic이 :root 50토큰 중 27개 미오버라이드 = 구조적 누락 반복 |
| 새 화면 자동 테마대응 | 레이아웃만 ✅ | **NbButton/NbCard/NbBadge 채택 0%**(import 0, 직접 className 64~74곳) |
| 하드코딩 재유입 차단 | 없음 ❌ | 가드 스크립트 수동만 — pre-commit/CI/npm script 미연결 |

## 축1 — 디자인 샘플 일괄 변경
- ✅ 색(99% 토큰), 그림자, 보더두께(CSS), 폰트(tape), 사이드바, 구조요소(.tape-title 테마분기)
- ❌ **spacing/gap/padding 토큰 전무** — rem 수치 1,300+곳 산재, 단일 통제점 없음
- ❌ **font-size 스케일 전무**(--text-xs/sm/.. 없음) — fontSize 1,474곳 즉석 수치
- ❌ z-index 토큰 전무(28곳 리터럴)
- ⚠️ TSX 인라인 borderRadius 113건이 토큰 우회 → 테마 radius 변경 안 먹음
- ⚠️ 새 테마 추가 시 27개 토큰(특히 --success/danger/warning/info, --color-*, --gpu-* 22개) 자동 누락 구조

## 축2 — 새 기능/화면 정합
- ❌ **공용 컴포넌트 채택 0%** — NbButton/Card/Badge 만들었으나 아무도 import 안 함. 새 화면은 계속 인라인 재구현 → 재유입
- ❌ **강제 장치 없음** — CI(.github) 없음, pre-commit 없음, package.json에 가드 미연결, 디자인 lint 룰 없음 → 빌드가 하드코딩 못 막음
- ⚠️ 컨벤션 문서 1줄만(CLAUDE.md) — "NbButton 써라" 구체 지침 없음
- ✅ 레이아웃 자동상속 — (member)/admin layout이 MobileShell 전파(신규 페이지 자동 적용)
- ⚠️ page-inner 48p 중 11p만 사용 / SWRConfig 전역 없음 / admin은 status-colors SSOT 미채택
- ⚠️ globals.css 3,786줄 평면(GPU만 1,280줄) — 새 CSS 파일 끝 누적

## 결론
"앞으로 디자인 바꾸면 일괄·새 화면도 정확히"가 **완전히 보장되진 않음**. 색은 되지만 **간격/폰트 토큰 부재 + 컴포넌트 미채택 + 강제장치 부재**가 3대 구멍.

## 권고 우선순위 (구현 시 — 별도 승인)
### P0 (즉효·저비용)
- P0-1 **CLAUDE.md에 "신규 버튼=NbButton/카드=NbCard/뱃지=NbBadge/색·치수=토큰" 강제 규칙** 등록
- P0-2 **가드 스크립트 pre-commit(또는 CI) 연결** → hex/치수 재유입 자동 차단
### P1 (테마 완성)
- P1-1 classic 등 테마에 누락 27토큰 보완 + "테마별 필수 오버라이드 목록" 명문화
- P1-2 GPU 토큰 22개 테마 대응(또는 메인 토큰 alias)
- P1-3 TSX borderRadius 113건 → var(--radius) codemod(기존 스크립트 재사용)
### P2 (신규개발 표준)
- P2-1 **spacing·font-size 스케일 토큰 신설**(--space-*, --text-*) + 신규부터 강제
- P2-2 page-inner 준수율↑(가드에 룰 추가)
- P2-3 globals.css 도메인 분리(@import: tokens/components/gpu/calendar)
### P3
- z-index 토큰화 / admin status-colors SSOT 채택 / SWRConfig 전역

## 핵심 메시지
지금은 **"색 디자인 시스템"은 섰지만 "형태·간격·타이포 + 강제·채택"은 미완**. P0(컴포넌트 규칙+가드 연결)와 P2-1(spacing/typography 토큰)을 하면 "어떤 디자인 샘플이든 일괄, 새 화면도 자동 정합"이 실현됨.
