# 확정 기획 — GPU 경쟁사 시세 흡수 재설계 (구현 금지 / 승인 후 착수)

확정일 2026-07-20 · 상태: **기획 확정(락) / 구현 0**
스코프: "통화 확장"이 아니라 **"관측 원본 완전 보존 + 결정론 정규화 + 비교가능 게이트"**.

## 확정 원칙 (락)
1. **저장 = 관측 원본 불변** (환산값은 파생, 언제든 재계산). float 금지 → NUMERIC.
2. **환산·나눗셈은 코드(SSOT)만**. AI에게 환율·÷장수·÷시간 금지.
3. **환율 이원화**: 표시=최신환율(+적용환율 명시) / 판가·이력=관측일 환율 스냅샷 고정. 재환산은 항상 원금액에서.
4. **비교가능한 것만 콕핏 밴드에**. 번들·오래된·미지 시세는 격리(참고).
5. 공유 SSOT 수정 → **골든셋 회귀 게이트 먼저**.

## 확정 저장 필드 (시세 1건)
`원금액(NUMERIC) · 통화(ISO3) · 기간(분/월…) · 장수 · 세금기준(税別/税込) · 번들포함범위 · 세그먼트(raw/번들) · 그날환율 · 환율일자 · 환율소스 · 수집일 · 출처(URL/증빙) · 확정주체(auto/human) · 근거(provenance)`

## 확정 항목 12 (5핵심 + 7갭)
| # | 항목 | 확정 |
|---|---|---|
| 1 | 원본 저장 | 관측 그대로(위 필드). 지금의 "÷8÷730 된 파생값=원본" 폐기 |
| 2 | 전통화 환율 | fx_rates 다통화 확장(AP01 이미 전통화). **JPY 100단위 ÷100**, 콤마파싱, 휴일→직전영업일 폴백 |
| 3 | 환산함수 SSOT | normalize-money: 원본→원화/GPU·시간. 표시=최신/판가=스냅샷 |
| 4 | 추출 합치기 | 전사(원가보존)+구조화(모델인식, specContext오염 제거) **병합+수치대조**. 근거 span 필수 |
| 5 | 번들·요금라벨 게이트 | `GPU利用料金` 오통과 차단(“GPU”단독 금지), 번들=별도표시(순수GPU와 분리) |
| 6 | **통화기호 중의성** | `$`=USD/SGD/HKD/AUD, `¥`=JPY/CNY → **사이트 국가/도메인으로 통화 확정**, 애매하면 검수 |
| 7 | **세금기준** | 税別/税込(부가세 포함여부) 원본에 기록 |
| 8 | **재수집 = 관측 누적** | 덮어쓰기 아님 → 관측을 시계열로 append(추세 정직), "최신" 뷰 별도 |
| 9 | **표시 전체 + Export** | 콕핏·시장탭·가격표·엑셀 Export·고객판매가표 전부 같은 SSOT. Export는 **내보낸 시점 환율로 파일 내 동결** |
| 10 | **FX 자동수집** | 기존 일1회 트리거를 전통화로 확장 + 수출입은행 **일일 호출한도** 처리 |
| 11 | **판가 변화 고지** | 번들격리·stale제외로 콕핏 추천판가가 달라짐 → 배포 시 사용자 안내 |
| 12 | **기존 데이터** | additive, NULL=USD가정 유지, 과거 무손상 backfill |

## 확정 순서 (큰 오류부터)
P0 골든셋 4대 사고 fixture(회귀게이트) → P1 원본저장 스키마 → P2 FX 다통화 → P3 정규화 SSOT(이원화) → P4 비교가능성(세그먼트·번들·신선도·통화중의성) → P5 추출 병합.
