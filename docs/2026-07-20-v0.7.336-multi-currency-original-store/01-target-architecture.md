# GPU 경쟁사 시세 흡수 — 목표 아키텍처 (최고 정확도·최소 데이터오류) / 미구현

- 작성 2026-07-20 · 상태: **기획 확정 / 구현 금지(사용자 승인 대기)**
- 근거: 🟦 DC-ANA(현황·오류점) + 🟦 DC-RES(업계 정석) + 🟦 DC-BIZ(데이터정합·사각지대) 종합
- 스코프 재정의: **"통화 확장"이 아니라 "시세 1건의 관측 사실 완전 보존 + 결정론 정규화 + 비교가능성 게이트"**. 통화는 그 안의 한 필드.

---

## 0. 핵심 반전 — 통화는 가장 작은 오류원이다

오류 기여도 순위(🟦 DC-BIZ):
**① 번들/세그먼트(수십~수백%) > ② 단위·수량(수십%~자릿수) > ③ 신선도(시장 변동분 통째) > ④ 통화 환율(수%)**

→ 사용자가 출발점으로 삼은 "통화"는 4순위. 통화만 고치면 **"환율까지 명시된 정밀해 보이는 오염 시세"**가 늘어 오히려 위험(정밀함이 신뢰를 위장). 그래서 아래 5축을 함께 설계.

## 0-1. 더 큰 반전 — 지금 저장되는 "원본"은 원본이 아니다 (🟦 DC-BIZ #1)

현 스키마(마이그 134)의 `original_price`는 이미 **÷8장 ÷730h 환산이 끝난 파생값**. 소프트뱅크 ¥2,500,000(월·8장·스토리지 포함)이라는 **진짜 원본은 어디에도 안 남고**, `428円/h`가 "원본"이란 이름으로 저장됨. 나눗셈 가정(장수·월시간·포함범위)이 전부 증발 → 나중에 교정·감사 불능. **목표: 관측된 그대로(금액+통화+기간단위+수량단위+포함범위+근거)를 원본으로 보존.**

## 1. 목표 데이터 모델 — "관측 원본 + 결정론 파생" 분리

시세 1건 = **관측 사실(불변) + 파생 표시값(재계산 가능)**.

**관측 원본(저장, 불변):**
- `amount_original` NUMERIC — 관측된 그대로 (예 2,500,000). **float 금지, NUMERIC**(DC-RES: Stripe/Google micros 원리 = 이진부동소수 금지).
- `currency` CHAR(3) ISO4217 (JPY/KRW/USD…)
- `pricing_unit` — hour/month/minute/… (FOCUS 스펙 PricingUnit 분리)
- `pricing_quantity` + `gpu_count` — "1분당·1장" 등 수량 축
- `inclusions` — 번들 포함범위(storage/network/software…) + `bundle_inclusive` bool
- `segment` — `raw_gpu` | `managed_bundle` (콕핏 밴드 격리용)
- **환율 스냅샷**: `fx_rate` + `fx_rate_date` + `fx_source`(koreaexim) — 관측 시점 환율 동결(회계 ASC830/IAS21 정석)
- **품질메타(DC-BIZ 안전장치)**: `observed_at`(수집일) · `source_url`/증빙 · `confirmed_by`(auto|human) · `provenance`(원문 근거 span)

**결정론 파생(표시·집계, 재계산 가능):**
- `unit_price_krw_per_gpu_hour` = 결정론 함수(관측원본, 환율)로 산출. **표시=최신환율 재환산 / 판가·밴드=스냅샷 고정** (이원화).

## 2. 환율 이원화 (DC-RES 정석 + DC-BIZ #5)

| 용도 | 환율 |
|---|---|
| 이력·추세·감사·판가결정·콕핏 밴드 | **스냅샷 동결**(관측일 환율) — 시세를 "역사적 사실"로 고정 |
| 현재가 비교("지금 기준") | **원본에서 최신환율 재환산** |

**절대 규칙**: 재환산은 **항상 원통화 원금액에서**. 동결 KRW에 또 환율 곱하는 "이중 환산" 금지(오염 1순위).

## 3. FX 인프라 (DC-RES 사실확인 반영)

- `fx_rates`를 **다통화**로 확장(현재 usd_krw만). 소스 AP01은 이미 전통화 반환 → 파서만 확장.
- **JPY(100)·IDR(100) ÷100 정규화** 필수(미처리=100배 사고).
- `deal_bas_r` **콤마 문자열**("1,342.5") 파싱.
- 휴일·비영업일 결측 → **직전 영업일 폴백** + 실제 적용 `rate_date` 기록.
- 교차환율(JPY→USD 등) = **같은 날짜·같은 소스** 두 고시로만 (A/KRW ÷ B/KRW).

## 4. 추출 — "택일" 폐기, **양패스 병합+상호검증** (DC-RES + DC-ANA #1·#5)

현재: `useTranscription`이면 classify를 통째 폐기(비교표에서 모델 유실). classify는 specContext 오염(모델 둔갑)이라 못 씀 → 딜레마.

**목표(Self-Consistency·CoVe 정석):**
1. **전사 패스**(verbatim, 해석 금지) — 원문·원가 보존.
2. **구조화 패스**(모델 인식) — **specContext 주입 제거**한 "원문 모델명 그대로" 버전으로 별도 수행(오염 제거).
3. **결정론 수치 인벤토리 대조** — 원문 숫자토큰 집합 vs 추출 집합 → 누락/환각률(LLM에 "다 뽑았니" 금지). `reconcile.ts` 재사용.
4. 2D 표(플랜=열/속성=행) **양방향 해석 후 스키마적합 높은 쪽** 채택, 애매하면 보류.
5. **전 필드 근거 span 강제**(provenance). 근거 없는 값 = 자동반려.
6. **산술(환율·÷장수·÷시간)은 LLM 절대 금지** — 결정론 코드(normalize-money SSOT)만.

→ 소프트뱅크 케이스: 전사(원가 ¥2.5M/월 보존) + 구조화(H100 모델 인식) 병합 → "H100 플랜, ¥2,500,000/월, 8장, 스토리지포함, managed_bundle" 로 **모델·원가 다 살고**, per-GPU 환산은 결정론 + 번들 플래그.

## 5. 비교가능성 게이트 (DC-BIZ #3·#6, DC-RES 번들 3단)

- **세그먼트 격리**: 콕핏 competitor 밴드 = 기본 `raw_gpu`만. `managed_bundle`은 별도 "참고 시세" 트랙(비교 위장 금지).
- **번들 3단**: ① 구성분리 가능(스토리지/네트워크가 명시)→감산 후 순수가+감산내역 provenance ② 분리 불가→`bundle_inclusive` 플래그+포함목록, 랭킹 별도트랙 ③ 비교불가(최소약정·커스텀견적)→`comparable=false`, 랭킹 제외·데이터 보존.
- **예약/온디맨드/스팟** 플래그 분리(한 풀 혼합 금지).
- **신선도**: `observed_at` 필수 + stale(예 90일↑) 배지 + 밴드 제외. 기존 `expiry.ts` 축과 단일화(신규 개념 금지).

## 6. 자동확정 임계 (DC-BIZ #6 — "오류 최소"의 핵심)

시세는 틀린 1건이 판가를 오염 → **확실한 것만 자동, 나머지 검수큐**:
- 자동확정 = **통화·기간단위·GPU장수·모델** 4필드가 **명시적 근거(provenance)와 함께** 추출됐을 때만.
- **번들 의심·미지모델·통화추정(기호없이 숫자만)·장수추정 = 사람 확정 강제.**
- 기존 `confidence-gate`·검수 CRUD·USAI 자기일관성 재사용(신규 루프 금지). 커버리지 손실은 "보류큐 잔존"으로 흡수(무손실과 양립).

## 7. 게이트 정교화 (DC-ANA #4)

- `GPU_MODEL_SIGNAL` 단독 `\bgpu\b` → 요금라벨(GPU利用料金) 오통과. **"GPU" 단독으론 통과 금지, 모델/tier 토큰 동시 요구** 또는 competitor 경로 분리. **단, 공유 SSOT라 골든셋 회귀검증 필수.**

## 8. 회귀 안전판 (DC-ANA #5 — 없으면 재설계 금지)

현 골든셋(good3/bad5, 1차원)엔 이번 4대 사고(**세로피벗표·classify폐기·비KRW통화·\bgpu\b오통과**) 사례 0건. → **재설계 전, 이 4유형 각 2~3건 fixture를 golden-corpus에 신규 추가**해 회귀 게이트로 세운 뒤 착수.

## 9. 단계 (승인 후, 오류기여도 역순 = 큰 것부터)

- **P0 안전판**: 골든 코퍼스에 4대 사고 fixture 추가(회귀 게이트).
- **P1 관측원본 스키마**: `market_prices`(+supply_quotes)에 관측원본 필드군 + 환율스냅샷 + 품질메타 추가(additive, 기존 무손상). 저장 권위를 파생→**원본**으로 전환.
- **P2 FX 다통화**: fx_rates 확장 + AP01 전통화 파서(JPY100·콤마·휴일폴백·교차환율).
- **P3 정규화 SSOT**: normalize-money에 (관측원본→$/GPU-hr KRW, 스냅샷/최신 이원화) 결정론 함수. 표시=최신+적용환율 명시, 판가=스냅샷.
- **P4 비교가능성**: 세그먼트·번들 3단·신선도·예약형 플래그 + 콕핏 밴드 격리.
- **P5 추출 병합**: 전사+구조화(specContext 제거) 병합·수치 인벤토리 대조·근거 span 강제·자동확정 임계.
- (게이트 P7: `\bgpu\b` 정교화는 골든셋 통과 조건부.)
