# FAST PATH Summary — 실제 렌더 경로 우선 수정 정책

작업: UI/표시 수정 시 "현재 활성 렌더 경로"를 먼저 확인·수정하도록 강제하는 정책 조항을 CLAUDE.md·AGENTS.md에 추가.

대상:
- `CLAUDE.md` — "재사용·단일구현 정책" 다음에 "## 실제 렌더 경로 우선 수정 정책" 신설(4개 규칙)
- `AGENTS.md` — 동일 조항 동기화(Codex 정책 파일)

이유: v0.7.173에서 GPU 장수 표기를 구 탭뷰 `PriceTableTab`에 적용했으나, 실제 기본 렌더는 `unified` feature flag(`DEFAULT_ON=true`) ON 상태의 `UnifiedTableConnected`라 화면에 반영되지 않음 → v0.7.174 재수정. 구/신 컴포넌트 공존 시 죽은 경로를 고치는 재발을 차단한다.

정책 핵심(4조):
1. 수정 전 분기 코드를 직접 읽어 실제 렌더 경로 확정(추정 금지)
2. 활성(기본 ON) 경로 먼저 → 공존 경로(롤백 구뷰 포함) 모두 동일 SSOT 적용
3. 검증은 실제 렌더 경로에서(tsc/design/unit 통과만으로 완료 금지)
4. 표시 변환도 SSOT(`lib/` 공용 함수, 뷰별 인라인 복붙 금지)
