# 00 · 요구사항 — 주간보고 AI 자동선작성(push) 전환

> 상태: **기획(분석)만**. 본 폴더 전체는 구현 착수 전 설계 문서다. 코드/마이그레이션 실행 없음.
> 버전 슬러그: v0.7.281 (실제 버전 부여는 구현 착수 시)

## 1. 배경 / 문제

현행 주간보고는 **pull 방식**이다. 사용자가 그 주의 일일업무를 `DailyTaskSelector`에서 체크박스로 골라 → `generate-from-tasks` → 폼에 병합한다. 즉 "백지에서 시작해, 사용자가 능동적으로 끌어와야" 내용이 생긴다.

→ 결과: **미작성·누락·지연이 구조적으로 발생**한다. 바쁘면 안 끌어오고, 안 쓰면 빈 보고가 된다.

## 2. 목표 (한 줄)

주간보고를 "백지 작성"에서 **"AI가 채워둔 초안을 다듬기"** 로 바꿔, 미작성·누락을 구조적으로 제거한다.

## 3. 핵심 원칙 (확정)

- **P0 — 자동이 디폴트:** 주간보고 화면을 열면 AI가 그 주 일일업무+캘린더를 분석한 **초안이 이미 채워진 상태**로 시작한다. "고르는(pull)" 단계는 기본 동선에서 사라진다.
  - 신규 사용자/신규 주간 = 자동 경험.
  - **폴백(fallback)의 의미는 신규 UX가 아니라:** ① 기존 저장 주간보고/취합본의 데이터 호환 보존, ② AI 생성 실패·그 주 데이터 0건 시 빈 자동영역+수동 에디터로 graceful degrade. (사용자가 자동/수동을 "고르는" 게 아님)
- **P1 — 빈칸/판단대기 0:** 모든 자동항목은 이미 어딘가(카테고리×성과/계획/이슈)에 배치돼 있다. 별도 "미분류 버킷"을 만들지 않는다. 신뢰도 낮은 항목만 **시각 마커('확인요')** 로 표시해 사용자가 이동/삭제로 교정.
- **P2 — 조정은 가볍게:** 자동작성 영역은 **체크박스(포함/제외)·삭제(X)** 로 가감. 사용자 자유작성은 **에디터 영역**으로 공존(별도).
- **P3 — 미조정 자동확정:** 사용자가 조정하지 않고 기한이 지나면 **AI 초안 그대로 확정 저장**(actor=system 로그). 단 기한 임박 시 **사전 알림** 선행(품질 안전장치).
- **P4 — 출처 기반 취합:** 부서 취합은 자동생성분·수동작성분을 **origin(auto/manual) 태그만 달고 동등 취합**. 기존 취합 엔진B 재사용.

## 4. 분류 규칙 (확정)

2층 분류:
1. **카테고리(구분)** — 기본 분류축. AI가 일일업무+캘린더를 카테고리로 묶는다.
   - 카테고리 어휘 참조 계층: **개인 지난주 카테고리 → (확장) 부서 카테고리**. (기존 `prevWeekCategories`/`categoriesFromBodies` 자산 재사용)
2. 각 카테고리 내 **성과 / 계획 / 이슈** 3필드로 배치. (현행 `weekly_reports` 3필드 그대로 유지)

시점 + 의미 2축으로 성과/계획/이슈 판정:
- **성과** ← 일일업무(완료 기록) + 이번 주(보고주 월~일) 종료된 캘린더 일정
- **계획** ← **다음 주(+1주 월~일)** 미래 캘린더 일정 + 이번 주 미완료/이월 항목
- **이슈** ← 미해결(`is_resolved=false`)·블로커·미래지향/리스크 표현
- AI 의미분석이 시점 신호를 보정(예: 일일업무 본문의 "~예정"은 계획으로).
- **신뢰도 낮은 항목 = 강제 배치 + '확인요' 마커.**

## 5. 기능 요구사항 (FR)

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| FR-1 | 주간보고 화면 진입 시, 해당 주 저장 초안이 없으면 일일업무+캘린더를 AI 분석해 자동초안 생성·표시 | P0 |
| FR-2 | 자동초안 항목은 카테고리×(성과/계획/이슈)에 배치. 각 항목 origin=auto, 신뢰도 마커 | P0 |
| FR-3 | 자동항목을 체크박스(포함/제외)·X(삭제)로 조정. 사용자 자유작성은 에디터(origin=manual) | P0 |
| FR-4 | 다음 주 계획은 +1주 캘린더 미래 일정에서 도출 | P0 |
| FR-5 | 생성된 초안은 저장(캐시). 재진입 시 AI 재호출 없이 저장본 로드 | P0 |
| FR-6 | 미조정+기한경과 시 사전알림 후 AI초안 자동확정(actor=system, 불변로그 기록) | P1 |
| FR-7 | 부서 취합 시 auto/manual 출처 태그 보존하며 동등 취합 | P1 |
| FR-8 | 기존 pull 흐름(DailyTaskSelector) 및 기존 저장 데이터는 회귀 없이 동작(하위호환) | P0 |

## 6. 비기능 요구사항 (NFR)

- **성능:** 화면열때 생성 = 첫 진입 로딩 표시 필수(스켈레톤). 저장본 재로드는 즉시.
- **비용:** Gemini 호출은 "주당 1회 생성"이 기본(저장 캐시로 재호출 차단). 토큰은 `token-logger`로 기록.
- **정합성:** datetime는 `lib/datetime/kst.ts` SSOT. 주간 범위는 `kstRangeToUtc`. 표시 HTML은 `RichText`/`html-to-plain` SSOT.
- **보안:** RLS — 본인 주간보고만 생성/수정. 자동확정 system actor도 RLS·불변로그 정책 준수.
- **공정성(중요):** 자동확정된 보고가 기존 timeliness "정시 제출"로 집계될 때의 인사평가 영향 → 04-completion-criteria의 공정성 항목 참조.

## 7. 제외 범위 (이번 기획 + 권고 Phase 분리)

- 코드 구현·마이그레이션 실행·UI 작성 (이번 산출물 전체 제외)
- 음성/첨부 기반 일일업무 → 주간 (별도)
- 캘린더 양방향 동기화(주간보고→캘린더 일정 생성)는 P2 이후
