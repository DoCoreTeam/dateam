# 요구사항 — 신규 모노크롬 테마 + 테마 프리뷰 버그 + 개인별 테마

생성: 2026-06-15 / 버전: v0.7.120

## 배경
이미 `[data-theme]` 토큰 기반 테마 시스템(nb 기본 / classic) 보유. 여기에 확장.

## 요구사항
1. **신규 테마 추가**: `3rd design.html`(모노크롬/스위스 — 흑백·직각·hairline·레드 액센트) 를 신규 테마 `mono`로 추가. 기존 테마 시스템(globals.css `[data-theme]` + `lib/themes.ts`)에 흡수.
2. **테마 프리뷰 버그 수정**: 어드민 설정의 테마 선택 카드에서, 전역 테마가 classic일 때 nb 카드 프리뷰가 classic처럼 잘못 렌더됨(Image #2). 선택 전에도 각 카드가 **그 테마의 실제 비주얼을 보여줘** 유추 가능해야 함(Image #3는 정상 케이스).
3. **개인별 테마 선택**: 좌하단 사용자명 클릭 메뉴에 "테마변경" 항목 신설 → **오른쪽 서브메뉴**로 개인 테마 선택.
4. **권한 모델**: 어드민(`/admin/settings`)은 **전역 디폴트 테마** 관리. 일반 사용자는 **본인이 선택한 테마**로 화면 표시(미선택 시 디폴트 추종).
5. **영속**: 개인 테마는 DB(`profiles.theme_preference`)에 영속 — 기기/세션 바뀌어도 유지. SSR 주입(FOUC 없음).
6. **검증**: 기획→구현→**Playwright 브라우저 실검증**까지 완료.

## 제약
- 앱은 한국어 UI. 모노스페이스 글꼴 전역 교체 금지(한글 글리프 없음 → 깨짐). 폰트는 Pretendard 유지, 테마 정체성은 색/형태로 표현. (DECISION-20260615-mono-font 참조)
- 디자인 토큰 SSOT 준수, 하드코딩 금지(`design:check` 통과).
- RLS 필수. 개인 테마 변경은 본인만.
- 10px 미만 폰트 금지 등 기존 디자인 정책 유지.
