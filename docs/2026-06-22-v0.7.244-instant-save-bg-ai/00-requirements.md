# 00 요구사항 — 일일업무 즉시저장 + 백그라운드 AI분해

## 배경
현재 일일업무 "저장" 버튼(`handleAiSave`, page.tsx:261)은 AI 분석을 끝까지 await한 뒤에야
DB 저장한다. 결과: ① 버튼이 AI 생성시간만큼 묶여 느림, ② 원문이 독립 저장되지 않아
AI 실패 시 입력이 통째로 사라짐.

## 사용자 요구 (확정)
- R1. "저장" 클릭 → 입력칸 즉시 비고 **원문 1건 즉시 표시**, 저장 UI 응답 **< 300ms**.
- R2. AI 분해는 **백그라운드**로 같은 그룹에 분해항목을 **별도 추가**(원문은 헤더로 유지).
- R3. **원문 보존** — AI 실패/지연에도 원문 행이 사라지지 않음.
- R4. 브라우저(Playwright) 실측 + 성능 측정으로 R1~R3 검증.

## 비범위
- AI 분류 프롬프트/모델 변경 없음. 음성·OCR 무관.
- 마이그레이션 없음(기존 daily_logs 컬럼 재사용).

## 수용 기준
- 저장 클릭→원문 표시까지 체감 즉시(낙관적 업데이트, 측정 < 300ms).
- 분해항목은 수 초 뒤 같은 그룹 헤더 아래 등장.
- AI 강제 실패 시에도 원문 행 유지 + "분석 결과 없음/재분석" 안내.
