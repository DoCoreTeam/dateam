# 00 · 요구사항 (목록 심층분석 v2)

> 상위 설계=`00-plan.md`, 결정 잠금=`DECISIONS.md`. 이 문서=요구사항 정본.

## 1. 배경 / 문제
현 `/ai-chat/analyze`(v1)는 (a) 원문 앞 8,000자만 슬라이스해 뒤 항목이 무맥락, (b) 취합이 30,000자 slice 요약형이라 생략·유실, (c) 진행상태가 클라 4상태 배지라 탭 닫으면 중단·`running` 고아, (d) 사용자 명령이 세션에 미영속. → "완성형·무손실·실시간·명령주도"를 만족 못함.

## 2. 사용자 스토리
1. 사용자가 **다양한 형태 문서**(기획서/md/이미지/엑셀/ppt/html/pdf)를 투입한다.
2. 시스템이 **기획서 맥락 중심**으로 항목을 발췌한다(단순 항목 나열 아님).
3. 사용자가 **자유 명령**(예: "각 항목을 리스크·대안·실행계획 관점으로 확산 전개")을 준다.
4. 각 항목이 **N개 병렬 심층 분석**으로 전개된다.
5. 완료되면 **생략 없이 하나의 완성형 보고서**로 취합된다(흐름상 이상한 것만 교정).
6. 진행 중 **실시간 상태**(무슨 일을 하는지)를 본다 — 상태값 하드코딩 없이.
7. 브라우저를 **이탈해도 작업이 계속**되고, 다시 오면 이어 본다.
8. 사용자가 **임의로 취소/일시정지**할 수 있다.

## 3. 기능 요구 (FR)
- FR1 문서 투입: 기존 document-extract 재사용(이미지=OCR 폴백).
- FR2 항목 추출: `parseListItems` 결정론 + AI 보정 병합(유실0 계약).
- FR3 맥락 팩: 항목마다 원문 앵커 발췌 + AI 의도 주석(idx 참조 JSON only).
- FR4 명령 영속: 세션 `command` 컬럼, 3단계 프롬프트 전체 지배.
- FR5 병렬 심층: 서버 워커풀 K개(META 설정), 항목별 상태 전이 DB 즉시 기록.
- FR6 무손실 취합: map(항목 다이제스트 `[#idx]`) → reduce → **커버리지 게이트**(누락 시 보수 패스→부록 결정론 append) → 완성형 보고서.
- FR7 실시간: `count(status)` 파생 + 서버 SSE + 폴링 폴백. 열람 항목 text delta.
- FR8 백그라운드: 서버+DB가 실행 주체, 크론 드레인 워커가 미완 이어받음.
- FR9 임의 중단: 세션 `control`(running/paused/cancelled), 워커가 항목 경계마다 확인 + in-flight abort.
- FR10 재개: DB 복원 + 미완 이어감(멱등 claim). 부분 완료 즉시 활용.

## 4. 신규 엔티티 → Feature Defaults 자동 적용 (완료기준서에 전개)
`ai_analysis_sessions`·`ai_analysis_items` 확장(신규 아님, 마이그 157 확장). 세션은 컬렉션 엔티티 → **CRUD·List·검색/정렬/필터·페이지네이션·RLS**를 `04-completion-criteria.md`에 체크박스 전개.

## 5. 비기능 요구 (NFR)
- 유실0/생략0(코드 게이트 최종). 진행상태 파생만(하드코딩 금지). npm 의존성 0.
- 권한 admin. 비용 상한(항목수×응답). Vercel maxDuration 내 드레인.
- 프롬프트 인젝션 방지: 명령/항목/원문 분리 주입.
