## 변경 이력 (v0.1.1 → v0.1.2)

본부장 의사결정으로 다음을 반영했습니다.

1. **프로젝트 정의 확장 (6 → 10개):** 사이트 프로젝트 6개에 더해 **영업·공급 4개 프로젝트도 본부 공식 프로젝트로 정의**
   - gcube 수요 영업 (채널 + 직접)
   - gcube 공급자 확보
   - 시티큐브 영업
   - 공공기관 프로젝트 영업
2. **이도현 실장 R&R 강화:** 가격정책·공급자 풀 관리·공급자별 공급단가 관리 책임 명시 (시간 배분 30% 비중)
3. **공급자 확보 라인 명확화:** 본부장이 외근으로 발굴 → 이도현 백오피스에서 풀 등록·계약·단가 처리 (직선 책임)
4. **김태형 실장 책임 추가:** 사이트 PM 6개 + **gcube 수요 영업의 채널 발굴 동참** (사이트 발주처와 인접한 지자체·기관 네트워크 활용)
5. **박현숙·오유정 = 페어 (한팀처럼 운영):** 백엔드/프론트엔드 메인 구분은 유지하되 **풀스택이 기본**
6. **AI 코딩 적극 활용:** 박현숙·오유정의 핵심 행동 양식으로 명시 (Claude/Cursor/Copilot 등 적극 활용 → 1주 출시 사이클의 생산성 핵심)

## 1. 본부 비전 (Vision)

> **"gcube와 시티큐브를 두 축으로, 글로벌 AI 인프라와 스마트시티 시민서비스 시장을 동시에 락인(Lock-in)하는 프로덕트 중심 본부."**

데이터얼라이언스는 2016년 스마트시티/IoT 데이터 플랫폼에서 출발해, 2026년 CES Innovation Award(AI 부문) 수상에 이르기까지 GPU 공유경제 플랫폼 **gcube**를 중심으로 AI 인프라 기업으로 확장했다. 본부는 회사가 보유한 두 핵심 자산 — gcube(글로벌 GPU 네트워크)와 시티큐브(블록체인 기반 인증·통합 포인트) — 를 결합해 다음을 달성한다.

1. 시티큐브를 **블록체인 기반 시민 인증·포인트의 표준 SaaS**로 완성한다.
2. gcube 위에 **개발자/비개발자가 1주 단위로 사용할 수 있는 오픈소스 프로덕트**를 지속 출시하여 gcube 리소스 락인 구조를 만든다.
3. 공공/민간 채널 영업을 통해 **gcube 수요와 공급을 동시에 확보**한다.

## 2. 본부 미션 (Mission) — 9개 책임 영역

본부는 다음 9개 영역의 실행 책임을 진다. 우선순위는 순서대로다.

1. **시티큐브 완성/디벨롭** — 블록체인 기반 인증 + 통합 포인트 플랫폼의 코어 완성
2. **gcube 오픈소스 프로덕트 1주 단위 출시** — 개발자/비개발자용 도구를 gcube 리소스에 락인
3. **본부원 브레인스토밍 기반 신규/고도화 프로덕트 발굴**
4. **gcube 공급자 확보·관리·가격정책 수립** — 공급자 커뮤니케이션 상시 운영
5. **다양한 채널을 통한 gcube 수요 기업 확보·확대** (채널 영업 + 직접 영업)
6. **공공사업/신규 프로젝트 기획 → 방향 수립 → 제안 완료**
7. **개발 우선순위 엄수:** 시티큐브 완성 → gcube 오픈소스 프로덕트 → 고도화
8. **gcube와 연관성이 깊은, 블록체인 기반 신규 프로젝트 발굴**
9. **다양한 채널의 요구사항 분석·발굴**

## 3. 업무 지침 (Working Principles) — 9개 원칙

본부원은 다음 9개 원칙에 따라 일한다.

**3.1. 우선순위 원칙 (Priority First)**
시티큐브 완성이 모든 개발 작업의 최우선이다. gcube 오픈소스 프로덕트는 그다음, 기존 프로덕트의 고도화는 마지막이다. 충돌 시 즉시 본부장 에스컬레이션.

**3.2. 1주 단위 출시 원칙 (Weekly Ship)**
gcube 오픈소스 프로덕트는 매주 월요일에 다음 1주 출시 분량을 확정하고, 금요일에 출시한다. **금 12:00 = Code Freeze, 14:00 = GitHub Release** (오유정 Release Manager 책임).

**3.3. 글로벌·다국어 우선 (Global by Default)**
모든 신규 프로덕트는 처음부터 한국어/영어 2개 언어 이상을 기본으로 설계. SEO/AEO/GEO 메타를 첫 빌드부터.

**3.4. 락인(Lock-in) 설계 원칙**
모든 오픈소스 프로덕트는 gcube/시티큐브 락인 포인트 1개 이상을 가진다. 락인 없는 프로덕트는 만들지 않는다.

**3.5. 버전 관리 원칙**
모든 산출물(코드·문서·제안서·디자인)은 `vX.Y.Z` 형식. 커밋 메시지에 동일 버전. 롤백 가능.

**3.6. 명확성 원칙 (No Ambiguity)**
모든 지시에 (1) 무엇을 (2) 왜 (3) 언제까지 (4) 산출물 형태 4가지 포함.

**3.7. 분석 → 발굴 원칙**
영업·공공·파트너·외부 사용자 피드백 채널의 모든 요구사항을 패턴으로 분류·기록 → 신규 프로덕트 시드.

**3.8. 다중 프로젝트 추적 원칙**
본부는 **10개 프로젝트(사이트 6 + 영업·공급 4)** 를 동시 운영. 매주 월 09시 위클리에서 모두 1줄 상태 갱신.

**3.9. AI 코딩 + 페어 활용 원칙 (NEW)**
박현숙·오유정 페어는 **AI 코딩 도구(Claude/Cursor/Copilot 등)를 적극 활용**해 1주 출시 사이클을 가능하게 한다. 백엔드/프론트엔드 메인 구분은 유지하되 **풀스택이 기본**으로, 필요 시 서로의 영역을 넘나든다. AI가 생성한 코드는 박현숙(개발 PM) 검수를 거친다.

## 4. 본부원 R&R (Roles & Responsibilities)

본부 R&R은 **외부/내부 분리선**과 **사이트/개발 PM 분리선**과 **백/프 페어**라는 세 축을 기준으로 설계되었다.

### 4.1. 김도현 본부장 — 본부 총괄 + 외근 다수 + 영업·공급 4개 PM
- **본부 차원 책임:** 본부 OKR, 본부원 1:1, 임원 주간보고(월 11시), 신규 사업 의사결정, 채용
- **PM 직접 담당 (5개 프로젝트, Primary):**
  - 충남 AI 대전환 (사이트 프로젝트, 김태형과 공동)
  - **gcube 수요 영업** (채널 영업 + 직접 영업, 김태형 채널 동참)
  - **gcube 공급자 확보** (외근으로 공급자 발굴 → 이도현이 백오피스 처리)
  - **시티큐브 영업** (시티큐브 SaaS 신규 유료 고객)
  - **공공기관 프로젝트 영업** (신규 공공사업 제안)
- **외부 미팅 책임 (외근 다수):**
  - 외부 의사결정자 미팅 (대기업·공공 C-레벨)
  - GPU 공급자 외부 미팅 (해외·국내) → **물어온 결과는 24시간 내 이도현에게 전달**
  - 공공기관 미팅 (신규 사업 제안 영업)
  - IR / 언론 / 글로벌 파트너십
- **공동 오너십:** 모든 프로젝트의 마일스톤·리스크 최종 책임
- **시간 배분:** 외부 미팅(영업·공급·공공) 50% / 본부 운영 30% / 충남 PM·신규 사업 20%

### 4.2. 이도현 실장 — 내부 백오피스 (기획·문서·정책 + 가격·공급 풀 관리)
- **도메인 오너십:**
  - **가격정책 v1.X 수립·갱신** (gcube 공급/수요 단가, 시티큐브 SaaS 요금제)
  - **공급자 풀 관리** (공급자 데이터베이스, 상태, 계약 단계, Tier 분류)
  - **공급자별 공급단가 관리** (단가표 v1.X, 인센티브, 정산)
  - 본부 운영서·운영루틴 매뉴얼 갱신
  - 모든 프로젝트의 **제안서·기획서·정책 문서** 작성·관리
  - 시민증·포인트 등의 **정책** (서비스 약관, 데이터 거버넌스, 개인정보 처리방침)
- **PM 직접 담당:** 없음 — 사이트 PM은 김태형, 개발 PM은 박현숙, 영업·공급 PM은 본부장
- **본부장과의 직선 책임 (공급자 확보 처리 라인):**
  - 본부장 외근 → 신규 공급자 발굴 → **이도현 백오피스 처리:** 풀 등록·계약서 작성·단가 협상 자료·정산 관리
  - 24시간 내 본부장 외근 결과 정리 → 공급자 풀에 반영
- **외부 미팅 책임:** 본부장·김태형 외부 미팅 **자료 준비 (전·후)**. 본인 단독 외부 미팅은 거의 없음
- **백오피스 산출물 (월간 단위):**
  - 임원 주간보고 자료 패키지 (매주 월 11시 보고 1시간 전까지)
  - 신규 제안서 5건/분기 리드 작성 (본부장 검토)
  - 본부 운영서/루틴 매뉴얼 격주 갱신
  - 공급자/수요자 단가표 v1.X 관리 (본부장 외근 결과 즉시 반영)
- **시간 배분:** 가격·공급 풀 관리 30% · 기획·문서 30% · 정책·계약 20% · 제안서 20%

### 4.3. 김태형 실장 — 사이트 PM 총괄 (6개) + gcube 수요 영업 채널 동참
- **PM 담당 (모두 Primary):**
  - 평택 스마트시티 통합 시민앱
  - 광주 스마트시티 데이터허브
  - 용인 시티포인트 → 시티큐브 SaaS 전환
  - 칠곡 공공자전거
  - 김해 시민증 + 전자지갑
  - 충남 AI 대전환 (본부장과 공동)
- **사이트 PM의 책임 영역 (= 고객 측):**
  - 발주처(지자체·기관) 단일 창구
  - 사이트별 일정·요구사항·진척 관리
  - 발주처 외부 미팅 (각 지자체 정기 점검)
  - 매주 월요일 위클리에 6개 1줄 상태 갱신
- **gcube 수요 영업 채널 동참:**
  - 사이트 발주처와 인접한 지자체·기관 네트워크를 gcube 수요로 발굴
  - 화·목 외부 미팅 슬롯에서 발주처 미팅과 함께 진행 가능 (본부장이 직접 영업으로 못 가는 빈자리 보완)
- **박현숙(개발 PM)과의 분리선:** 김태형 = 고객/일정/요구사항, 박현숙 = 구현/아키텍처/기술 부채. 화·목 09:30 정기 30분 동기화
- **외부 미팅 슬롯:** 화·목 14:00–17:00 발주처 + gcube 수요 채널 미팅
- **시간 배분:** 사이트 PM 60% · 발주처+gcube 수요 채널 외부 미팅 25% · 데이터/AI 컨설팅 15%

### 4.4. 박현숙 선임 — 개발 PM + 시티큐브 코어 백엔드 (페어 메인: 백엔드)
- **개발 PM 책임 (= 구현 측):**
  - 개발팀(박현숙·오유정 페어 + 향후 채용)의 백로그·기술 결정·코드 리뷰·멘토링
  - 시티큐브 SaaS / gcube 오픈소스 프로덕트의 아키텍처 결정
  - 사이트 PM(김태형)의 일정·요구사항을 개발 일정으로 번역, 무리한 일정은 즉시 협상
  - **AI 코딩 도구의 본부 표준 결정** (어떤 모델/IDE/워크플로를 쓸지) 및 본부원 가이드
  - **AI 생성 코드 검수** — 페어가 AI로 만든 코드를 박현숙이 최종 검수
- **개발 도메인 (메인 빌더, 백엔드 메인 + 풀스택):**
  - 시티큐브 코어 엔진(블록체인 인증 + 통합 포인트), 결제·정산
  - 김해 시민증 인증 모듈
  - 평택 시민앱 포인트 연동 백엔드
  - 풀스택이 기본이라 프론트엔드도 필요 시 직접 작업
- **gcube 오픈소스 기여:** 인증·결제 관련 모듈 (메인 빌더는 오유정, 박현숙은 백엔드 기여)
- **오유정과의 페어 운영:** 매일 09:30 짧은 데일리 sync (15분), 수 15:30–17:00 정기 페어 코딩 (60분)
- **시간 배분:** 시티큐브 SaaS 코어 50% · 개발 PM(백로그·리뷰·멘토링·AI 코딩 검수) 30% · 시민증 백엔드 20%

### 4.5. 오유정 사원 — 프론트엔드 메인 (페어) + Release Manager + 커뮤니티 오너
- **개발 도메인 (메인 빌더, 프론트엔드 메인 + 풀스택):**
  - gcube 위에서 동작하는 비개발자/개발자용 오픈소스 프로덕트 (1주 단위 출시 메인 빌더)
  - 사이트 프로젝트 프론트엔드 (용인 SaaS, 칠곡 모바일, 광주 데이터허브 UI)

## 8. 본부장이 본부원에게 요청하는 것

1. **모든 산출물에 버전을 붙여라.** vX.Y.Z + 동일 메시지의 커밋. 롤백 가능.
2. **글로벌·다국어를 처음부터.** ko/en은 기본.
3. **락인이 없는 프로덕트는 만들지 마라.** gcube 또는 시티큐브 락인 포인트 1개 이상.
4. **요구사항을 그냥 처리하지 마라.** 패턴화 → 신규 프로덕트 시드로.
5. **잊혀진 프로젝트를 만들지 마라.** 10개 모두 매주 1줄 상태 갱신.
6. **막히면 30분 안에 손을 들어라.** 우선순위 충돌 즉시 본부장에게.
7. **본부장 외근 중에도 본부는 멈추지 않는다.** 이도현(백오피스)·김태형(사이트 PM)·박현숙·오유정(개발 페어)이 자율 운영.
8. **AI 코딩을 적극 쓰되 검수를 거쳐라.** 박현숙(개발 PM)이 AI 생성 코드의 본부 표준을 잡는다.
