## 1. 본부 비전 (Vision)

> **"gcube와 시티큐브를 두 축으로, 글로벌 AI 인프라와 스마트시티 시민서비스 시장을 동시에 락인(Lock-in)하는 프로덕트 중심 본부."**

데이터얼라이언스는 2016년 스마트시티/IoT 데이터 플랫폼에서 출발해, 2026년 CES Innovation Award(AI 부문) 수상에 이르기까지 GPU 공유경제 플랫폼 **gcube**를 중심으로 AI 인프라 기업으로 확장했다. 본부는 회사가 보유한 두 핵심 자산 — gcube(글로벌 GPU 네트워크)와 시티큐브(블록체인 기반 인증·통합 포인트) — 를 결합해 다음을 달성한다.

1. 시티큐브를 **블록체인 기반 시민 인증·포인트의 표준 SaaS**로 완성한다.
2. gcube 위에 **개발자/비개발자가 1주 단위로 사용할 수 있는 오픈소스 프로덕트**를 지속 출시하여 gcube 리소스 락인 구조를 만든다.
3. 공공/민간 채널 영업을 통해 **gcube 수요와 공급을 동시에 확보**한다.

## 2. 본부 미션 (Mission) — 9개 책임 영역

본부는 다음 9개 영역의 실행 책임을 진다. 우선순위는 순서대로다.

1. **시티큐브 완성/디벨롭** — 블록체인 기반 인증 + 통합 포인트 플랫폼의 코어 완성
2. **gcube 오픈소스 프로덕트 1주 단위 출시** — 개발자/비개발자용 도구를 gcube 리소스에 락인
3. **본부원 브레인스토밍 기반 신규/고도화 프로덕트 발굴**
4. **gcube 공급자 확보·관리·가격정책 수립** — 공급자 커뮤니케이션 상시 운영
5. **다양한 채널을 통한 gcube 수요 기업 확보·확대**
6. **공공사업/신규 프로젝트 기획 → 방향 수립 → 제안 완료**
7. **개발 우선순위 엄수:** 시티큐브 완성 → gcube 오픈소스 프로덕트 → 고도화
8. **gcube와 연관성이 깊은, 블록체인 기반 신규 프로젝트 발굴**
9. **다양한 채널의 요구사항 분석·발굴**

## 3. 업무 지침 (Working Principles) — 8개 원칙

본부원은 다음 8개 원칙에 따라 일한다.

**3.1. 우선순위 원칙 (Priority First)**
시티큐브 완성이 모든 개발 작업의 최우선이다. gcube 오픈소스 프로덕트는 그다음, 기존 프로덕트의 고도화는 마지막이다. 개발자가 우선순위 충돌을 느끼면 본부장에게 즉시 에스컬레이션한다.

**3.2. 1주 단위 출시 원칙 (Weekly Ship)**
gcube 오픈소스 프로덕트는 매주 월요일에 다음 1주 출시 분량을 확정하고, 금요일에 출시(또는 베타 공개)한다. **금요일 12:00 = Code Freeze, 14:00 = GitHub Release** (오유정 Release Manager 책임).

**3.3. 글로벌·다국어 우선 (Global by Default)**
모든 신규 프로덕트는 처음부터 한국어/영어 2개 언어 이상을 기본으로 설계한다. SEO/AEO/GEO를 모두 고려한 메타 구조를 첫 빌드부터 포함한다.

**3.4. 락인(Lock-in) 설계 원칙**
모든 오픈소스 프로덕트는 "왜 gcube에서 돌려야 하는가"에 대한 명확한 락인 포인트(GPU 비용, 데이터 게이트웨이, 인증 레이어 등) 1개 이상을 가져야 한다. 락인이 없는 프로덕트는 만들지 않는다.

**3.5. 버전 관리 원칙**
모든 산출물(코드, 문서, 제안서, 디자인 파일)은 `vX.Y.Z` 형식의 시맨틱 버전을 부여한다. 커밋 메시지에도 동일한 버전을 명시한다. 롤백 가능성을 항상 확보한다.

**3.6. 명확성 원칙 (No Ambiguity)**
모든 지시·요청은 (1) 무엇을, (2) 왜, (3) 언제까지, (4) 산출물 형태 — 4가지를 반드시 포함한다. 외부 연계가 필요하면 링크와 행동 지시를 함께 제공한다.

**3.7. 분석 → 발굴 원칙**
영업·공공·파트너 채널, 그리고 **외부 사용자 피드백(Discord/GitHub Issue)** 으로 들어오는 모든 요구사항은 단순히 처리하지 않고, **수요 패턴**으로 분류·기록하여 신규 프로덕트 아이데이션의 입력으로 사용한다. (오유정 커뮤니티 오너가 분기당 패턴 리포트 작성)

**3.8. 다중 프로젝트 추적 원칙**
본부는 동시에 6개 이상의 프로젝트를 운영한다. 매주 월요일 9시 본부 위클리에서 **모든 프로젝트의 상태를 1줄로** 공유하여 어떤 프로젝트도 잊혀지지 않도록 한다. (김태형 사이트 PM 총괄이 6개 모두 1줄 상태 갱신 책임)

## 4. 본부원 R&R (Roles & Responsibilities)

본부 R&R은 **외부/내부 분리선**과 **사이트/개발 PM 분리선**이라는 두 축을 기준으로 설계되었다.

| 분리선 | 외부 측 | 내부 측 |
|---|---|---|
| 영업·공급 | **김도현 본부장** (외근 다수) | **이도현 실장** (자료·계약·정책 백오피스) |
| PM 라인 | **김태형 실장** (사이트 PM, 고객 측) | **박현숙 선임** (개발 PM, 구현 측) |
| 출시·커뮤니티 | — | **오유정 사원** (Release Manager + 커뮤니티 피드백) |

### 4.1. 김도현 본부장 — 본부 총괄 + 외부 미팅 다수
- **본부 차원 책임:** 본부 OKR 수립·점검, 본부원 1:1, 임원 주간보고(월 11시), 신규 사업 의사결정, 채용
- **PM 직접 담당:** 충남 AI 대전환 (본부 최대 신규 사업, 김태형 실장과 공동)
- **외부 미팅 책임 (외근 다수):**
  - 외부 의사결정자 미팅 (대기업·공공 C-레벨)
  - GPU 공급자 외부 미팅 (해외·국내)
  - IR / 언론 / 글로벌 파트너십
- **공동 오너십:** 모든 프로젝트의 마일스톤·리스크에 대한 최종 책임
- **시간 배분:** 외부 미팅 50% / 본부 운영(1:1·임원보고·회고) 30% / 충남 PM·신규 사업 Deep Work 20%
- **외부 미팅 슬롯:** 화 09:30–17:00 거의 전체, 목 09:30–17:00 거의 전체, 수·금 일부 가능

### 4.2. 이도현 실장 — 내부 백오피스 (기획·문서·정책)
- **도메인 오너십:**
  - 본부 운영서·운영루틴 매뉴얼 갱신 (이 문서의 직접 오너)
  - 모든 프로젝트의 **제안서·기획서·정책 문서** 작성·관리
  - **공급자/수요자 계약서·단가표·MOU** 정리·관리
  - 시민증·포인트 등의 **정책 (서비스 약관, 데이터 거버넌스, 개인정보 처리방침 등)**
- **PM 직접 담당:** 없음 — 사이트 PM은 김태형, 개발 PM은 박현숙
- **외부 미팅 책임:** 본부장·김태형 실장의 외부 미팅 **자료 준비 (전·후)**. 본인 단독 외부 미팅은 거의 없음
- **백오피스 구체 산출물 (월간 단위):**
  - 임원 주간보고용 자료 패키지 (월 11시 보고 1시간 전까지)
  - 신규 제안서 5건/분기 리드 작성 (본부장 검토)
  - 본부 운영서/루틴 매뉴얼 격주 갱신
  - 공급자/수요자 단가표 v1.0 → v1.X 관리
- **시간 배분:** 기획·문서 50% · 정책·계약 25% · 제안서 25%

### 4.3. 김태형 실장 — 사이트 PM 총괄 (6개 사이트 프로젝트 모두)
- **PM 담당 (모두 Primary PM):**
  - 평택 스마트시티 통합 시민앱
  - 광주 스마트시티 데이터허브
  - 용인 시티포인트 → 시티큐브 SaaS 전환
  - 칠곡 공공자전거
  - 김해 시민증 + 전자지갑
  - 충남 AI 대전환 (본부장과 공동)
- **사이트 PM의 책임 영역 (= 고객 측):**
  - 발주처(지자체·기관) 커뮤니케이션 단일 창구
  - 사이트별 일정·요구사항·진척 관리
  - 발주처 외부 미팅 (각 지자체 정기 점검)
  - 매주 월요일 위클리에 6개 프로젝트 1줄 상태 갱신 (필수)
- **박현숙(개발 PM)과의 분리선:** 김태형 = 고객/일정/요구사항, 박현숙 = 구현/아키텍처/기술 부채. 매주 화·목 09:30~10:00 정기 동기화
- **외부 미팅 슬롯:** 화·목 14:00–17:00 발주처 미팅 (지자체 1~2개 / 일)
- **시간 배분:** 사이트 PM 70% · 발주처 외부 미팅 20% · 데이터/AI 도메인 컨설팅 10%

### 4.4. 박현숙 선임 — 개발 PM + 시티큐브 코어 백엔드
- **개발 PM 책임 (= 구현 측):**
  - 개발팀(박현숙·오유정 + 향후 채용)의 **백로그 관리·기술 결정·코드 리뷰**
  - 시티큐브 SaaS / gcube 오픈소스 프로덕트의 **아키텍처 결정**
  - 사이트 PM(김태형)의 일정·요구사항을 **개발 일정으로 번역**, 무리한 일정은 즉시 협상
  - 오유정 멘토링 (수 16:00–17:00 페어 코딩 정기 슬롯)
- **개발 도메인 (메인 빌더):**
  - 시티큐브 코어 엔진(블록체인 인증 + 통합 포인트), 결제·정산
  - 김해 시민증 인증 모듈
  - 평택 시민앱 포인트 연동 백엔드
- **gcube 오픈소스 기여:** 인증·결제 관련 모듈 (메인 빌더는 오유정, 박현숙은 백엔드 기여)
- **시간 배분:** 시티큐브 SaaS 코어 50% · 개발 PM(백로그·리뷰·멘토링) 30% · 시민증 백엔드 20%

### 4.5. 오유정 사원 — 개발 + Release Manager + 커뮤니티 오너
- **개발 도메인 (메인 빌더):**
  - gcube 위에서 동작하는 비개발자/개발자용 오픈소스 프로덕트 (1주 단위 출시)
  - 사이트 프로젝트 프론트엔드 (용인 SaaS, 칠곡 모바일, 광주 데이터허브 UI)
  - 다국어(i18n) 빌드 파이프라인 구축
- **Release Manager 책임 (1주 출시 사이클의 단일 책임자):**
  - **금 12:00 = Code Freeze.** 그 이후 새 기능 PR은 다음 사이클로 이월
  - **금 14:00 = GitHub Release.** 시맨틱 버전·체인지로그·릴리스 노트 자동 생성
  - 출시 후 24시간 모니터링 (Discord 알림 등)
  - 분기당 출시 메트릭 (출시 수·외부 채택 수·오류율) 리포트
- **커뮤니티/외부 사용자 피드백 오너:**
  - gcube **Discord/X(트위터)/GitHub Issue** 트리아지 (라벨링·우선순위)
  - 매주 수 14:00–16:00 정기 트리아지 슬롯
  - 분기당 **외부 피드백 패턴 리포트** 작성 → 신규 프로덕트 아이데이션 입력
- **성장 목표:** 분기당 1개 이상 오픈소스 프로덕트의 메인 메인테이너로 성장
- **시간 배분:** 오픈소스 + 사이트 프론트 60% · Release Manager 운영 25% · 커뮤니티/피드백 15%

## 6. 성과 목표 (KPI / OKR)

성과는 **주간 / 분기 / 반기 / 연간** 4계층으로 관리한다.

### 6.1. 주간 KPI (전 본부원 공통)

- **출시 마일스톤 달성률:** 매주 금요일 17시 데모 기준 ≥ 90%
- **프로젝트 1줄 상태 업데이트:** 모든 프로젝트 매주 월요일 09:00 전 갱신 (100%, 김태형 책임)
- **수요·요구사항 수집:** 영업·파트너·외부 사용자 피드백 채널에서 신규 요구사항 ≥ 3건/주 기록 (오유정 커뮤니티 채널 포함)

### 6.2. 분기 OKR (Q2 2026 — 5/9 ~ 6/30 기준)

**Objective 1. 시티큐브 SaaS를 시장에 출시한다.** *(Lead: 박현숙 선임 / 보좌: 김태형)*

- KR2. 시티큐브 SaaS 멀티테넌시·과금 모듈 완성 (박현숙 리드)
- KR3. 평택·김해 통합 시민앱에 시티큐브 SDK 임베드 완료

**Objective 2. gcube 락인을 만드는 오픈소스 프로덕트를 1주 단위로 출시한다.** *(Lead: 오유정 사원)*

## 8. 본부장이 본부원에게 요청하는 것

1. **모든 산출물에 버전을 붙여라.** vX.Y.Z + 동일 메시지의 커밋. 롤백 가능 상태 유지.
2. **글로벌·다국어를 처음부터.** ko/en은 기본, 다른 언어는 SEO/AEO/GEO 우선순위에 따라 추가.
3. **락인이 없는 프로덕트는 만들지 마라.** gcube 또는 시티큐브와의 연결고리를 항상 가질 것.
4. **요구사항을 그냥 처리하지 마라.** 분류하고, 패턴을 찾고, 신규 프로덕트의 씨앗으로 보고하라.
5. **잊혀진 프로젝트를 만들지 마라.** 모든 프로젝트는 매주 1줄 상태가 갱신된다.
6. **막히면 30분 안에 손을 들어라.** 우선순위 충돌·외부 블로커는 즉시 본부장에게.
7. **본부장 외근 중에도 본부는 멈추지 않는다.** 이도현(백오피스)·김태형(사이트 PM)·박현숙(개발 PM)·오유정(Release)이 각자의 영역에서 자율 운영.
