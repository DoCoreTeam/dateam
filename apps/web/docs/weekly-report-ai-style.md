# 주간보고 AI 스타일 가이드

이 문서는 AI가 일일업무(daily_logs)를 주간보고 형식으로 변환할 때 따르는 스타일 기준이다.
AI는 이 문서를 매번 요청 시 읽어서 일관된 스타일로 주간보고를 작성해야 한다.

---

## 1. 주간보고 양식 구조

주간보고는 4개의 컬럼으로 구성된다:

| 컬럼 | 설명 |
|------|------|
| 구분 (category) | 업무 영역 또는 프로젝트명 (예: 영업, 개발, 기획, 운영, 마케팅) |
| 성과 (performance) | 이번 주에 실제로 완료하거나 진전시킨 업무 |
| 계획 (plan) | 다음 주에 진행할 예정인 업무 |
| 이슈/협조사항 (issues) | 문제점, 리스크, 타 부서 협조 필요 사항 |

---

## 2. 구분(category) 분류 기준

일일업무의 내용을 보고 아래 기준으로 구분을 결정한다:

- **단일 프로젝트/고객이 명확**하면 그 이름을 구분으로 사용 (예: "A사 제안", "B프로젝트 개발")
- **여러 동종 업무**는 상위 영역으로 묶는다 (예: "영업 활동", "개발", "기획", "운영")
- **구분 수**: 3~5개 권장. 너무 세분화하지 않는다.
- **이름**: 간결하고 명확하게. 동사형 금지 (예: "영업 활동 진행" → "영업")

---

## 3. 성과(performance) 작성 규칙

- **완료된 업무만**: entry_type이 'done' 또는 is_resolved=true인 업무 우선 사용
- **진행 중 업무**: 이번 주에 유의미한 진전이 있었다면 포함 가능 ("~진행", "~착수" 등으로 표현)
- **문체**: 명사형 또는 완료형 ("~완료", "~진행", "~수행", "~확인")
- **HTML 형식**: `<ul><li>내용</li></ul>` 형태로 작성
- **금지**: 미래 추측, 불확실한 내용, 수치 임의 생성

## 4. 계획(plan) 작성 규칙

- **출처**: entry_type이 'planned'인 업무, 또는 진행 중 업무의 다음 단계
- **문체**: 미래형 ("~예정", "~진행", "~완료 목표", "~검토")
- **HTML 형식**: `<ul><li>내용</li></ul>` 형태
- **구체성**: 막연한 계획보다는 구체적인 다음 액션 위주

## 5. 이슈/협조사항(issues) 작성 규칙

- **출처**: entry_type이 'blocker'인 업무, 또는 내용에 "문제", "이슈", "협조", "지연" 등이 포함된 업무
- **없으면**: 반드시 빈 문자열("") 반환. 없는 이슈를 지어내지 않는다.
- **문체**: 사실 중심 ("~필요", "~지연", "~협조 요청")

---

## 6. 전체 문체 원칙

- **간결하고 명확하게**: 불필요한 수식어 최소화
- **업무 보고서 어투**: 격식체, 명사형 종결 위주
- **중복 제거**: 같은 내용이 여러 일일업무에 있으면 하나로 통합
- **원본 내용 존중**: AI가 내용을 창작하지 않는다. 일일업무에 없는 내용은 추가하지 않는다.
- **수치 보존**: 원본에 수치가 있으면 그대로 사용

---

## 7. entry_type별 처리 방침

| entry_type | 처리 방침 |
|------------|----------|
| done | → 성과(performance)에 포함 |
| doing | → 성과(performance)에 포함 (진행 중 표기) |
| planned | → 계획(plan)에 포함 |
| blocker | → 이슈/협조사항(issues)에 포함 |
| note | → 관련 구분의 성과 또는 계획에 보조 정보로 포함 (선택) |

---

## 8. 출력 형식

순수 JSON 배열만 반환. 설명이나 마크다운 코드블록 없이:

```json
[
  {
    "category": "구분명",
    "performance": "<ul><li>성과1</li><li>성과2</li></ul>",
    "plan": "<ul><li>계획1</li></ul>",
    "issues": ""
  }
]
```

- 구분별 정확히 1개 항목 (중복 구분 절대 없음)
- 빈 필드는 `""` (null이나 "-" 사용 금지)
- category가 빈 문자열인 항목 포함 금지
