업무 지침 8원칙

본부원 모두가 따르는 운영 규칙

3.1 우선순위 원칙
시티큐브 완성 → gcube 오픈소스 → 고도화. 충돌 시 즉시 본부장 에스컬레이션.
3.2 1주 단위 출시
월요일 확정, 금요일 출시. 1주 못 끝나는 큰 기능은 MVP로 쪼갠다.
3.3 글로벌·다국어 우선
ko/en 기본 + SEO/AEO/GEO 메타 첫 빌드부터.
3.4 락인 설계
오픈소스마다 gcube/시티큐브 락인 포인트 1개 이상 필수.
3.5 버전 관리
vX.Y.Z + 동일 메시지 커밋. 롤백 항상 가능.
3.6 명확성 (No Ambiguity)
무엇·왜·언제까지·산출물 형태 4가지 항상 포함.
3.7 분석→발굴
요구사항을 패턴화 → 신규 프로덕트 시드로 등록.
3.8 다중 프로젝트 추적
월요일 09:00에 모든 프로젝트의 1줄 상태.

본부장이 본부원에게 요청하는 6가지

  1. 모든 산출물에 버전을 붙여라. vX.Y.Z + 동일 메시지의 커밋. 롤백 가능 상태 유지.
  2. 글로벌·다국어를 처음부터. ko/en은 기본, SEO/AEO/GEO 우선순위에 따라 추가.
  3. 락인이 없는 프로덕트는 만들지 마라. gcube 또는 시티큐브와의 연결고리를 항상 가질 것.
  4. 요구사항을 그냥 처리하지 마라. 패턴으로 분류하고 신규 프로덕트의 씨앗으로 보고.
  5. 잊혀진 프로젝트를 만들지 마라. 모든 프로젝트는 매주 1줄 상태가 갱신된다.
  6. 막히면 30분 안에 손을 들어라. 우선순위 충돌·외부 블로커는 즉시 본부장에게.
